# 阶段 05：剧本（单集）

> 工具：DeepSeek（纯文本推理）
> ⚠️ 必须先选结构模板，再填内容

## 前置条件

- `01-scripts/character-bible.md` 存在
- `01-scripts/series-bible.md` 存在（特别是分集大纲中本集的定位）
- `01-scripts/episode-skeleton.md` 存在（本集的四段骨架）
- `01-scripts/narrative-state.md` 存在（前面集的叙事状态）
- `03-storyboard/lighting-philosophy.md` 存在

## 流程

### 步骤 0：加载叙事上下文（🆕）

写剧本前，必须读取以下文件构建上下文：

| 文件 | 用途 |
|------|------|
| `series-bible.md` | 整季大纲、主题、角色弧全局定位 |
| `character-bible.md` | 角色四维 + 语言指纹（写对白的基础） |
| `episode-skeleton.md` | 本集的四段骨架（展开的锚点） |
| `narrative-state.md` | 前面集的累积状态（已揭露信息、角色当前态度、未兑现伏笔） |

> ⚠️ 不要读取前面所有集的完整剧本——narrative-state.md 已经浓缩了关键信息。
> 如果需要回溯某个具体细节（如某句台词的原文），再按需读取对应集的剧本。

### 步骤 1：选定结构模板

60 秒 = 4×15 秒段落。必须先选结构模板，再按四段填充。不能自由发挥。从以下 4 个模板中明确选一个：

| 模板 | 节奏 | 适用题材 |
|------|------|----------|
| **A：微三幕** | 钩子(3s) → 布设(10s) → 对抗(30s) → 高潮(10s) → 揭示/悬念(7s) | 完整叙事弧，有头有尾有钩子 |
| **B：反转结构** | 常态(10s) → 暗示异常(15s) → 反转爆发(25s) → 余震(10s) | 悬疑、惊悚、喜剧反转 |
| **C：提问-回答** | 抛出问题(5s) → 探索展开(45s) → 新问题(10s) | 推理、揭秘、信息驱动 |
| **D：对比结构** | A面(25s) → 切换 → B面(25s) → 并置揭示(10s) | 双线叙事、身份反差 |

### 步骤 2：生成剧本

产出 `01-scripts/ep01-script.md`：

```markdown

### 🪝 钩子（0-3 秒）
**画面**：{极具视觉冲击力的开场}
**对白/声音**：{一句话抓住注意力}
```

### 步骤 2.5：避免千篇一律——叙事节奏变体

> 同一套结构模板 × 同样的话术习惯 = 20 集看完像一集。必须在模板内注入变体。

#### 钩子类型轮换（每集换一种，20 集不重样）

| # | 钩子类型 | 示例 | 适用 |
|---|---------|------|------|
| 1 | 视觉奇观 | 日落时分的废墟全景，镜头缓缓升起 | 每季第 1 集、重大转折 |
| 2 | 对话开场 | 直接切入一句令人困惑的对白 | 悬疑、冲突集 |
| 3 | 声音优先 | 黑暗中只有呼吸声和心跳，画面渐亮 | 恐怖、紧张集 |
| 4 | 倒叙闪前 | 先展示结果（受伤/争吵后），再回溯原因 | 反转集 |
| 5 | 日常反差 | 极其平凡的日常画面，但台词暗示有事要发生 | 铺垫集 |
| 6 | 动作中开幕 | 角色正在做某件事（奔跑/格斗/坠落），无前情交代 | 动作集 |
| 7 | 空间穿行 | 镜头穿墙/穿窗/穿门，从外景一镜到底到内景 | 转场集 |
| 8 | 情绪特写 | 单一表情特写 3 秒，不说话，靠微妙表情变化推动 | 情感高潮集 |

#### 冲突引擎变体

不要每集都"两难选择"——以下类型轮换：

| 冲突类型 | 格式 | 频率建议 |
|---------|------|---------|
| 两难选择 | 障碍 → 选择（A或B，选哪个都损失）→ 后果 | 每 2-3 集一次 |
| 信息不对等 | A知道但B不知道 → B的误判导致行动 → 揭露 | 悬疑线常用 |
| 内在矛盾 | 角色想要A，但说出来的是B → 言行不一被抓住 | 情感线常用 |
| 环境压迫 | 外部环境（天气/空间/时间）逼迫角色行动 | 动作/生存线 |
| 道德困境 | 正确的代价 vs 错误的好处 | 每 3-5 集一次 |

#### 对白节奏变体

一集 60 秒（4段）内的对白节奏不能平铺。从以下模式中选一种：

| 节奏模式 | 描述 | 适合情绪 |
|---------|------|---------|
| 加速型 | 句越来越短、语速越来越快、停顿越来越短 | 争吵、追逐、倒计时 |
| 减速型 | 句越来越长、停顿越来越深、到最后沉默 | 诀别、告白、崩溃 |
| 乒乓球型 | 快速交替短句，每句 1-3 词，像乒乓球来回 | 斗嘴、审讯、机智对话 |
| 独白型 | 一个角色连续说 5-8 秒，另一个只给反应镜头 | 揭密、独白、宣言 |
| 断裂型 | 句子被动作/声音打断，说到一半停止 | 紧张、危险逼近 |

### 步骤 3：强制规则校验

**结构层面**
- [ ] 已选定结构模板（A/B/C/D）
- [ ] 每场戏标注"场景目的"（至少做两件事：推进剧情+揭示角色）
- [ ] 每集最多 4 次冲突引擎（分布在 4 段中，平均一段一次）
- [ ] 3 秒钩子有冲击力

**对白层面**
- [ ] 有互动感，你来我往
- [ ] 每句标注潜台词 + 对白行动 + 🎤 情感
- [ ] 角色对白风格不雷同（对照 character-bible 语言指纹）
- [ ] 表情丰富 + 动作具体 + 有停顿设计
- [ ] 🎤 同一集内 emotion 有变化弧线，一句话一个情绪
- [ ] 无空洞"嗯""啊"语气词
- [ ] 无全程点头摇头的单调动作

**镜头层面**
- [ ] 标注镜头序列（景别变化节奏）
- [ ] 非全程同一景别

### 步骤 4：用户确认

提交完整剧本。确认后进入步骤 5。

### 步骤 5：更新叙事状态追踪（🆕 强制）

剧本确认后，立即更新 `01-scripts/narrative-state.md`：

1. **已揭露信息**：本集观众获得了什么新信息？逐条追加
2. **未揭露信息**：本集是否兑现了之前计划的揭露？兑现的移入"已揭露"，标注实际集数
3. **角色当前状态**：
   - 态度变化轨迹追加本集区间（如 `→ {EP05 动摇}`）
   - Want 进度百分比更新
   - Flaw 松动程度更新
   - 最后关键台词/动作更新为本集的
4. **未兑现伏笔**：本集埋了新伏笔？追加。本集兑现了旧伏笔？标记 `done`
5. **上一集摘要**：覆盖写为本集的四段摘要

> ⚠️ 不更新 narrative-state 就不能进入故事板阶段。这是硬性门禁。
