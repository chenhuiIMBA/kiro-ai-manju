# 阶段 01：整季叙事大纲

> 工具：DeepSeek（纯文本推理，不需要图片/视频模型）

## 前置条件

- 项目已初始化，`STATE.md` 存在

## 输入

- 用户偏好（题材、角色方向、画风、集数、目标受众）

## 流程

### 步骤 1：收集创意输入

向用户询问：
- 故事题材（玄幻/都市/悬疑/甜宠/其他）
- 核心角色数量和基本设定方向
- 画风偏好（日漫/韩漫/国漫/写实/插画）
- 对话呈现风格：字幕模式 / 气泡台词模式 / 混合
- 每集时长（默认 60 秒）
- 总集数（如 20 集）
- 目标受众

### 步骤 2：画风细化（强烈推荐）

> "日漫"不是一种风格——90年代赛璐珞和2010s京都动画的视觉基因完全不同。细化画风大幅降低后续迭代次数。

| 维度 | 可选方向 | 示例 |
|------|---------|------|
| 年代参照 | 80s赛璐珞 / 90s经典 / 2000s数字 / 2010s现代 / 当代 | 2010s 现代数字动画 |
| 作品/工作室参照 | 京都动画风 / 新海诚风 / 吉卜力风 / MAPPA 风 | 新海诚风——极致光影、照片级背景 |
| 线条特征 | 粗重外轮廓 / 纤细均一 / 有笔触变化 / 无线条厚涂 | 纤细均一、干净利落 |
| 上色方式 | 硬边平涂 / 软边渐变 / 厚涂 / 水彩渗化 | 软边渐变、柔和过渡 |
| 阴影风格 | 分层色块式 / 柔和模糊 / 硬边阴影 / 网点纸 | 分层色块式、2-3 层明暗 |

将选定特征写入 `03-storyboard/visual-style-spec.md`。

### 步骤 3：生成整季大纲

产出 `01-scripts/series-bible.md`，内容结构：

```markdown
# {剧名} 整季大纲

## 主题
具体、可争论。不是"爱与正义"，而是"偏执的复仇最终吞噬复仇者本身"。

## 中心戏剧问题
驱动全剧的核心问题。每集都在回答这个问题的一小部分。
例："林小夜能在找到凶手的同时不失去自己的人性吗？"

## 系列大弧
主角从{起点}到{终点}的完整旅程，要体现主题。

## 人物弧（每个角色）
- **Want（表面欲望）**：conscious 想要什么
- **Need（深层需求）**：unconscious 真正需要什么（Want ≠ Need 才有戏剧）
- **Flaw（核心缺陷/错误信念）**：什么在阻碍他
- **Ghost（创伤事件）**：造成缺陷的过去经历

## 角色关系图
标注：信赖/敌对/暗恋/利用/保护/竞争/师徒等

## 分集大纲
| 集数 | 标题 | 核心事件 | 悬念钩子 | 主题推进 |
|------|------|---------|----------|---------|
| 01 | ... | ... | ... | ... |

## 冲突主线
- A 故事线（主线）：...
- B 故事线（支线/感情线）：...
```

### 步骤 4：用户确认

提交整季大纲，确认后进入步骤 5。

### 步骤 5：生成角色圣经（与步骤 3 同批确认亦可）

产出 `01-scripts/character-bible.md`（角色内在画像）：

```markdown
## {角色名}
### 外在
- 外貌：{描述}
- 服装：{描述}
- 标志性动作/物品：{咬嘴唇/转戒指/永远戴着帽子}

### 内在（四维）
- **Want**：{表面欲望}
- **Need**：{深层需求。关键：Want ≠ Need}
- **Flaw**：{核心缺陷/错误信念}
- **Ghost**：{创伤事件}

### 语言指纹
- 节奏：{快速/缓慢/时快时慢}
- 句长：{短句/长句/混合}
- 态度：{直接/迂回/讽刺/自我贬低/居高临下}
- 词汇特征：{文雅/粗俗/专业术语/网络用语}
- 口头禅或习惯句式：{...}
- 对白时的小动作：{摸耳朵/看地面/手不停比划}
```

### 步骤 6：生成分段骨架（episode-skeleton）

> ⚠️ 这是 20 集 × 4 段 = 80 个段落的全局叙事骨架。每段一句话，是后续逐集展开剧本的基础。
> 没有骨架就展开剧本 = 盲写，到第 10 集必然散架。

产出 `01-scripts/episode-skeleton.md`。

**编写方式**：分批生成，每次 5 集（20 段），每批带着前面已确认的骨架作为上下文。

```
第 1 批：EP01-EP05（用户确认后）
第 2 批：EP06-EP10（带 EP01-05 骨架作为上下文）
第 3 批：EP11-EP15
第 4 批：EP16-EP20
```

**格式**：

```markdown
# {剧名} 分段骨架

## EP01: {集标题}
- seg1: {一句话，≤30字，只写发生什么}
- seg2: {一句话}
- seg3: {一句话}
- seg4: {一句话} — 悬念：{钩子}

## EP02: {集标题}
...
```

**质量检查**（每批确认前）：
- 信息揭露节奏是否均匀（有没有连续 3 段以上在"聊天"而无新信息）
- 悬念钩子是否每集 seg4 都有
- 角色弧线是否平滑推进（对照 series-bible 的人物弧）
- 有没有重复的冲突模式（连续多集用同一种冲突类型）

### 步骤 7：升级分集大纲

骨架确认后，回头升级 `series-bible.md` 中的分集大纲表格，增加信息揭露和角色弧位置列：

```markdown
| 集 | 标题 | 核心事件 | 信息揭露 | 角色弧位置 | 集末状态变化 | 悬念钩子 |
|----|------|---------|---------|-----------|------------|---------|
| 01 | ... | ... | 观众得知林小夜失忆 | 0%（完全被 Flaw 驱动） | 从孤立→遇到顾鸢 | ... |
```

"角色弧位置"标注主角在 Want→Need 旅程中的百分比。这样 20 集的弧线一目了然。

### 步骤 8：初始化叙事状态追踪

产出 `01-scripts/narrative-state.md`（初始版本）。

此文件在每集剧本完成后更新（阶段 05 负责），此处只做初始化：

```markdown
# 叙事状态追踪 — {剧名}

**最后更新集**: EP00（初始化）

## 已揭露信息（观众已知）

（尚无，EP01 剧本完成后填入）

## 未揭露信息（按计划揭露集数）

| 计划集 | 内容 | 状态 |
|--------|------|------|
| EP{NN} | {从骨架和大纲中提取的关键信息揭露点} | pending |
...

## 角色当前状态

### {角色名}
- **对{角色B}的态度**：{初始态度}
- **Want 进度**：0%
- **Flaw 松动程度**：未松动
- **最后关键台词**：（尚无）

## 未兑现的伏笔

（尚无，随剧本推进填入）

## 上一集摘要

（尚无）
```

> "未揭露信息"在初始化时就从骨架中提取——哪些集计划揭露什么关键信息，提前登记。这是全剧的信息节奏蓝图。

### 步骤 9：用户最终确认

提交全部产出物（大纲 + 角色圣经 + 骨架 + 叙事状态初始版），用户确认后进入视觉基建阶段。

- [ ] 主题具体可争论（不是空洞口号）
- [ ] 每集有明确信息增量和主题推进
- [ ] 悬念钩子足够强
- [ ] 人物 Want ≠ Need 成立
- [ ] 人物弧有说服力
- [ ] 角色关系图产生天然戏剧冲突
- [ ] 每个角色有语言指纹
- [ ] 80 段骨架覆盖全部集数，每段一句话（🆕）
- [ ] 骨架中信息揭露节奏均匀，无连续 3 段以上空转（🆕）
- [ ] 分集大纲已升级，含信息揭露 + 角色弧位置列（🆕）
- [ ] narrative-state.md 已初始化，未揭露信息已从骨架提取（🆕）
- [ ] 用户已确认
