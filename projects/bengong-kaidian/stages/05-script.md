# 阶段 06：剧本（单集）

> 工具：DeepSeek（纯文本推理）
> ⚠️ 剧本写**镜头语义**（远景/推镜/切到），不写 seg 边界。seg 切分在阶段 07。

## 前置条件

- `01-scripts/character-bible.md` 存在
- `01-scripts/series-bible.md` 存在（特别是分集大纲中本集的定位）
- `01-scripts/episode-skeleton.md` 存在（本集 beat 骨架）
- `01-scripts/narrative-state.md` 存在（前面集的叙事状态）
- `01-scripts/lighting-philosophy.md` 存在

## 剧本 vs 视频的职责划分（重要）

| 工作 | 阶段 06 剧本 | 阶段 07 视频 |
|------|-------------|-------------|
| beat 节拍 | ✅ 展开为具体情节 | ❌ 不关心 |
| 镜头语义 | ✅ 远景/推镜/切到 | ✅ 继承并扩展 |
| 对白 + 情感 | ✅ 完整写出 | 继承 |
| seg 边界 | ❌ 不定 | ✅ 按规则切分 |
| ref-images 选择 | ❌ 不涉及 | ✅ 按场景和角色选 |
| 时间轴（精确到秒） | ✅ 写到场景级 | ✅ 切到段级 |

> 简言之：剧本**讲故事**并**给镜头建议**，视频阶段**按技术规则**把故事切成 Seedance 可执行的段。

## 流程

### 步骤 0：加载叙事上下文

写剧本前，必须读取以下文件构建上下文：

| 文件 | 用途 |
|------|------|
| `series-bible.md` | 整季大纲、主题、角色弧全局定位 |
| `character-bible.md` | 角色四维 + 语言指纹（写对白的基础） |
| `episode-skeleton.md` | 本集 beat 骨架（展开的锚点） |
| `narrative-state.md` | 前面集的累积状态（已揭露信息、角色当前态度、未兑现伏笔） |

> ⚠️ 不要读取前面所有集的完整剧本——narrative-state.md 已经浓缩了关键信息。
> 如果需要回溯某个具体细节（如某句台词的原文），再按需读取对应集的剧本。

### 步骤 1：选定结构模板

> ⚠️ **时长基准**：每集目标时长在项目初始化时通过 `--duration` 设定，见 STATE.md 的"每集目标时长"字段（默认 90s）。剧本总时长必须 ±5% 以内匹配这个值。

每集时长按 STATE.md 里的目标时长（默认 90s）。必须先选结构模板，再按模板填充内容：

| 模板 | 节奏 | 适用题材 |
|------|------|----------|
| **A：微三幕** | 钩子 → 布设 → 对抗 → 高潮 → 揭示/悬念 | 完整叙事弧，有头有尾有钩子 |
| **B：反转结构** | 常态 → 暗示异常 → 反转爆发 → 余震 | 悬疑、惊悚、喜剧反转 |
| **C：提问-回答** | 抛出问题 → 探索展开 → 新问题 | 推理、揭秘、信息驱动 |
| **D：对比结构** | A面 → 切换 → B面 → 并置揭示 | 双线叙事、身份反差 |

### 步骤 2：生成剧本

产出 `01-scripts/ep{NN}-script.md`：

**故事线推进**：A线:{...} | B线:{...} | ...

【0-3s，共 3s】钩子

### 步骤 3：避免千篇一律——叙事节奏变体

> 同一套结构模板 × 同样的话术习惯 = 20 集看完像一集。必须在模板内注入变体。

#### 钩子类型轮换

| # | 钩子类型 | 示例 | 适用 |
|---|---------|------|------|
| 1 | 视觉奇观 | 日落时分的废墟全景，镜头缓缓升起 | 每季第 1 集、重大转折 |
| 2 | 对话开场 | 直接切入一句令人困惑的对白 | 悬疑、冲突集 |
| 3 | 声音优先 | 黑暗中只有呼吸声和心跳，画面渐亮 | 恐怖、紧张集 |
| 4 | 倒叙闪前 | 先展示结果（受伤/争吵后），再回溯原因 | 反转集 |
| 5 | 日常反差 | 极其平凡的日常画面，但台词暗示有事要发生 | 铺垫集 |
| 6 | 动作中开幕 | 角色正在做某件事，无前情交代 | 动作集 |
| 7 | 空间穿行 | 镜头穿墙/穿窗/穿门，一镜到底 | 转场集 |
| 8 | 情绪特写 | 单一表情特写 3 秒，不说话，靠微妙变化推动 | 情感高潮集 |

#### 冲突引擎变体

| 冲突类型 | 格式 | 频率建议 |
|---------|------|---------|
| 两难选择 | 障碍 → 选择（A或B，选哪个都损失）→ 后果 | 每 2-3 集一次 |
| 信息不对等 | A知道但B不知道 → B的误判导致行动 → 揭露 | 悬疑线常用 |
| 内在矛盾 | 角色想要A，但说出来的是B → 言行不一被抓住 | 情感线常用 |
| 环境压迫 | 外部环境逼迫角色行动 | 动作/生存线 |
| 道德困境 | 正确的代价 vs 错误的好处 | 每 3-5 集一次 |

#### 对白节奏变体

| 节奏模式 | 描述 | 适合情绪 |
|---------|------|---------|
| 加速型 | 句越来越短、语速越来越快、停顿越来越短 | 争吵、追逐、倒计时 |
| 减速型 | 句越来越长、停顿越来越深、到最后沉默 | 诀别、告白、崩溃 |
| 乒乓球型 | 快速交替短句，每句 1-3 词 | 斗嘴、审讯、机智对话 |
| 独白型 | 一个角色连续说 5-8 秒，另一个只给反应镜头 | 揭密、独白、宣言 |
| 断裂型 | 句子被动作/声音打断，说到一半停止 | 紧张、危险逼近 |

### 步骤 4：强制规则校验

**结构层面**
- [ ] 已选定结构模板（A/B/C/D）
- [ ] 每场戏标注"场景目的"（至少做两件事：推进剧情+揭示角色）
- [ ] 3 秒钩子有冲击力
- [ ] 结尾悬念有效

**对白层面**
- [ ] 有互动感，你来我往
- [ ] 每句标注潜台词 + 对白行动 + emotion/speed/loudness
- [ ] 角色对白风格不雷同（对照 character-bible 语言指纹）
- [ ] 表情丰富 + 动作具体 + 有停顿设计
- [ ] 同一集内 emotion 有变化弧线
- [ ] 无空洞"嗯""啊"语气词
- [ ] 无全程点头摇头的单调动作

### 步骤 5：用户确认

提交完整剧本。确认后进入步骤 6。

> 批次确认分级见 `_convention.md` 「用户确认门禁分级」。
> ⚠️ 剧本**不在** P2 预授权豁免范围内——任何集数剧本都需要用户确认（P0/P1 阶段每集都审，P2 阶段剧本仍审但视频 prompt 可豁免）。

### 步骤 6：更新叙事状态追踪（强制门禁 4）

剧本确认后，**立即更新** `01-scripts/narrative-state.md`：

1. **已揭露信息**：本集观众获得了什么新信息？逐条追加
2. **未揭露信息**：本集是否兑现了之前计划的揭露？兑现的移入"已揭露"
3. **角色当前状态**：
   - 态度变化轨迹追加本集区间
   - Want 进度百分比更新
   - Flaw 松动程度更新
   - 最后关键台词/动作更新（格式：`EP{NN}-seg{N}` 或 `EP{NN}-beat{N}`）
4. **未兑现伏笔**：本集埋了新伏笔？追加。本集兑现了旧伏笔？标记 `done`
5. **上一集摘要**：覆盖写为本集的摘要

> ⚠️ **门禁 4**：不更新 narrative-state 就不能进入阶段 07。
