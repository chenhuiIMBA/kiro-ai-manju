# 阶段 05：剧本（单集）

> 工具：DeepSeek（纯文本推理）
> ⚠️ 剧本写**镜头语义**（远景/推镜/切到），不写 seg 边界。seg 切分在阶段 07。
>
> **阶段顺序说明（2026-05-06 重排后）**：04-character → 05-script → 06-scenes。剧本确定后，阶段 06 按剧本需要处理场景图（复用/修改/新建）。

## 前置条件

- `01-scripts/character-bible.md` 存在
- `01-scripts/series-bible.md` 存在（特别是分集大纲中本集的定位）
- `01-scripts/episode-skeleton.md` 存在（本集 beat 骨架）
- `01-scripts/narrative-state.md` 存在（前面集的叙事状态）
- `01-scripts/lighting-philosophy.md` 存在

## 剧本 vs 视频的职责划分（重要）

| 工作 | 阶段 05 剧本 | 阶段 07 视频 |
|------|-------------|-------------|
| beat 节拍 | ✅ 展开为具体情节 | ❌ 不关心 |
| 镜头语义 | ✅ 远景/推镜/切到 | ✅ 继承并扩展 |
| 对白 + 情感 | ✅ 完整写出 | 继承 |
| seg 边界 | ❌ 不定 | ✅ 按规则切分 |
| ref-images 选择 | ❌ 不涉及 | ✅ 按场景和角色选 |
| 时间轴（精确到秒） | ✅ 写到场景级 | ✅ 切到段级 |

> 简言之：剧本**讲故事**并**给镜头建议**，视频阶段**按技术规则**把故事切成 Seedance 可执行的段。

## 流程

### 步骤 0：加载叙事上下文

写剧本前，必须读取以下文件构建上下文：

| 文件 | 用途 |
|------|------|
| `series-bible.md` | 整季大纲、主题、角色弧全局定位 |
| `character-bible.md` | 角色四维 + 语言指纹（写对白的基础） |
| `episode-skeleton.md` | 本集 beat 骨架（展开的锚点） |
| `narrative-state.md` | 前面集的累积状态（已揭露信息、角色当前态度、未兑现伏笔） |

> ⚠️ 不要读取前面所有集的完整剧本——narrative-state.md 已经浓缩了关键信息。
> 如果需要回溯某个具体细节（如某句台词的原文），再按需读取对应集的剧本。

### 步骤 1：选定结构模板

> ⚠️ **时长基准**：每集目标时长在项目初始化时通过 `--duration` 设定，见 STATE.md 的"每集目标时长"字段（默认 90s）。剧本总时长必须 ±5% 以内匹配这个值。

每集时长按 STATE.md 里的目标时长（默认 90s）。必须先选结构模板，再按模板填充内容：

| 模板 | 节奏 | 适用题材 |
|------|------|----------|
| **A：微三幕** | 钩子 → 布设 → 对抗 → 高潮 → 揭示/悬念 | 完整叙事弧，有头有尾有钩子 |
| **B：反转结构** | 常态 → 暗示异常 → 反转爆发 → 余震 | 悬疑、惊悚、喜剧反转 |
| **C：提问-回答** | 抛出问题 → 探索展开 → 新问题 | 推理、揭秘、信息驱动 |
| **D：对比结构** | A面 → 切换 → B面 → 并置揭示 | 双线叙事、身份反差 |

### 步骤 3：避免千篇一律——叙事节奏变体

> 同一套结构模板 × 同样的话术习惯 = 20 集看完像一集。必须在模板内注入变体。

#### 钩子类型轮换

| # | 钩子类型 | 示例 | 适用 |
|---|---------|------|------|
| 1 | 视觉奇观 | 日落时分的废墟全景，镜头缓缓升起 | 每季第 1 集、重大转折 |
| 2 | 对话开场 | 直接切入一句令人困惑的对白 | 悬疑、冲突集 |
| 3 | 声音优先 | 黑暗中只有呼吸声和心跳，画面渐亮 | 恐怖、紧张集 |
| 4 | 倒叙闪前 | 先展示结果（受伤/争吵后），再回溯原因 | 反转集 |
| 5 | 日常反差 | 极其平凡的日常画面，但台词暗示有事要发生 | 铺垫集 |
| 6 | 动作中开幕 | 角色正在做某件事，无前情交代 | 动作集 |
| 7 | 空间穿行 | 镜头穿墙/穿窗/穿门，一镜到底 | 转场集 |
| 8 | 情绪特写 | 单一表情特写 3 秒，不说话，靠微妙变化推动 | 情感高潮集 |

#### 冲突引擎变体

| 冲突类型 | 格式 | 频率建议 |
|---------|------|---------|
| 两难选择 | 障碍 → 选择（A或B，选哪个都损失）→ 后果 | 每 2-3 集一次 |
| 信息不对等 | A知道但B不知道 → B的误判导致行动 → 揭露 | 悬疑线常用 |
| 内在矛盾 | 角色想要A，但说出来的是B → 言行不一被抓住 | 情感线常用 |
| 环境压迫 | 外部环境逼迫角色行动 | 动作/生存线 |
| 道德困境 | 正确的代价 vs 错误的好处 | 每 3-5 集一次 |

#### 对白节奏变体

| 节奏模式 | 描述 | 适合情绪 |
|---------|------|---------|
| 加速型 | 句越来越短、语速越来越快、停顿越来越短 | 争吵、追逐、倒计时 |
| 减速型 | 句越来越长、停顿越来越深、到最后沉默 | 诀别、告白、崩溃 |
| 乒乓球型 | 快速交替短句，每句 1-3 词 | 斗嘴、审讯、机智对话 |
| 独白型 | 一个角色连续说 5-8 秒，另一个只给反应镜头 | 揭密、独白、宣言 |
| 断裂型 | 句子被动作/声音打断，说到一半停止 | 紧张、危险逼近 |

### 步骤 6：更新叙事状态追踪（强制门禁 4）

剧本确认后，**立即更新** `01-scripts/narrative-state.md`：

1. **已揭露信息**：本集观众获得了什么新信息？逐条追加
2. **未揭露信息**：本集是否兑现了之前计划的揭露？兑现的移入"已揭露"
3. **角色当前状态**：
   - 态度变化轨迹追加本集区间
   - Want 进度百分比更新
   - Flaw 松动程度更新

4. **未兑现伏笔**：本集埋了新伏笔？追加。本集兑现了旧伏笔？标记 `done`

### 对白完整性检查清单（🆕 强制）

剧本提交给用户审核前，逐项验证正文与汇总表的一致性。任一项不通过必须先修复：

- [ ] **正文对白扫描**：扫描剧本正文中所有形如 `"..."` 的引号对白（包括功能性对白——太监宣旨、旁白、广播、群体齐声、画外呼喊等）
- [ ] **汇总表数量匹配**：正文对白数 = 汇总表对白数。如有差异必须显式标注原因
  - 允许差异的情形：
    - 功能性对白（宣旨/旁白/广播）——在汇总表中补登，标注"功能性对白"，emotion/speed/loudness 可留空并注明"无情感标注"
    - 群体齐声对白（众嫔妃齐声"参见皇上"）——合并为一条，角色字段写"群体"
  - 不允许的情形：正文写了对白但完全没进汇总表
- [ ] **功能性对白标注**：功能性对白在汇总表的"角色"列后加 `[功能性]` 标记，方便阶段 07 识别
- [ ] **时间标注**：每条对白在汇总表中的时间区间必须与正文节拍段落吻合

> ⚠️ **为什么必检**：阶段 07 的 seg 切分和 prompt 组装默认以"对白汇总表"为审计基准。如果正文里有未登记的对白，阶段 07 的覆盖自检会漏掉它，生成的视频会丢失该对白。这是 EP03 seg-2/seg-3 漏掉"太监宣旨"的根因（见 MEMORY.md 教训 #16）。

### 场景合理性检查清单

剧本确认前，逐场景检查以下项目。不通过则修改剧本后再确认：

- [ ] **角色出场合理性**：每个角色出现在该场景中是否有合理理由？身份/地位是否允许其在场？
  - 侍女/宫女不应出现在正式典礼的核心区域（可在门口/侧面/画外）
  - 低位角色不应与高位角色平起平坐（除非剧情刻意安排并有解释）
  - 角色不应"凭空出现"——需要交代如何到达该场景
- [ ] **空间逻辑**：角色的物理位置是否前后一致？
  - 上一场在 A 地的角色，下一场不能无交代地出现在 B 地
  - 同一场景内角色的相对位置不应无故跳变
- [ ] **时间逻辑**：场景之间的时间流转是否合理？
  - 白天/夜晚是否连贯
  - 事件发生的先后顺序是否自洽
- [ ] **行为动机**：角色的行为是否符合其当前状态（参考 narrative-state.md）？
  - 角色不应做出与其 Want/Flaw 矛盾的行为（除非是弧线转折点）

### 自动化验收工具（🆕 强制）

以上清单的"对白完整性检查"部分可以用脚本机械完成，减少人工漏检：

```bash
python3 <ai-manju>/scripts/validate_script.py ./01-scripts/ep{NN}-script.md
```

脚本会扫描剧本正文所有引号对白（含功能性对白——太监宣旨、旁白、广播、群体齐声），对照末尾"对白汇总"表。差异输出到 stdout，退出码非 0 即校验未通过。

**强制要求**：

- [ ] `validate_script.py` 对本集剧本返回 exit 0（所有对白一致）

### 可拍性自检（🆕 强制）

> 把阶段 07 的 seg 切分规则反向移植到剧本阶段。越早发现违反规则的场次越便宜——剧本改文字成本低，到 seg 切分再返工要重新对齐 prompt 和 ref-images。

对剧本正文的每场戏逐场核对以下项目（治理教训：阶段 07 切分规则 3-10 本应在剧本阶段就约束）：

- [ ] **每场戏角色面部特写同框数 ≤ 2**（对应 07-video 规则 4）
  - 3+ 角色同时需要面部清晰可辨的场次 → 考虑用远景处理群像，或拆成多场
  - 例：选秀大殿众嫔妃齐齐跪拜 → ✅ 远景群像，不需要多角色面部清晰
  - 反例：沈鹿溪、萧珩、柳如烟三人面对面辩论 → ⚠️ 应拆成两组正反打或改用正反打焦点切换
- [ ] **每场戏对白密度符合规则 3**（对应 07-video 规则 3）
  - 4-8 秒场：最多 1 句对白
  - 8-12 秒场：最多 2 句对白
  - 12-15 秒场：最多 3 句对白
  - 超密度的剧本内容应拆成多场或延长场次时长
- [ ] **无"同时多动作"复合描述**（对应 07-video 规则 3 与 🛡️ 防错规则 3）
  - 反例："A 转身的同时，右手从口袋抽出来，手指蜷缩，嘴唇微张，眼神从空白变柔软" → 单场里描述超过 2 个并行动作
