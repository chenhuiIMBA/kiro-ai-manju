# 阶段 06：场景图（按需生成）

> 工具：Seedream 5.0（场景图生成 / 图生图修改）
> ⚠️ 本阶段在剧本（阶段 05）之后、视频生成（阶段 07）之前执行
> ⚠️ 每集剧本确认后，处理该集涉及的场景图，然后进入视频生成

## 前置条件

- 当前集剧本（`01-scripts/ep{NN}-script.md`）已确认
- `narrative-state.md` 已更新(门禁 4)
- `01-scripts/lighting-philosophy.md` 存在
- `01-scripts/color-palette.md` 存在

## 核心理念

**场景图不再一次性提前生成全部场景**，而是"按需生成、逐集积累"：

1. 剧本确定了具体场景描述后再出图，更精准
2. 场景库随集数推进逐步丰富
3. 同一场景跨集复用同一张图（保持一致性）

> ⚠️ **场景图不需要 multiview**。场景的核心价值是色调+材质+空间氛围，一张高质量 main.png 足够传递给 Seedance。视频中的运镜变化靠 prompt 文字描述控制，不靠场景参考图的多视角。角色 multiview 保留（角色需要正/侧/背外观一致性）。

## 场景图使用优先级

每集剧本确认后，逐个检查该集涉及的场景，按以下优先级处理：

| 优先级 | 策略 | 条件 | 操作 |
|--------|------|------|------|
| **1. 复用** | 直接使用现有场景图 | `03-scenes/{场景名}/main.png` 或 `{布局名}.png` 已存在，且剧本中该场景的描述与现有图一致 | 无需操作，直接在阶段 07 引用 |
| **2. 修改** | 基于现有图做图生图 | `03-scenes/{场景名}/main.png` 已存在，但剧本中该场景需要变体（如"白天版→夜晚版"、"日常→宴席布置"、"完好→被破坏后"） | 用现有 main.png 做图生图，生成变体，存为 `main-{变体名}.png` |
| **3. 变体（群演布局）** | 基于现有 main.png 做图生图，锚定同场景多 seg 的群演布局 | 同场景 ≥ 2 个 seg 需要相同的群演布局（如请安两列嫔妃跪拜/宫宴列班大臣/街市路人布局），纯 prompt 文字描述不足以锚定几何关系 | 见下方「场景变体图（群演布局锚定）」章节 |
| **4. 新建** | 从零生成全新场景图 | 场景库中完全没有该场景 | 按步骤 2 流程生成 main.png |

> ⚠️ **优先级 3 vs 优先级 2 的区别**：优先级 2 是**视觉变化**（光线/时间/布置/状态变化）；优先级 3 是**群演锚定**（空间布局不变但增加群演作为环境的一部分）。两者 prompt 结构不同，见下方章节。

## 流程

### 步骤 1：场景需求分析

读取当前集剧本，列出该集涉及的所有场景，逐个判断优先级：

```markdown
## EP{NN} 场景需求分析

| 场景 | 场景库状态 | 优先级 | 操作 |
|------|----------|--------|------|
| 御膳房 | ✅ 已有 main.png | 1-复用 | 无需操作 |
| 冷月宫（夜间版） | ✅ 已有 main.png（夜间版） | 1-复用 | 无需操作 |
| 冷月宫（白天版） | ⚠️ 已有夜间版，需白天变体 | 2-修改 | 图生图：改为白天光线 |
| 赵太师密室 | ❌ 不存在 | 3-新建 | 从零生成 |
```

### 步骤 2：生成新场景图（优先级 4）

**Prompt 结构**（自然语言连贯描述）：

```
{画风关键词}，广角镜头，{场景类型描述}场景图，空无一人。
{一句话场景氛围总述——从剧本中提取}。
前景{具体物品/材质}，中景{空间主体/陈设}，后景{纵深/远景元素}，空间层次分明。
整体色调以{语义标记引用 color-palette.md}为主，{氛围/情绪描述}，
材质强调{3-5种具体材质}，电影级光影，无人物，无文字。
```

> ⚠️ 色调描述**必须**引用 `color-palette.md` 中定义的语义标记（如 `cool_key`、`warm_key`、`lantern_warm`），而不是临时编造颜色名。这确保跨场景一致性。

**生成命令**：

```bash
python3 <video-gen>/scripts/seedream.py create \
  -p "<场景prompt>" \
  --size 16:9 \
  --format png \
  -d ./03-scenes/{场景名}/
# 重命名为 main.png
mv ./03-scenes/{场景名}/seedream_*.png ./03-scenes/{场景名}/main.png
```

### 步骤 3：修改现有场景图（优先级 2）

基于现有 main.png 做图生图，生成变体。

**变体 Prompt 模板**（图生图）：

```
{画风关键词}，广角镜头，{场景类型描述}场景图，空无一人。
空间布局与参考图完全一致，建筑结构与陈设位置保持不变。
变化点：{明确描述改变了什么——光线/时间/布置/状态}。
{变化后的氛围描述}。
整体色调以{变体对应的语义标记}为主，材质与参考图一致，
电影级光影，无人物，无文字。
```

> ⚠️ 图生图变体的核心原则：**明确写"保持什么不变"+"改变什么"**。不写保持约束，模型可能重新发挥空间布局。

**生成命令**：

```bash
python3 <video-gen>/scripts/seedream.py create \
  -p "<变体描述prompt>" \
  -i ./03-scenes/{场景名}/main.png \
  --size 16:9 \
  --format png \
  -d ./03-scenes/{场景名}/
# 重命名为 main-{变体名}.png
mv ./03-scenes/{场景名}/seedream_*.png ./03-scenes/{场景名}/main-{变体名}.png
```

**变体命名规则**：
- `main.png` — 默认版本（最常用的光线/布置）
- `main-daytime.png` — 白天版
- `main-night.png` — 夜间版
- `main-banquet.png` — 宴席布置版
- `main-damaged.png` — 被破坏后版
- `main-{自定义}.png` — 其他变体

> 💡 阶段 07 的 ref-images 槽位-场景传入的是 `main.png`（默认）或 `main-{变体名}.png`（变体），按剧本中该 seg 实际使用的场景版本选择。

### 步骤 3.5：生成场景变体图（优先级 3 — 群演布局锚定）🆕

**适用场景**：同场景 ≥ 2 个 seg 需要固定的群演布局（如选秀大殿两列嫔妃跪拜、宫宴列班大臣、集市路人布局），单靠 prompt 文字描述"两列纵队跪拜"容易被模型漂移成"几人一排并肩站立"或"三三两两散布"，布局无法跨 seg 保持一致。

**核心设计原则**（严格遵守）：

> ⚠️ **场景变体图只含环境布局，严禁包含角色圣经里的主要角色**（有 `04-character/{角色}/` 三件套的角色）。只能有：
> - 建筑空间（立柱/穹顶/地面/家具）
> - 光线/色调/材质
> - **群演布局**（没有角色圣经个体条目的群体布景，如众嫔妃/众大臣/路人），仅作为环境的一部分，**不渲染面部细节**
>
> 主要角色必须由 Seedance 的 ref-images（multiview）+ prompt 文字描述联合注入，**不能烙在场景图里**。否则该图只能用于该角色在场的剧情，失去跨 seg / 跨集的复用价值，且与 Seedance 的角色 ref 产生语义冲突。

**判断标准**：在 `04-character/` 下有目录的 → 主要角色，不能进变体图；没有的 → 群演，可作为环境布局进变体图。

**变体图 Prompt 模板**（图生图，基于 main.png）：

```
{画风关键词}，广角镜头{俯拍/平视/具体视角}。

空间布局、建筑结构、材质与色调与参考图完全一致——{复述 main.png 的关键空间元素}。

在参考图的同一空间内添加群演环境布局：{具体描述群演群体——位置、数量、姿态、服饰色系}。{不参与布局的区域保持空（如"殿中央地面空出来无人""殿门口边缘无人"，明确边界避免模型自由发挥）}。

整体色调与参考图完全一致——{复述 main.png 色调}，电影级光影，无文字，无水印，无字幕，人物无明确面部细节（仅提供群演布局参考）。
```

**关键要点**：
- **显式声明"空间/材质/色调与参考图完全一致"**（多次重复强化）
- **主要角色在场景里的位置区域必须明确为"空"**（如"殿中央地面空出来无人"），避免模型自由发挥填入不该有的角色
- **群演面部无细节**：只要几何布局，不要面部
- **必须基于 main.png 图生图**（`-i main.png`），不能文生图——否则空间细节会漂移

**命名规则**：`{场景}/{布局名}.png`，例如：
- `xuanxiu-dadian/jibai-buju.png` — 选秀大殿集体跪拜布局（两列嫔妃跪拜）
- `gongyan-dadian/liban-buju.png` — 宫宴大殿列班布局（大臣列班）
- `yushangfang/zaoshi-buju.png` — 御膳房早市布局（宫女忙碌备膳）

**生成命令**：

```bash
python3 <video-gen>/scripts/seedream.py create \
  -p "<变体图 prompt>" \
  -i ./03-scenes/{场景名}/main.png \
  --size 16:9 \
  --format png \
  -d ./03-scenes/{场景名}/
# 重命名为 {布局名}.png
mv ./03-scenes/{场景名}/seedream_*.jpg ./03-scenes/{场景名}/{布局名}.png
# 注意：Seedream 的 --format png 只影响 API 返回格式，下载文件扩展名由 URL 决定（可能是 .jpg）；
# 重命名时显式写 .png 即可
```

**使用方式**（在阶段 07）：

- 同场景需要该布局的所有 seg，ref-images 的"槽位-场景"传入 `{布局名}.png` 替代 `main.png`
- seg 需要不同 seg 独立机位（如一个 seg 是近景特写，另一个是远景全景），所有机位都可以共用同一张变体图作为空间锚点
- prompt 里引用时显式说明"（图片1，{布局描述}参考图）"，如"选秀大殿（图片1，集体跪拜群演布局参考图）"

**成本分析**：

| 方案 | tokens/seg | 跨 seg 一致性 | 资产复用性 | 推荐度 |
|------|-----------|-------------|-----------|--------|
| 纯 prompt 文字描述 | ~217K/10s | 低（容易漂）| — | ❌ |
| 视频延长（reference_video，传入前段 14s 视频）| ~519K/10s（2.4x）| 高 | — | 特殊场景 |
| **场景变体图（本方案）** | ~217K/10s + 17.8K/变体（一次性）| 高 | **可多 seg 多集复用** | ✅ **默认首选** |

### 步骤 4：资产登记

每次生成或修改后，立即更新该场景的 `assets.md`（即时登记门禁 2）。

```markdown

| main-{变体名}.png | "{变体完整 prompt 原文}" | -i main.png --size 16:9 --format png | task_zzz | 2026-05-XX |
| {布局名}.png | "{群演布局变体 prompt 原文}" | -i main.png --size 16:9 --format png | task_www | 2026-05-XX |

## 产出文件

| 文件 | 来源 | 下载 URL |
|------|------|----------|
| main.png | Seedream 5.0 文生图 | https://... |
| {布局名}.png | Seedream 5.0 图生图（参考 main.png）| https://... |

## 场景描述

**场景类型**：{室内/室外/广场/长道/...}
**主色调**：{引用 color-palette.md 语义标记，如 cool_key + lantern_warm}
```

**登记规则**：
- Prompt 列必须是发送给 Seedream 的**完整原文**，不是摘要
- 每生成一张图立即登记，禁止批量补录（门禁 2）
