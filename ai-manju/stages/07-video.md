# 阶段 07：视频 Prompt 组装 + 依赖链并行生成

> 工具：Seedance 2.0（多模态参考生视频）
> ⚠️ 所有 seg 的 prompt 一次性组装并审核，然后按**依赖链并行**提交

## 前置条件

- 本集剧本已确认
- `narrative-state.md` 已更新（门禁 4）
- `03-scenes/` 场景库已就绪（本集用到的场景都存在）
- `04-character/{角色}/multiview.png` 和 `voice-ref.mp3` 存在

## 进入本阶段前的强制校验（🆕 门禁 4 自检）

**AI 执行进入阶段 07 前必须完成以下自检，任一未通过即回退到阶段 05：**

1. 读取 `01-scripts/narrative-state.md`，确认文件头部的 `最后更新集` = `EP{当前集}`
2. 如不等于当前集（例如仍是前一集），说明阶段 05 的步骤 6 未正确执行
3. 此时禁止进入视频组装，立即回退到阶段 05 步骤 6 完成 narrative-state 更新
4. 更新完成后重新进入阶段 07

> ⚠️ narrative-state 门禁不过不得绕过——未来集的视频 prompt 组装依赖本集的状态快照。跳过会导致角色态度/已揭露信息不连续，漂移风险显著上升。

## P0 工作流（强制，仅 P0 等级适用）

P0 等级用于项目初期、核心资产、工作流变更验证期。必须严格按以下 7 步执行，**禁止跳步、禁止 batch_gen.py 批量提交**。

### Step 1：统一切分 seg

按"流程"章节步骤 1 的规则把全集切分成 seg。**一次切完全集**，不允许边切边组装。

产出：seg 切分方案（时长、场景、角色、依赖链、首帧来源）

### Step 2：更新 assets.md 片段划分表

把 Step 1 的切分结果写入 `05-videos/ep{NN}/assets.md`：
- 片段划分表（10 行，状态全 pending，task_id 和 seed 留空）
- 依赖链 DAG 图
- 每个 seg 的**段详情骨架**（时长/场景/首帧来源/依赖，prompt 留空待 Step 4 填）

> ⚠️ 这是门禁 2（即时登记）的前置动作——必须先有骨架，后续才能逐项填入。

### Step 3：seg 切分自检（强制门禁）

自检 Step 1 的切分方案与剧本的对应关系。任一不通过，AI **自行修正切分方案并重跑 Step 2**，直到全部通过才进入 Step 4。

**自检清单**：

- [ ] **对白覆盖（双源对照）**：
  - **源 1 — 剧本正文扫描**：扫描剧本正文中所有引号对白（含功能性对白：太监宣旨、旁白、广播、群体齐声、画外呼喊等）
  - **源 2 — 对白汇总表**：剧本末尾的对白汇总表（理论上应覆盖正文所有对白，见阶段 05 验收）
  - **对照结果**：正文扫描得到的所有对白 + 汇总表的所有对白 = 全集对白清单。清单中每一句都必须被分配到某个 seg 的时间段
  - ⚠️ **禁止只查汇总表**：汇总表可能漏登功能性对白。必须**先扫正文**，再用汇总表交叉验证
- [ ] **功能性对白归属规则**：功能性对白（宣旨/旁白/广播/群体齐声）必须明确归属到某个 seg 的某个节拍，且归属位置符合剧情因果（如"宣旨 → 出列"必须宣旨节拍在前）
- [ ] **关键动作覆盖**：剧本"画面"字段中描述的所有关键动作（表情变化、肢体动作、道具互动、角色进出场）都有对应 seg 覆盖
- [ ] **时长对齐**：所有 seg 时长总和 = 剧本总时长（偏差 ≤ 2s）
- [ ] **场景边界对齐**：剧本中的场景切换都是 seg 边界（不跨 seg 切场景）
- [ ] **时长约束**：每个 seg 4-15 秒
- [ ] **对白密度**：符合规则 3（4-8s 最多 1 句，8-12s 最多 2 句，12-15s 最多 3 句）
- [ ] **同场景连续 seg 的首帧链**：正确（前一 seg 尾帧 → 后一 seg 起始画面）
- [ ] **场景跳转的首帧处理**：跳转点必须是文生视频（规则 5）

**自检不通过的修复策略**：
- 对白/动作遗漏 → 新增 seg 或扩展现有 seg 时长
- 时长超长 → 拆分 seg
- 时长过短 → 与相邻 seg 合并（若同场景）
- 场景切换跨 seg → 在切换点强制切断 seg
- 功能性对白归属错位 → 调整 seg 归属或节拍顺序

自检报告写入 assets.md 末尾，作为审计凭证。**自检报告必须明确列出"正文对白扫描结果"**（不能只列汇总表条目），见下方模板：

```markdown
### 对白覆盖（N/N ✅）

**正文扫描结果**（包括功能性对白）：

| 对白时间 | 角色 | 台词 | 类型 | 分配 seg | seg 时间段 | ✅ |
|---------|------|------|------|---------|----------|---|
| 5-7s | 太监 | 皇上有旨，新入宫嫔妃逐一上前请安 | 功能性 | seg-2 | 5-18s | ✅ |
| 22-24s | 小桃 | 左脚先迈 | 情感性 | seg-3 | 18-32s | ✅ |
| ... | ... | ... | ... | ... | ... | ... |

**与汇总表交叉验证**：正文 N 句 vs 汇总表 M 句。差异 K 句均为功能性对白，已显式标注。
```

### Step 4：统一组装所有 seg 的 prompt

Step 3 通过后，一次性把**全集所有 seg** 的 prompt 写完。

**产出双写**（强制）：每个 seg 的 prompt 必须同时写入以下两处：

1. `05-videos/ep{NN}/seg-config.py` 的 `prompt` 字段（batch_gen.py 的输入）
2. `05-videos/ep{NN}/assets.md` 的对应段详情块的 `#### Prompt (v1, 当前)` 区段 + `#### ref-images（逻辑槽位 → 物理编号）` 表 + `#### audio` 区段 + `#### 命令参数` 区段

> ⚠️ **为什么双写**：seg-config.py 是执行输入，assets.md 是审计凭证（门禁 2）。Step 4 只写 seg-config.py 不写 assets.md = 段详情块在 Step 7 才回填 prompt = 会话中断时丢失 prompt 原文。

**不允许"边审边组装"**——必须全部组装好，形成完整方案，再进 Step 5 逐 seg 审核。

组装时遵循：
- `references/prompt-guide.md` §四「图片指代规则」（内联指代、首次出现标注、不加拼音）
- `references/prompt-guide.md` §六「提示词优化 Checklist」
- 本阶段步骤 2 的 prompt 模板

### Step 5：逐 seg 发送 prompt 给用户审核

按 seg 顺序（按依赖链从根节点开始），逐个发送以下内容给用户：

- 完整 prompt（含画风、反面约束、起始画面声明）
- ref-images 列表（逻辑槽位 + 物理编号 + 文件路径）
- audio 列表
- 命令参数（--duration / --ratio / --generate-audio / --return-last-frame）

用户可在审核时**修改 prompt 或参数**。修改后重新走 Step 5 发送最新版，直到用户明确"通过"。

### Step 6：提交前 prompt 自检（强制门禁）

用户通过后、提交 Seedance API 前，AI 做最后一次自检：

**一致性自检**
- [ ] **prompt 与 seg 描述一致**：prompt 中的动作/对白/情绪与 Step 3 通过的 seg 切分方案中该 seg 的备注一致
- [ ] **元素完整性**：剧本中该 seg 时间段内的所有对白、关键动作都在 prompt 中有体现
- [ ] **图片编号一致**：prompt 中的"图片N"编号与实际传入 ref-images 顺序一致
- [ ] **起始画面编号正确**：同场景衔接段的"参考图片N作为起始画面"中 N = ref-images 总数
- [ ] **命令参数完整**：--ratio / --duration / --generate-audio / --return-last-frame 齐全，--audio 按需

**景别承接自检（🆕）**
- [ ] **起始画面景别 ≈ prompt 0-Ns 景别**：如果本 seg 有起始画面（`start_frame != None`），扫描起始画面来源（上一 seg 尾帧所在节拍的景别），与本 seg 0-Ns 节拍的景别比较
  - 同景别或相邻景别（差 ≤ 1 级）→ 通过
  - 跨 ≥ 2 级景别（如特写/极近特写 → 全景/远景）→ **不通过，必须在 0-Ns 节拍加入运镜过渡**（如"镜头从上一 seg 的特写缓慢向后拉远并俯降到全景平视"），让模型用运动过渡而不是硬切
  - ⚠️ **严禁跨景别硬切**：模型会自由运镜重决构图，连带朝向/位置/群演布局都会漂移。典型事故：起始画面是中近景仰拍沈鹿溪特写 → 0-5s 写远景全景 → 模型把沈鹿溪转身朝向大门
- [ ] **景别递进节奏**：prompt 中多个节拍的景别变化是否合理——通常"远景 → 中景 → 特写"（情绪推进）或"特写 → 中景 → 远景"（情绪释放），不宜频繁跳跃
- [ ] **起始画面朝向与首节拍一致**：如果起始画面中人物朝向特定（如"面朝高台"/"背对镜头"），本 seg 0-Ns 描述必须沿用该朝向，否则模型会重决朝向

**剧本忠实度自检（🆕）**
- [ ] **镜头意图覆盖**：剧本中该 seg 时间段的"镜头意图"字段已在 prompt 的某个节拍中有对应实现
- [ ] **潜台词视觉化**：剧本中该 seg 对白的"潜台词"已转化为 prompt 中的具体动作或情绪标注（emotion/speed/loudness）
- [ ] **节奏词兑现**：剧本中"漫长沉默"/"长镜头"/"急切"等节奏词在 prompt 中体现为具体时长分配
- [ ] **情感强度匹配**：prompt 的 emotion/speed/loudness 与剧本对白汇总表一致，不擅自削弱

**prompt 规范自检**
- [ ] **每节拍必填字段齐全**：景别 + 机位 + 主体位置 + 运镜（含叙事意图）+ 动作描述
- [ ] **图片内联指代**：角色/场景在首次出现时标注图片编号，后续不重复
- [ ] **不加拼音**：角色名后只跟图片编号，无拼音标注

自检通过才能提交。不通过则回到 Step 5（告知用户问题并附修改建议）。

### Step 7：提交视频生成任务 + 等待 + 质量确认

- 调用 Seedance API 提交该 seg
- 轮询等待完成（AI 主动轮询，完成后立刻通知用户，禁止"等用户来问"）
- 下载视频 + 尾帧，重命名为 `seg{N}.mp4` 和 `lastframe_seg{N}.png`
- 更新 **assets.md 片段划分表**：该 seg 行的状态 pending → completed，填入 task_id + seed
- 更新 **assets.md 段详情块**的版本历史表：追加 v1 行（task_id + seed + 时间 + 说明）
- 发送用户确认质量
- 用户通过 → 回到 Step 5 处理下一个 seg
- 用户不通过 → 按用户反馈调整 prompt（回到 Step 5 该 seg），必要时把调整理由反哺到 prompt-guide.md

> ⚠️ **主动轮询规则**：Step 7 的等待阶段 AI 必须主动检查生成状态，完成后第一时间通知。不允许启动后台进程后就"等用户来问"。

> ⚠️ **资产清单更新粒度**：每个 seg 完成后即时更新 assets.md（门禁 2），不允许攒几个 seg 再批量补录。assets.md 的段详情块已在 Step 4 预填 prompt，此处只需补 task_id + seed + 版本历史。

## 流程

### 步骤 1：seg 切分

拿到确认的剧本后，按以下规则将一集拆分为多个 Seedance 任务。

#### 基础规则

- **规则 1：时长范围** — 每个 seg 4-15 秒，同一 seg 内保持同一场景
- **规则 2：尾帧衔接** — 同场景相邻 seg 中，后者的**起始画面**来自前者的尾帧（通过 `--ref-images` + prompt 声明实现）

#### 优化规则

- **规则 3：对白密度限制**
  - 4-8 秒段：最多 1 句对白
  - 8-12 秒段：最多 2 句对白
  - 12-15 秒段：最多 3 句对白
  - 超过密度的剧本内容应拆成多 seg

- **规则 4：单 seg 尽量 ≤ 2 个角色面部特写同框**（分镜层面的构图建议，非硬约束）— 3+ 角色面部同时清晰可辨时一致性和动作协调易出错
  - 涉及 3+ 角色的时间轴，尽量视角锁定在 1-2 个角色，其他作为背景或画外音
  - 展示群像（如选秀大殿全景）用远景，不靠面部识别
  - ⚠️ 此规则约束的是**画面构图**（谁的脸需要清晰），不约束 ref-images 传入数量。画面中出现的所有有角色库的角色都应传入参考图

- **规则 5：场景跳转段起始画面必须文生视频** — 避免场景 B 起始画面继承场景 A 残影

- **规则 6：单 seg 只允许一种主运镜** — 复合运镜（推完再拉、推完再摇）执行经常失败

- **规则 7：回忆/闪回独立成段** — 回忆段起始画面用文生视频（哪怕前段是同场景），prompt 中明确"闪回，画面偏暖黄，老照片质感"

- **规则 8：对白段起始画面留 1-2 秒预热** — Seedance 对白启动有 0.3-0.8 秒延迟。对白段开头安排静默动作（看向对方、深呼吸），再进入对白

- **规则 9：段尾 0.5 秒留白** — 每段最后 0.5 秒尽量静态，便于下一段用尾帧作为起始画面时更自然

#### 片段划分表格式（assets.md 标准格式，强制）

写入 `05-videos/ep{NN}/assets.md`，严格遵守 `_convention.md` 的视频 seg 标准格式：

```markdown
## 片段划分表

| seg | 时长 | 状态 | task_id | seed | 首帧来源 | 依赖 | 备注 |
|-----|------|------|---------|------|---------|------|------|
| 1 | 10s | pending | — | — | 文生视频 | — | 幼年沈鹿溪揉面团 |
| 2 | 8s | pending | — | — | seg-1 尾帧 | seg-1 | 爷爷教做面 |
| 3 | 12s | pending | — | — | 文生视频 | — | 萧珩高坐审视（场景跳转） |
| 4 | 6s | pending | — | — | seg-3 尾帧 | seg-3 | 萧珩特写 |
| ... | ... | ... | ... | ... | ... | ... | ... |
```

**首帧来源规则**：
- **文生视频**：场景跳转 / 回忆闪回（规则 5、7）
- **seg-N 尾帧**：同场景连续叙事时，上一 seg 尾帧作为起始画面（规则 2）

**依赖列规则**：
- 若 `首帧来源 = 文生视频`，`依赖 = —`
- 若 `首帧来源 = seg-N 尾帧`，`依赖 = seg-N`
- 该列决定并行提交的拓扑顺序（见步骤 5）

### 步骤 2：组装每个 seg 的 Prompt

为每个 seg 编写 Seedance prompt，采用**时间段节拍拆分结构**。

> ⚠️ **Prompt 审核门禁（门禁 1）**：所有 seg 的 prompt 组装完成后，对照 `references/prompt-guide.md` §六「提示词优化 Checklist」自查。**批次审核**——一整集所有 seg 的 prompt + ref-images 组合 + 参数一次性发给用户审核，确认后才可提交。

**Prompt 模板**：

> ⚠️ **单源声明**：Seedance 视频 prompt 的完整模板、示例和格式规则**统一定义在** `references/prompt-guide.md` §四「视频生成 Prompt（Seedance 图生视频）」。本节只列出本阶段特有的强制字段补充，不重复模板。
>
> 执行时打开 `references/prompt-guide.md` §四 `漫剧专用模板`，按模板填充。

**本阶段对模板的强制补充**：每节拍必填以下字段（🆕 强制，prompt-guide 模板的展开细化）：

| 字段 | 取值范围 | 示例 |
|------|---------|------|
| **景别** | 特写 / 中近景 / 中景 / 中远景 / 远景 / 极近特写（眼睛/嘴唇） | 极近特写 |
| **机位** | 正面 / 侧面 / 背拍 / 俯拍 / 仰拍 / 主观视角 | 俯拍 |
| **主体位置** | 画面 9 宫格之一（左上/中上/右上/左中/中/右中/左下/中下/右下） | 画面中央 |
| **运镜** | 固定 / 推进 / 拉远 / 跟拍 / 摇向 / 升降 / 移焦；**必须带叙事意图** | 从玉佩摇到嘴角——把伤痕拉到笑意 |
| **动作描述** | 本节拍发生什么 | 萧珩嘴角微微上扬一丝弧度 |

**为什么必填**：
- 不写机位 → 模型随机机位，画面基调不稳定
- 不写主体位置 → 主体位置随机，构图混乱
- 不写运镜意图 → 运镜变成功能性（推/拉）而非叙事性，情绪断裂

**本阶段对模板的格式要点强调**（以下为 prompt-guide §四 模板的关键落地点，不遵守会导致自检失败）：
- **画风关键词**：独立首行
- **反面约束**：紧跟画风关键词独占一段，固定写"视频中不要出现水印。保持无字幕，避免画面生成字幕。"
- **起始画面声明**（条件必选）：同场景相邻 seg 在反面约束后加"参考图片{N}作为起始画面"（N 是尾帧在 `--ref-images` 中的**物理编号**，按本 seg 实际传入顺序计算）。场景跳转段不加此行
- **起始画面是软约束（🆕）**：多模态参考生视频模式下"起始画面"属于 `reference_image`，API 层面是**软约束**，模型可以不严格遵守。如果本 seg 0-Ns 节拍的景别/构图与起始画面差异过大，模型会"自由运镜"重决构图，导致朝向/位置/群演布局漂移。**对策**：保持 0-Ns 节拍的景别与起始画面接近（同景别或相邻景别），如需跨景别必须写明运镜过渡（见下一条）
- **景别衔接规则（🆕）**：同场景相邻 seg 的**0-Ns 节拍景别**应与**上一 seg 末节拍景别**（即起始画面景别）匹配或相邻：
  - **匹配**（同景别）→ 直接衔接，prompt 开头节拍可以直接是目标景别
  - **相邻**（差 1 级景别）→ 可直接衔接或用短运镜过渡（0-1s 内）
  - **跨级**（差 ≥ 2 级，如特写→远景）→ **必须用运镜过渡节拍显式实现**，不得硬切。例如"0-3s：中近景起始逐渐拉远至全景 + 先仰拍后俯降至平视 + 镜头缓慢向后拉远并俯降——承接起始画面的仰视定格，画面逐渐展开显露整个大殿"
- **时间段节拍**：`0-4s:`、`4-8s:` 等，每节拍 3-5 秒，每节拍一件事
- **景别术语**：特写/中近景/中景/中远景/远景（5 种，不混用英文缩写）
- **镜头动词**：推进/拉远/跟拍/摇向/闪白/切入（简洁中文）
- **对白格式**：双引号包裹，紧跟动作后同节拍内；**对白后圆括号内注入 emotion/speed/loudness 三维情感标注**（从剧本对白汇总表复制）
- **情感标注作用**：Seedance 配音软提示，影响语气/语速/音量
- **环境音**：`<>` 包裹，和对白视觉分离
- **无对话段**：末尾加"无对话。"
- **图片编号**：角色名后只跟图片编号（如 `萧珩（图片3）`），不加拼音。同一角色只在首次出现时标注编号（见教训 #11）

### 步骤 3：ref-images 逻辑槽位 + 物理索引

每个 seg 的 ref-images 按**逻辑槽位**组织，但 Seedance 按**物理顺序**编号"图片N"。两者的映射必须在 assets.md 中显式记录。

#### 逻辑槽位定义

| 逻辑槽位 | 内容 | 默认路径 |
|---------|------|----------|
| 槽位-场景 | 场景主视角图 | `03-scenes/{场景名}/main.png` |
| 槽位-角色A | 角色A 多视图组合图 | `04-character/{角色A}/multiview.png` |
| 槽位-角色B | 角色B 多视图组合图 | `04-character/{角色B}/multiview.png` |
| 槽位-起始画面 | 本 seg 的起始画面（来源：上一 seg 的尾帧文件） | `05-videos/ep{NN}/lastframe_seg{N-1}.png` |
| 槽位-补充 | 手部/道具/换装参考图 | 按需生成（**可 1-3 张**） |

**传入顺序**（固定）：场景 → 角色A → 角色B ... → 起始画面 → 补充。缺失的槽位**跳过**，不占物理编号。

> ⚠️ **ref-images 总数上限 9 张**（Seedance API 硬限制）。4-5 张是常规配置，6-9 张在角色多或需要补充参考时使用。

> ⚠️ **角色参考图传入原则**：画面中出现的所有有角色库的角色，都传入 multiview 参考图，不管是主体还是配角。群演参考图（如众嫔妃）作为补充参考图传入——如果 ref-images 数量剩余较多且画面中有嫔妃出现，尽量传入；ref-images 已接近上限时可省略。

> 🛡️ **visible_chars 自检规则（强制）**：组装每个 seg 的 `visible_chars` 列表时，必须逐行扫描该 seg 的 prompt 文本，凡是提到名字或描述了动作的角色库角色（不论主配、不论是否说话、不论镜头是否聚焦），一律加入 `visible_chars`。判断标准是"画面中可辨识"而非"叙事焦点"。常见遗漏场景：
> - 角色在背景中跺脚/捂脸/探头/跑过 → 仍需传入
> - 角色被推/被拉/被搀扶（施动者在画面中）→ 施动者需传入
> - 角色在前景虚化但可辨识 → 仍需传入
> - 只有画外音、不在画面中 → 不传入 visible_chars（只传 speaking_chars）

> ⚠️ **visible_chars 与 speaking_chars 解耦**：`visible_chars`（seg-config.py）决定视觉参考——谁在画面中出现；`speaking_chars` 决定音色参考——谁在说话。两者可以不一致（如画面是萧珩+沈鹿溪，但说话的是画外音小桃）。
>
> 新项目应使用 `visible_chars` 和 `speaking_chars`。旧项目使用 `chars` 和 `audio` 仍兼容（带弃用警告）。

> ⚠️ **补充槽位可多张**：手部参考图 + 道具参考图 + 换装参考图 + 群演参考图可以同时存在，在补充槽位里按传入顺序各占一个物理编号。

> ⚠️ **起始画面 vs 尾帧的区分**：从**当前 seg** 的角度，这张图承担"起始画面"的角色（即 prompt 里写"参考图片N作为起始画面"）。它的文件名是 `lastframe_seg{N-1}.png`（上一 seg 的尾帧文件），但在当前 seg 的 ref-images 语境下，它的**逻辑角色**是起始画面。槽位从消费者角度命名。

#### 物理编号（Seedance "图片N" 指代）

Seedance prompt 里的"图片N"按**实际传入的物理顺序**编号（1-based）。官方指南不支持稀疏槽位——只传 3 张图就不能写"图片4"。

**示例 1**（双人场景衔接段，5 张图全传）：

| 逻辑槽位 | 物理编号 | 文件 |
|---------|---------|------|
| 槽位-场景 | 图片1 | scene/main.png |
| 槽位-角色A | 图片2 | charA/multiview.png |
| 槽位-角色B | 图片3 | charB/multiview.png |
| 槽位-起始画面 | 图片4 | lastframe_seg3.png |
| 槽位-补充 | 图片5 | hand-ref.png |

Prompt 里写："参考图片4作为起始画面"。

**示例 2**(单人场景，无角色B/补充，含起始画面，传 3 张)：

| 逻辑槽位 | 物理编号 | 文件 |
|---------|---------|------|
| 槽位-场景 | 图片1 | scene/main.png |
| 槽位-角色A | 图片2 | charA/multiview.png |
| 槽位-角色B | （跳过） | — |
| 槽位-起始画面 | 图片3 | lastframe_seg3.png |

Prompt 里写："参考图片3作为起始画面"（不是图片4）。

#### 组装 prompt 时的编号计算规则

AI 组装 prompt 时，必须**根据本 seg 的实际 ref-images 列表**动态计算物理编号：

```
1. 按固定顺序（场景 → 角色A → 角色B → 起始画面 → 补充）枚举要传入的图片
2. 跳过缺失的槽位
3. 第 K 个传入的图片 = "图片K"
4. prompt 里所有"图片N"指代都用物理编号
```

**反模式**：不要为了跨 seg 对齐而硬留编号——如 prompt 写"图片4 作为起始画面"但实际只传 3 张图。这会指向不存在的图，模型会忽略或胡乱关联。

#### assets.md 中的双重记录

每个 seg 的详情块必须记录逻辑槽位 → 物理编号的映射：

```markdown
#### ref-images 映射

| 逻辑槽位 | 物理编号 | 文件路径 |
|---------|---------|----------|
| 槽位-场景 | 图片1 | `./03-scenes/shisi/main.png` |
| 槽位-角色A | 图片2 | `./04-character/alice/multiview.png` |
| 槽位-角色B | — | —（本段无） |
| 槽位-起始画面 | 图片3 | `./05-videos/ep01/lastframe_seg2.png` |
| 槽位-补充 | 图片4 | `./03-scenes/_props/hand-ring.png` |
| 槽位-补充 | 图片5 | `./03-scenes/_props/ring-closeup.png` |
```

> ⚠️ **补充槽位可多行**：同一逻辑槽位允许多张图（手部/道具/换装可共存），按传入顺序各占物理编号。

**ref-images 总数 4-5 张最佳**。超过 6 张风险上升。

### 步骤 4：用户审核

审核方式按信心度等级不同：

| 等级 | 审核方式 | 提交方式 |
|------|---------|---------|
| **P0** | 逐 seg 发送 prompt + ref-images + 参数，用户确认后提交该 seg，生成完成后用户确认质量再提交下一个 | 手动逐 seg 提交（禁止 batch_gen.py） |
| **P1** | 全集所有 seg 的 prompt + ref-images + 参数一次性发给用户批次审核，确认后提交 | batch_gen.py 批次提交 |
| **P2** | 免审，AI 自主组装并提交 | batch_gen.py 批次提交 |

> ⚠️ **P0 逐 seg 流程**：提交 seg-1 → 等待生成完成 → 用户确认质量（角色一致性、场景色调、动作准确性）→ 确认通过后提交 seg-2 → ... 任一 seg 不通过则调整 prompt 重试，不继续后续 seg。这确保工作流本身的正确性在第一个 seg 就得到验证。

### 步骤 5：依赖链并行提交（P1/P2 专用）

> ⚠️ P0 等级不使用本步骤，改为步骤 4 中的逐 seg 手动提交。
> 以下流程适用于 P1/P2 等级。

#### 依赖拓扑分析

从片段划分表的"依赖"列构造有向无环图（DAG）：
- 无依赖的 seg（文生视频段）= 根节点
- `依赖 = seg-N` 的 seg = seg-N 的子节点

同一场景的连续 seg 形成链：`seg-A (文生) → seg-B → seg-C`（后一个依赖前一个的尾帧）。
不同场景的 seg 形成独立链。

**示例 DAG**（一集 8 seg）：

```
链1（食肆场景）：seg-1 (文生) → seg-2 → seg-3
链2（大殿场景）：seg-4 (文生) → seg-5 → seg-6
链3（独立特写）：seg-7 (文生)
链4（食肆结尾）：seg-8 (文生)
```

4 条链可并发提交，每条链内部串行。

#### 自动化执行（推荐）：`batch_gen.py`

用封装好的 Python 脚本自动完成调度、提交、等待、下载、重命名、更新 assets.md 一条龙：

```bash
python3 <ai-manju>/scripts/batch_gen.py <项目路径> <集号> [--max-parallel 3]
```

**配置入口**：`05-videos/ep{NN}/seg-config.py`

AI 写完剧本和 seg 切分方案后，只需生成一份 `seg-config.py`（结构化配置），脚本会自动：
1. 加载配置，构建依赖 DAG
2. 动态调度：就绪 seg 并发提交到 Seedance，上限 `--max-parallel`（默认 3）
3. 每 seg 完成后自动重命名（`seg{N}.mp4` + `lastframe_seg{N}.png`）
4. 自动更新 `assets.md` 片段划分表（pending → completed，填入 task_id + seed）
5. 失败时标记失败链，不阻塞其他链

**seg-config.py 结构**（模块级变量）：

```python
STYLE = "3D渲染，CG动画风格，游戏原画,国风3D动漫,二次元"
NO_MARK = "视频中不要出现水印。保持无字幕，避免画面生成字幕。"

CHARS = {
    "shenluxi": "04-character/shenluxi",   # 角色键 → 目录
    "liuruyan": "04-character/liuruyan",
}
SCENES = {
    "xuanxiu-dadian": "03-scenes/xuanxiu-dadian",
    "lengyuegong": "03-scenes/lengyuegong",
}

SEGS = [
    {
        "id": 1,
        "duration": 5,
        "scene": "xuanxiu-dadian",             # 可为 None
        "visible_chars": ["shenluxi"],         # 角色顺序即 ref-images 中角色顺序
        "speaking_chars": [],                  # 纯动作段留空；有对白/独白/画外音时列出对应角色键
        "start_frame": None,                   # None = 文生视频；int = 依赖该 seg 尾帧
        "prompt": """...""",                   # seg 正文（不含画风和反面约束，脚本自动拼接）
    },
    {
        "id": 2,
        "duration": 10,
        "scene": "xuanxiu-dadian",
        "visible_chars": ["liuruyan"],
        "speaking_chars": ["liuruyan"],
        "start_frame": 1,                      # 依赖 seg-1 尾帧
        "prompt": """参考图片3作为起始画面。\n\n...""",
    },
    # ...
]
```

> 💡 **字段重命名说明**：新字段名 `visible_chars` / `speaking_chars` 比旧字段名 `chars` / `audio` 更清晰
> （治理 MEMORY.md 教训 #5 / #10 — 约束对象混淆）。
> 旧字段名仍兼容，运行时会打印弃用警告，迁移时全文替换即可。

**字段规则**：
- `scene` 指定 `SCENES` 键，None 表示本段无场景参考图
- `visible_chars` 列出本段**画面中可辨识的所有角色库角色**（不论主配、不论是否说话）。按顺序决定 ref-images 中角色的物理编号。判断标准：逐行扫描 prompt，凡提到名字或描述了动作的角色一律加入。只有画外音不在画面中的角色不加入 visible_chars（只加 speaking_chars）
- `speaking_chars` 只列出本段会发声的角色（纯动作段置空，对应 `--audio` 不传）
- `start_frame` 为 int 时自动把 `lastframe_seg{N}.png` 加到 ref-images 末尾；prompt 中需要手动写"参考图片{N}作为起始画面"（N 为实际传入顺序编号）
- `prompt` 不需要包含画风关键词和反面约束——脚本自动在头部拼接 `STYLE\n\nNO_MARK\n\n{prompt}`

**ref-images 物理编号计算规则**（提示脚本使用者）：
固定顺序 `场景 → 角色A → 角色B ... → 起始画面 → 补充`。跳过缺失项。例如 `scene=xxx, chars=[A, B], start_frame=1` 会传入 4 张图，prompt 里的起始画面是"图片4"。

> ⚠️ prompt 里的"图片N"仍需人工写，脚本只管按顺序传入 ref-images。

#### 手动执行（备用）

如需跳过 batch_gen.py 手动操作，参考本节末的"执行命令"小节逐段提交。

#### 执行命令

> ⚠️ **`--audio` 传入条件**：只在本段**有对白 / 内心独白 / 画外音**（有人声需要 TTS 生成）时传入对应角色的 `voice-ref.mp3`。纯动作段（无人声）**省略 `--audio`**。`--generate-audio true` 不变（仍用于生成环境音效），prompt 中的 `<环境音>` 标注走独立生成通道，与 voice-ref 无关。

**文生视频段**（场景跳转/独立段，有对白示例）：

```bash
python3 <video-gen>/scripts/seedance.py create \
  --prompt "{seg 视频 prompt}" \
  --ref-images \
    ./03-scenes/{场景名}/main.png \
    ./04-character/{角色A}/multiview.png \
    ./04-character/{角色B}/multiview.png \
  --audio \
    ./04-character/{角色A}/voice-ref.mp3 \
    ./04-character/{角色B}/voice-ref.mp3 \
  --ratio 9:16 \
  --duration {seg 时长} \
  --generate-audio true \
  --return-last-frame true
```

**同场景衔接段**（尾帧作为起始画面，双人有对白示例）：

```bash
python3 <video-gen>/scripts/seedance.py create \
  --prompt "{seg 视频 prompt，开头含『参考图片4作为起始画面』——因为实际传入顺序场景/角色A/角色B/尾帧 = 图片1/2/3/4}" \
  --ref-images \
    ./03-scenes/{场景名}/main.png \
    ./04-character/{角色A}/multiview.png \
    ./04-character/{角色B}/multiview.png \
    ./05-videos/ep{NN}/lastframe_seg{N-1}.png \
  --audio \
    ./04-character/{角色A}/voice-ref.mp3 \
    ./04-character/{角色B}/voice-ref.mp3 \
  --ratio 9:16 \
  --duration {seg 时长} \
  --generate-audio true \
  --return-last-frame true
```

**纯动作段**（无对白/独白/画外音，省略 `--audio`）：

```bash
python3 <video-gen>/scripts/seedance.py create \
  --prompt "{seg 视频 prompt，末尾含『无对话。』}" \
  --ref-images \
    ./03-scenes/{场景名}/main.png \
    ./04-character/{角色A}/multiview.png \
    ./05-videos/ep{NN}/lastframe_seg{N-1}.png \
  --ratio 9:16 \
  --duration {seg 时长} \
  --generate-audio true \
  --return-last-frame true
```

> ⚠️ **图片编号随实际传入顺序动态变化**：同一 "尾帧" 槽位，双人场景时是图片4、单人场景时是图片3。必须根据本 seg 实际 ref-images 列表计算物理编号，并在 assets.md 的"ref-images 映射"表中记录。

> ⚠️ **为什么尾帧作为 ref-images 而不是 `--first-frame`**：Seedance 的 `--first-frame` 参数与多模态参考（`--ref-images`）互斥。我们用多模态参考传入场景图和角色图，所以尾帧必须作为 `--ref-images` 的一张传入，并在 prompt 中用"参考图片N作为起始画面"声明。

#### 批量管理

并发任务用 task_id 追踪，统一轮询：

```bash
# 提交所有根节点 seg（无依赖）
for seg in 根节点列表; do
  task_id=$(python3 <video-gen>/scripts/seedance.py create ... | grep -oP 'ID:\s*\K\S+')
  记录到 assets.md
done

# 等待完成并下载
python3 <video-gen>/scripts/seedance.py wait <task_id> --download ./05-videos/ep{NN}/

# 重命名
# 视频：seedance_{task_id}_{timestamp}.mp4 → seg{N}.mp4
# 尾帧：lastframe_{task_id}.png → lastframe_seg{N}.png

# 子节点 seg 此时可启动（前置尾帧已到位）
```

> 💡 优化：`seedance.py` 支持单次 `wait` 多个 task_id 可进一步并行化。如不支持，用 shell 后台进程轮询。

### 步骤 6：seed 策略

| 场景 | seed 值 | 理由 |
|------|---------|------|
| 首次生成 | 不传（默认 -1，随机） | 让模型自由发挥 |
| 重新生成（修 prompt） | 传上一次的 seed | 锚定"好方向" |
| 重新生成（碰运气） | 不传（默认 -1） | 换个随机方向 |

### 步骤 7：失败重试策略

| 失败类型 | 判断方式 | 处理 |
|----------|---------|------|
| API 超时/网络错误 | `seedance.py wait` 返回错误 | 原参数重试，最多 2 次 |
| 安全过滤拒绝 | 返回 content_filter 错误 | 调整 prompt 中可能触发过滤的描述，重试 |
| 质量不合格 | 目测：角色变形/面部崩塌/光影严重偏离 | 用 `regen_seg.sh` 重新生成，最多 2 次 |
| 连续 3 次失败 | 同 seg 连续 3 次未通过 | 停止，报告问题，和用户讨论方向调整 |
| API 整体不可用 | 所有任务都失败 | **暂停项目，等服务恢复**。禁止切换到备用模型（会导致风格漂移） |
