## 流程

### 步骤 1：生成段 N 关键帧（每段 2 帧）

> ⚠️ **最少化原则**：故事板帧的唯一用途是作为 Seedance ref-images 参考——提供场景空间、光源方向和角色在该场景光影中的外观。每段 2 帧足够，多的帧不会提升视频质量反而增加约束冲突风险。

> ⚠️ **Prompt 审核门禁**：每段组装完成后，对照 `references/prompt-guide.md` §八「图片 Prompt 审查清单」自查。然后将完整 prompt 列表 + 参数发送给用户审核，确认后才可提交。

| 帧 | 类型 | 用途 |
|----|------|------|
| 帧 1 | 定场镜头（远景/全景） | 场景空间 + 光源方向 + 色调氛围 |
| 帧 2 | 该段情绪最高点的特写/中近景 | 角色在该场景光影中的脸，为 Seedance 提供面部光照参考 |

每帧单独生成（图生图，挂角色设定图）：

```bash
python3 <video-gen>/scripts/seedream.py create \
  -p "{画风}风格。{画面描述，含构图/景别/光影/色调}。角色外观参考角色设定图。" \
  -i ./04-human-design/{角色}/front.png \
  --size 9:16 \
  -d ./03-storyboard/ep01/seg1/
```

### 步骤 2：Prompt 模板（每帧）

```
{画风}风格，{景别}。
{角色名}{动作/姿态}，{表情}。
场景：{环境描述}。
光线：{光源方向}，{光比描述}，{光质描述}，{色温描述}。
{色调氛围}。
构图：{视角/机位/景深}。
```

示例：
```
新海诚风格，中近景镜头。
林小夜站在窗边，侧头望向窗外，表情冷静而若有所思。
场景：昏暗的公寓房间，窗外是夜空和城市灯光。
光线：从左侧窗外的月光为主光源，硬质冷蓝光，角色脸部被微弱的暖黄色台灯从右侧补光，冷暖对比约 3:1。
清冷忧郁氛围，深蓝与暖橙色调。
构图：低角度仰拍，浅景深，前景虚化窗框框架构图。
```


> 每帧都用同一套模板刷，20 集会形成视觉惯性——观众闭眼都知道下一个镜头长什么样。
> 模板保下限（不漏关键信息），以下变体拉上限（帧帧有差异）。

#### 景别节奏变体

不要每段都按"远→中→近→特"线性走。根据该段的情绪峰值选一种节奏：

| 节奏类型 | 帧序列 | 情绪效果 | 何时用 |
|---------|--------|---------|--------|
| **线性爬坡**（默认） | 远→中→近→特 | 稳步升温 | 日常叙事、角色登场 |
| **压缩爆发** | 特写开场→中景→特写→远景释放 | 紧张→爆发→余韵 | 冲突、对峙、惊吓 |
| **呼吸交替** | 远景→特写→远景→特写 | 内心与环境的反差 | 独白、思考、犹豫 |
| **跳跃错位** | 特写→远景→极特→中景 | 不安、错乱 | 悬疑、梦境、醉酒 |
| **空间穿梭** | 俯拍→平视→仰拍→俯拍 | 视角在三维空间跳转 | 追逐、空间转换 |

> 连续两段必须用不同的节奏类型。

#### 光线签名变体

同一场景类型不要每集都写"左侧窗光"。从以下池子轮换，注意匹配灯光哲学的基调约束：

| 光源配置 | Prompt 短语 | 适合情绪 |
|---------|------------|---------|
| 高侧窗 | "从高处窄窗打入的硬光束，在地面投射清晰的窗格阴影" | 孤独、等待、囚禁 |
| 低角度逆光 | "光源来自角色身后偏下，角色身体被光包围成剪影，面部全暗" | 神秘、威胁、登场 |
| 点光源阵列 | "多个分散点光源（蜡烛/屏幕/灯串），角色脸部被不均匀照亮" | 亲密、碎片化、焦虑 |
| 反射光为主 | "无直射光源，光线从墙壁/水面/天花板反射而来，柔和均匀" | 平静、日常、清晨 |
| 移动光源 | "光源在画面外移动（车灯/手电），光斑在角色身上扫过" | 搜索、逃离、时间流逝 |
| 底光邪异 | "光源从下方打在角色脸上，眼眶和鼻下形成倒影" | 恐怖、黑化、审讯 |
| 门缝/缝隙光 | "光线从门缝/窗帘缝挤入，形成极细的光刃切开黑暗" | 偷窥、秘密、希望 |

#### 构图破格

70% 的帧用标准构图（角色居中或三分位），30% 的帧故意破格：

| 破格手法 | 描述 | 效果 | 每段最多 |
|---------|------|------|---------|
| 负空间构图 | "角色被挤到画面边缘，大面积空白/纯色填充剩余" | 孤独、渺小、被遗忘 | 1 帧 |
| 失焦主体 | "焦点在角色手部/物品上，角色脸部完全虚化" | 关注点转移、悬念 | 1 帧 |
| 极端比例 | "角色只占画面底部 1/5，上方是压倒性的建筑/天空" | 宿命、压迫、无力 | 1 帧 |
| 镜面/反射 | "画面是镜子/水面/玻璃中的倒影，非直接视点" | 自我审视、虚幻 | 1 帧 |
| 部分遮挡 | "前景物体遮住角色半张脸，只露出一只眼睛" | 隐藏、窥视、不完整 | 1 帧 |

> 每段 4-6 帧中，标准构图 3-4 帧 + 破格 1-2 帧。破格帧必须服务于该帧的情绪目的，不为破而破。

### 步骤 5：逐帧灯光核查

对照 `03-storyboard/lighting-philosophy.md`：

- [ ] 每帧主光方向明确，与灯光哲学场景规则一致
- [ ] 光比在灯光哲学定义范围内（高调/低调约束）
- [ ] 色温符合色彩策略
- [ ] 灯光母题在关键情绪帧中可见
- [ ] 实用光源驱动正常（现实主义模式下无无源光）
- [ ] 分离光在暗调帧中有效（角色不融入背景）
- [ ] 特写帧有眼神光（catchlight）
- [ ] 灯光节拍与剧本对齐
- [ ] 相邻帧景别或视角有明显差异

如有帧违反灯光哲学，重新生成该帧——不改哲学迁就帧。

### 步骤 6：故事板人工审核（🆕 强制）

> ⚠️ 故事板帧是视频生成的 ref-images 基础。帧中的错误（戒指位置、手指数量、服装不一致）会直接传导到视频中，修复成本远高于在故事板阶段拦截。

**审核流程**：

1. AI 先完成自检（景别节奏、色调、角色面部、构图、光影、AI 生成异常），自主修复能发现的问题
2. 将所有帧（每段 2 帧，共 8 帧）连同 assets.md 中的 prompt 和审核要点提交用户
3. 用户逐帧确认：构图、角色外观、光影、景别、关键细节（戒指位置、手指数量、服装一致性等）
4. 用户标记需要修改的帧，说明问题
5. AI 重新生成问题帧（备份旧版后替换），再次提交用户审核
6. **全部帧通过后，用户明确确认"故事板通过"，才能进入视频生成阶段**

**AI 自检清单**（步骤 1，在提交用户前完成）：

- [ ] 景别序列有节奏感（远→中→近→特 或 反转）
- [ ] 色调与 Moodboard 一致
- [ ] 角色面部清晰可辨，与设定图一致（对比 front.jpg）
- [ ] 构图中角色位置合理
- [ ] 帧与帧之间视觉连贯
- [ ] 段间首尾帧视觉可衔接
- [ ] 无 AI 生成异常（多指/肢体变形/面部崩塌）
- [ ] 光影与灯光哲学一致

> 自检发现问题的帧自主重新生成（备份旧版后替换）。自检通过后提交用户审核。

**故事板未经用户确认，不进视频阶段。**
