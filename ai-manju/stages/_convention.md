# 阶段文件规范

每个 `NN-description.md` 是 AI 漫剧工作流的一个自包含执行单元。

## 格式约定

```markdown
# 阶段 NN：{名称}

> 工具：{Seedream | Seedance | DeepSeek | FFmpeg | 人工确认}

## 前置条件
{执行前必须存在的文件清单}

## 输入
{需要读取的文件}

## 流程
### 步骤 1：{做什么}
{具体操作 + 命令模板}

### 步骤 2：...
## 产出
| 文件 | 说明 |
|------|------|

## 验收
- [ ] {检查项}

## 资产登记
- [ ] assets.md 已写入完整的生成 Prompt + 命令参数 + 任务 ID + 下载 URL

## 完成后
→ `NN+1-description.md`
将 STATE.md 中 `当前阶段` 更新为 `NN-next`

⚠️ **STATE.md 更新是硬性门禁**：不更新 STATE.md 就不能进入下一阶段。这是跨会话恢复上下文的唯一入口。
```

## 设计原则

### ⚠️ Prompt 审核门禁（全局）

**调用任何外部生成 API 之前，必须将完整 prompt + 全部参数发送给用户审核，确认后才可提交。**

适用所有生成工具：Seedance、Seedream、TTS/豆包语音合成。

执行顺序：组装 prompt → 发用户审核 → 用户确认 → 提交 API。

此规则在 AGENTS.md 中亦有全局定义，此处为 ai-manju 专项强制约束。

### 其他原则

1. **自包含**：阶段文件不引用其他阶段文件。所有 prompt 模板、命令、约束写在本文件内。
2. **命令可复制**：`bash` 代码块中的路径用相对路径（以项目根为基准），可直接执行。
3. **显式验收**：每个阶段以 checklist 结束，逐项核对后手动勾选。
4. **🆕 跨会话可追溯**：每个生成阶段必须在 `assets.md` 中记录**完整 Prompt 原文 + 命令参数**。URL 会过期、图片可能丢失，但 Prompt 是唯一的可复现锚点。新会话只需读 `assets.md` 即可恢复上下文。
5. **🆕 STATE.md 门禁**：每个阶段完成后，必须立即更新 STATE.md（勾选已完成阶段、更新当前阶段、更新最后更新时间）。不更新 STATE.md 不能进入下一阶段。STATE.md 是新会话恢复项目进度的唯一入口。
5. **🆕 目录命名**：所有目录和文件使用**英文或拼音**，禁止中文。`04-human-design/guyuan/` ✅ | `04-human-design/顾鸢/` ❌
6. **路径约定**：

| 占位符 | 含义 | 解析规则 |
|--------|------|----------|
| `<video-gen>` | video-generation skill 目录 | 优先检查工作区根目录下的 `video-generation/`；如不存在，退化到 `/root/.openclaw/workspace/skills/video-generation` |
| `<tts-skill>` | tts skill 目录 | 优先检查工作区根目录下的 `tts/`；如不存在，退化到 `/root/.openclaw/workspace/skills/tts` |
| `<ai-manju>` | 当前 skill 目录 | 优先检查工作区根目录下的 `ai-manju/`；如不存在，退化到 `/root/.openclaw/workspace/skills/ai-manju` |

> ⚠️ **环境适配**：在 Kiro / Claude 等 IDE 中，skill 文件直接在工作区根目录下，占位符解析为相对路径（如 `video-generation/scripts/seedance.py`）。在 OpenClaw 中解析为绝对路径。执行命令前确认占位符已替换为当前环境的实际路径。

所有项目内路径以项目根为基准（如 `./04-human-design/`）。命令中占位符需替换为实际路径后执行。

## assets.md 规范（强制 🆕）

每个有内容产出的阶段，必须在对应目录写入 `assets.md`：

```markdown
# {阶段/角色} 资产清单

## 生成 Prompt

| 文件 | 完整 Prompt | 命令参数 | 任务 ID | 生成时间 |
|------|------------|---------|---------|----------|
| file_01.png | "完整的 seedream prompt..." | --size 9:16 --seq auto --max-images 4 | task_xxx | 2026-04-27T21:28 |

## 产出文件

| 文件 | 来源 | 下载 URL |
|------|------|----------|
| file_01.png | Seedream 文生组图 | https://... |
```

**规则**：
- **Prompt 列必须是发送给生成工具的完整原文**，不是摘要、不是关键词、不是"场景描述"。
- 命令参数（--size, --ratio, --duration, --seq 等）一并记录。
- URL 可能过期，但 Prompt 不会。
- 后续阶段读取此文件获取风格/视觉基准。
- ⚠️ **执行顺序**：生成 → 重命名 → 立即登记 assets.md → 下一张。不允许攒两轮再补。

## 即时登记纪律（全局强制 🆕）

> 适用所有涉及资产生成的阶段（04 角色设计、06 故事板、07-10 视频生成、11 合成）。

**规则**：每生成一个资产文件（图片/视频/音频），必须立即完成以下三步后才能生成下一个：

1. **重命名**：将生成的文件重命名为规范文件名
2. **登记 assets.md**：将完整 Prompt + 命令参数 + 文件名写入 assets.md
3. **确认写入成功**：检查 assets.md 已保存

**禁止**：
- ❌ 批量生成多个文件后回头补录 assets.md
- ❌ 先生成全部再统一重命名
- ❌ 在 assets.md 中写"待补录"占位符然后继续生成

**原因**：assets.md 是跨会话恢复的唯一锚点。如果会话中断（超时/崩溃/手动停止），未登记的资产 = 丢失 Prompt 记录 = 无法复现。即时登记确保任何时刻中断，已完成的部分都是完整可追溯的。

## narrative-state.md 规范（强制 🆕）

叙事状态追踪文件，存放于 `01-scripts/narrative-state.md`。每集剧本完成后必须更新。
这是 AI 编写后续集剧本时的核心上下文输入——替代"读取前面所有集完整剧本"的低效方式。

```markdown
# 叙事状态追踪 — {剧名}

**最后更新集**: EP{NN}

## 已揭露信息（观众已知）

| 集 | 揭露内容 |
|----|---------|
| EP01 | {观众在这集获得的关键信息} |
| EP02 | ... |

## 未揭露信息（按计划揭露集数）

| 计划集 | 内容 | 状态 |
|--------|------|------|
| EP12 | {待揭露的信息} | pending |
| EP18 | ... | pending |

## 角色当前状态

### {角色名}
- **对{角色B}的态度变化轨迹**：{EP01-04 警惕} → {EP05-09 信任} → {EP10起 怀疑}
- **Want 进度**：{百分比}（{当前阶段描述}）
- **Flaw 松动程度**：{未松动 / 开始质疑 / 已突破}
- **最后关键台词**：EP{NN}-seg{N} "{台词}"
- **最后关键动作**：EP{NN}-seg{N} {动作描述}

### {角色名B}
...

## 未兑现的伏笔

| 埋设位置 | 伏笔内容 | 计划兑现 | 状态 |
|----------|---------|---------|------|
| EP02-seg3 | {伏笔描述} | EP14 | pending |
| EP06-seg1 | {伏笔描述} | EP11 | pending |

## 上一集摘要（EP{NN}）

- seg1: {一句话}
- seg2: {一句话}
- seg3: {一句话}
- seg4: {一句话} — 结尾悬念：{描述}
```

**更新规则**：
- 每集剧本确认后，立即更新此文件，不允许跳集补写
- "已揭露信息"只增不删——观众知道的不能收回
- "未揭露信息"兑现后移入"已揭露"并标注实际集数
- "未兑现伏笔"兑现后标记 `done`，超过计划集数 3 集未兑现标记 `overdue`
- 角色状态的态度变化轨迹必须用 `→` 连接，标注集数范围
- 上一集摘要每次覆盖写（只保留最近一集）

## episode-skeleton.md 规范（强制 🆕）

80 段骨架文件，存放于 `01-scripts/episode-skeleton.md`。整季大纲确认后编写，是所有集剧本展开的基础。

```markdown
# {剧名} 分段骨架

## EP01: {集标题}
- seg1: {一句话描述这 15 秒发生什么}
- seg2: {一句话}
- seg3: {一句话}
- seg4: {一句话} — 悬念：{钩子}

## EP02: {集标题}
- seg1: ...
...
```

**编写规则**：
- 建议分批生成（每次 5 集），每批带着前面已确认的骨架作为上下文
- 每段一句话，不超过 30 字，只写"发生什么"，不写镜头语言
- 每集 seg4 必须标注悬念钩子
- 骨架确认后不轻易改动——改骨架意味着后续所有集的连锁调整
- 如需调整，必须同步检查 narrative-state 中的伏笔和信息揭露计划
