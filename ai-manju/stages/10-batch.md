# 阶段 10：批量生产（后续集）

> 工具：循环阶段 06-09
> ⚠️ 逐集独立——新集重新生成剧本和视频，不沿用上一集素材（角色多视图、场景库、character-bible、Moodboard 除外）

## 前置条件

- 第 1 集成片已通过检查
- 角色多视图、场景库、Moodboard、灯光哲学、配色方案作为常量沿用

## 流程

### 对每一集循环执行

1. **阶段 05** — 剧本（新集）+ 更新 narrative-state.md
2. **阶段 06** — 场景图（按需处理该集新增/变体场景）
3. **阶段 07** — 视频生成（prompt 组装 + 依赖链并行提交）
4. **阶段 08** — 合成（concat 拼接 + 字幕烧录）
5. **阶段 09** — 成片检查

### 每集独立原则

- 每个新集从 `05-videos/ep{NN}/` 开始独立产出
- 剧本中出现场景库没有的新场景时，按阶段 06 的"按需补充"流程处理（复用/修改/新建）
- 依赖链内并发提交，链间并发 ≤ 3（见阶段 07 步骤 5）

## 信心度分级（🆕）

逐集推进时，根据项目进展调整用户确认门禁等级（详见 `_convention.md`「用户确认门禁分级」）：

| 集数 | 默认等级 | 确认点 |
|------|---------|--------|
| EP01 | P0 全确认 | 逐 seg 审核 prompt → 逐 seg 提交 → 逐 seg 确认质量 |
| EP02-EP05 | P1 批次确认 | 全集 prompt 批次审核 → batch_gen.py 批次提交 → 成片后确认 |
| EP06+ | P2 预授权批处理 | prompt 免审 → batch_gen.py 批次提交 → 成片后确认 |

**升级**：连续 3 集 P1 通过无重大返工 → 升级 P2
**降级**：触发条件见下节「漂移降级触发机制」

STATE.md 中记录当前 `信心度等级`。

## 漂移降级触发机制（🆕）

漂移的判定和降级在**每 5 集跨集审计**时执行，不在单集 review 时触发。

### 漂移判定（跨集审计 = 触发时机）

每 5 集暂停批量生产做审计。AI 对比第 1 集和当前集的成片帧/对白片段，按以下标准判定：

| 维度 | 漂移程度 | 判定标准 |
|------|---------|---------|
| 角色外观 | 轻微 | 眼型/发色/脸型有 1 项明显变化 |
| 角色外观 | 严重 | ≥2 项明显变化 |
| 场景色调 | 轻微 | 相同场景色温偏移 ≤30% |
| 场景色调 | 严重 | 相同场景色温偏移 >30%，或主色调变化 |
| 角色音色 | 轻微 | 闭眼听有一点不同，但可认出同角色 |
| 角色音色 | 严重 | 3 句对白里 2 句以上听不出是同一角色 |

### 降级动作

任何一项**严重**漂移 = 触发降级：

1. 立即暂停当前集往后的批量生产
2. 当前信心度 **降一级**（P2→P1，P1→P0）
3. 回滚：用 `regen_seg.sh` 重新生成当前集的漂移 seg
4. 跨集审计重新通过后才恢复批量
5. STATE.md 中更新 `信心度等级`，并记录降级原因

### 未来升级限制

降级后，需要**重新累计 3 集无返工**才能再次升级。不能刚降级就立刻恢复。

## 剧本追溯修改决策树（🆕）

写到第 N 集发现早期集（如第 2 集）的伏笔/细节有问题：

```
修改锚点在哪里？
├── 改当前及之后集（不动历史集）
│   └── 在 narrative-state 中记录"口径修正"，后续集按新口径圆回来
│   └── 成本：小（仅改后续剧本）
│
├── 改历史集剧本，但不改已生成的视频
│   └── 只改 ep{N}-script.md 文档，视频保持原样
│   └── 风险：剧本和成片不一致（仅当用户不在意文档时适用）
│
└── 改历史集 + 重生成受影响的视频
    └── 1. 改 ep{N}-script.md
    │   2. 重跑 narrative-state 全局扫描（检查连锁影响）
    │   3. 列出所有受影响的后续集（涉及该伏笔的）
    │   4. 用 regen_seg.sh 重新生成受影响的 seg
    │   5. 重新合成受影响的集
    └── 成本：大，慎用。仅在"不改就无法继续"时选
```

**决策原则**：
- 默认选"方案 1"——剧本的弹性 > 完美一致性
- 方案 3 保留给关键信息归属错误（如凶手身份搞错）
- 任何方案前，先评估影响范围（哪些集受影响），再决定

## 每 5 集：跨集一致性审计

暂停批量生产，执行以下审计。

### 场景蔓延监控（🆕）

统计初始场景库场景数 vs 当前累计场景数（含按需补充）：

```bash
INITIAL=$(ls -d projects/{项目名}/03-scenes/*/ 2>/dev/null | wc -l)  # 初始时用户确认的场景数
# 补充生成的场景在 assets.md 里应标注生成时间
```

| 蔓延比率 | 判定 | 动作 |
|---------|------|------|
| ≤30% 新增 | 正常 | 继续批量 |
| 30-50% 新增 | 警戒 | 提示用户"剧本场景散乱"，建议收敛后续集 |
| >50% 新增 | 失控 | 暂停批量，检查剧本是否偏离 series-bible 的场景预期 |

场景蔓延失控时成本会大幅超预算。提醒用户**"为现有场景写戏"优于"为戏写新场景"**。

### 视觉审计

从各集成片中提取角色面部特写帧：

```bash
# 从第 1 集和第 N 集成片提取同角色面部特写
ffmpeg -i ./06-output/ep01-final.mp4 -ss {时间} -vframes 1 ./07-consistency-check/faces/ep01_{角色名}.jpg
ffmpeg -i ./06-output/ep0N-final.mp4 -ss {时间} -vframes 1 ./07-consistency-check/faces/ep0N_{角色名}.jpg
```

- [ ] 眼型、发色、脸型是否一致
- [ ] 同类型场景色调是否漂移
- [ ] 配色方案语义标记是否持续使用

**修复漂移**：
- 检查 `04-character/{角色}/multiview.png` 是否与第 1 集一致（如果被改过，恢复）
- 在 prompt 中加 "严格按照角色多视图外观生成，不引入风格漂移"
- 严重漂移时，重新生成角色多视图，并重新生成后续集的相关 seg

### 声音审计

**Step 0 — 前置：Golden 签字已在阶段 09 完成**

EP01 已在阶段 09 步骤 7 完成 Golden 签字，记录在 `07-consistency-check/audio/golden/GOLDEN_SIGNOFF.md`，所有主要角色的 golden 对白片段已归档到 `07-consistency-check/audio/golden/`。

本阶段的声音审计基于已冻结的 golden 做**对比审计**。

> ⚠️ 如果 `GOLDEN_SIGNOFF.md` 不存在或存在未签字（❌）角色，说明 EP01 未正确完成阶段 09 的硬门禁——回退到阶段 09 步骤 7，不进批量生产。

**Step 1 — 对比审计（每 5 集）**

从第 N 集成片截取同角色对白：

```bash
ffmpeg -i ./06-output/ep0N-final.mp4 \
  -ss {对白起始秒} -t {对白时长} \
  -vn -acodec copy \
  ./07-consistency-check/audio/{角色名}_ep0N.mp3
```

人工对比（对照 `07-consistency-check/audio/golden/{角色名}_ep01_*.mp3`）：

| 检查项 | 判断标准 |
|---|---|
| 音色一致性 | 闭眼听，同角色像同一个人吗？ |
| 角色区分度 | A 和 B 还明显不同吗？ |
| 情感表现力 | 同样标记 angry/悲伤，还一样有张力吗？ |
| 音量/音质 | 有没有变轻、变闷、变尖、变糊？ |

> 取至少 3 句不同情绪对比。一句听不出，两句可能巧合，三句基本能判断。

**修复漂移**：

| 漂移程度 | 方案 |
|---|---|
| 轻微 | 重新将 ep01 的 voice-ref.mp3 挂入后续 `--audio` |
| voice-ref 本身质量不够 | TTS 精调参数重新生成 voice-ref |
| 严重 | 切到 B 方案：TTS 逐句合成全部对白，FFmpeg 替换音频轨 |

审计通过后继续后续 5 集。
