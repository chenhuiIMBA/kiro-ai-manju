# 阶段 04：场景库

> 工具：Seedream 5.0（场景图 + 多视图）
> ⚠️ 必须在灯光配色之后、角色设计之前生成

## 前置条件

- `01-scripts/lighting-philosophy.md` 存在（灯光哲学）
- `01-scripts/color-palette.md` 存在（配色方案）
- `01-scripts/series-bible.md` 已确认
- `01-scripts/episode-skeleton.md` 已确认

## 流程

### 步骤 1：提取场景列表

从 `series-bible.md` 和 `episode-skeleton.md` 中提取所有主要场景，列出场景清单：

| 场景名 | 英文目录名 | 描述 | 出场频率 |
|--------|----------|------|---------|
| 食肆 | shisi | 主角家的江南小食肆 | 高频 |
| 选秀大殿 | daadian-nei | 金碧辉煌的宫廷大殿 | 高频 |
| 大殿外广场 | daadian-wai | 皇宫外开阔广场 | 中频 |
| 宫道 | gongdao | 高墙夹峙的宫廷长道 | 高频 |
| 宫门 | gongmen | 巨大朱红宫门 | 中频 |
| 太师府 | taishifu | 权臣府邸内厅 | 中频 |
| 冷月宫 | lengyuegong | 偏远破败的冷宫 | 高频 |
| ... | ... | ... | ... |

> ⚠️ 场景目录名使用英文或拼音，禁止中文。

### 步骤 2：生成主视角图（main.png）

每个场景用独立 prompt 生成一张主视角图（16:9 横版，展示空间全貌）。

> ⚠️ **Prompt 审核门禁（门禁 1）**：组装完成后，对照 `references/prompt-guide.md` §八「图片 Prompt 审查清单」自查。**批次审核**——所有场景的主视角 prompt 一次性发给用户确认，不逐个发。

**Prompt 结构**（自然语言连贯描述）：

```
{画风关键词}，广角镜头，{场景类型描述}场景图，空无一人。
{一句话场景氛围总述}。
前景{具体物品/材质}，中景{空间主体/陈设}，后景{纵深/远景元素}，空间层次分明。
整体色调以{语义标记引用 color-palette.md 中的色板}为主，{氛围/情绪/叙事功能描述}，
材质强调{3-5种具体材质}，电影级光影，无人物，无文字。
```

> ⚠️ 色调描述**必须**引用 `color-palette.md` 中定义的语义标记（如 `night_ambient`、`warm_key`），而不是临时编造颜色名。这确保跨场景一致性。

**生成命令**：

```bash
python3 <video-gen>/scripts/seedream.py create \
  -p "<场景prompt>" \
  --size 16:9 \
  -d ./03-scenes/{场景名}/
# 重命名为 main.png
mv ./03-scenes/{场景名}/seedream_*.jpg ./03-scenes/{场景名}/main.png
```

### 步骤 3：用户批次审核主视角

将所有场景的 main.png 一次性提交用户审核（不逐场景审）。审核维度：
- 空间布局是否符合剧情需要
- 色调是否符合灯光哲学和配色方案
- 材质细节是否可辨
- 情绪氛围是否匹配场景叙事功能

用户确认所有主视角图后，进入步骤 4。

### 步骤 4：生成多视图组合图（multiview.png）

用主视角做图生图，生成正/背/左/右四个方向拼在一张图里的组合图（16:9 横版）。

**Prompt 模板**：

```
{画风关键词}，广角镜头，{场景类型}多视图场景设定图，空无一人。
空间布局与参考图完全一致，材质与色调与参考图完全一致。
一张组合图，包含四个视角：左上为正面视角，右上为背面视角，左下为左侧面视角，右下为右侧面视角。
每个视角展示同一空间的不同方向，保持一致的色调和材质。
电影级光影，无人物，无文字。
```

**生成命令**：

```bash
python3 <video-gen>/scripts/seedream.py create \
  -p "<多视图prompt>" \
  -i ./03-scenes/{场景名}/main.png \
  --size 16:9 \
  -d ./03-scenes/{场景名}/
# 重命名为 multiview.png
```

### 步骤 5：文件整理与资产登记

```
03-scenes/
├── {场景名}/
│   ├── main.png          # 主视角（金标准，用户审核用）
│   ├── multiview.png     # 多视图组合（正/背/左/右）→ 传入 Seedance ref-images
│   └── assets.md         # prompt + task_id + 生成参数
└── ...
```

⚠️ **即时登记门禁（门禁 2）**：每张图生成后立即登记到 assets.md。

**assets.md 模板**：

```markdown
# {场景名} 场景库资产

## 生成 Prompt

| 文件 | 完整 Prompt | 命令参数 | 任务 ID | 生成时间 |
|------|------------|---------|---------|----------|
| main.png | "<主视角完整 prompt>" | --size 16:9 | task_xxx | 2026-05-05T... |
| multiview.png | "<多视图完整 prompt>" | -i main.png --size 16:9 | task_yyy | 2026-05-05T... |

## 场景描述

**场景类型**：{室内/室外/广场/长道/...}
**主色调**：{引用 color-palette.md 语义标记}
**情绪氛围**：{一句话}
**使用说明**：作为 Seedance ref-images 传入（使用 multiview.png）
```

## 按需补充规则

剧本创作不受场景库限制。如果剧本中出现场景库没有的场景：

1. 在视频 prompt 组装阶段（阶段 07）发现缺失场景
2. 按步骤 2-5 流程补充生成
3. 用户确认后加入场景库
4. 继续视频生成

## 跨项目复用

本阶段产出的场景库可复用到同世界观的续作项目。使用 `init_project.py --from-template <源项目>` 复制场景库到新项目（见 SKILL.md「跨项目资产复用」章节）。
