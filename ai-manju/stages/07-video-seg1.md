# 阶段 07：视频生成（四段 prompt 组装 + 衔接规划 + 精简审核）

> 工具：Seedance 2.0（多模态参考生视频）
> ⚠️ 四段 prompt 一次性组装并审核，再串行提交。不逐段审。

## 前置条件

- 故事板 4 段关键帧已确认（每段 2 帧）
- 角色设定图 + voice-ref 就绪
- 补充参考图（服装/道具/灯光）已生成
- 剧本段拆分已确定

## 流程

### 步骤 1：一次性组装四段完整 prompt（不约束长度）

按 `prompt-guide.md` §九 迁移流程，为每段写出细节完整的 prompt。同时填写段间衔接规划表。

### 步骤 2：段间衔接规划

| 衔接 | 场景变化 | 色温变化 | 推荐方式 | 段尾指令 | 段头指令 |
|------|---------|---------|---------|---------|---------|
| 段1→2 | 场景A→B | 暖→冷/冷→冷 | crossfade / 黑场fade / 硬切 | `画面缓慢暗下` 或无 | `画面从黑场渐亮` 或无 |

**衔接方式选择规则**：

| 条件 | 方式 | FFmpeg 命令 | 尾帧传入下一段？ |
|------|------|------------|-----------------|
| 色温/场景大跳（室内→室外、白天→黑夜） | **黑场fade** | `fade=t=out:st=14.5:d=0.5` + `fade=t=in:st=0:d=0.5` | ❌ 不需要——场景已切换，尾帧与下一段无关 |
| 色温相近、不同空间 | **crossfade** | `xfade=transition=fade:duration=0.5:offset=14.5` | ❌ 不需要——不同空间，尾帧无参考价值 |
| 同一场景内连续叙事 | **硬切** | `concat` | ✅ 传尾帧——同一场景，尾帧可作为下一段开场参考 |

> ⚠️ **尾帧用途限制**：`--return-last-frame` 的尾帧仅在同一场景连续叙事时传入下一段 `--ref-images`。跨场景/跨时间时尾帧不能作为下一段的首帧参考——两段之间本来就不要求画面连续。尾帧仍需下载到本地，供 FFmpeg 合成前人工检查衔接。

### 同场景硬切的尾帧传递细节（🆕）

> ⚠️ 因为使用 `--ref-images`（参考模式）而非 `--image`（首帧模式），Seedance 自己决定每段的第一帧。同场景连续叙事时，如果不传尾帧，段间硬切会出现角色位置跳跃、光影突变等不连贯问题。

**当衔接方式为硬切时，执行以下操作**：

1. 上一段完成后，下载尾帧 `lastframe_segN.png`
2. 将尾帧加入下一段的 `--ref-images`（占用 1 个配额，可用位从 9 降为 8）
3. 在下一段的 prompt 中增加开场衔接指令：
   ```
   视频开场画面参考图片N（上一段尾帧），保持角色位置、光影方向和构图与该图一致，从该画面状态自然延续。
   ```
4. 在 prompt 中用内联指代标注：
   ```
   视频开场画面参考`图片N`中上一段尾帧的角色位置和光影，从该画面状态自然延续。
   ```

**ref-images 配额调整示例**（双角色 + 尾帧传递）：

```
图片1: 定场帧
图片2: 情绪特写帧
图片3: 角色A 设定图
图片4: 角色B 设定图
图片5: 角色A 表情变体
图片6: 角色B 表情变体
图片7: 上一段尾帧（硬切衔接用）
[图片8-9: 补充参考图，如有]
```

**不传尾帧的情况**（crossfade / 黑场 fade）：
- 跨场景时不传尾帧，配额保持 9 张
- crossfade 的 0.5s 叠化足以掩盖首帧差异

### 步骤 3：设计内心独白位置（💭 画外音）

对照剧本中的心理转折点，为每段确定内心独白位置（每段最多一句）。内心独白是画外音——角色嘴不动，观众听到心声。格式参考 `prompt-guide.md` §四「内心独白使用指南」。

| 段 | 时机 | 作用 |
|----|------|------|
| 段1 | （如有） | 揭示伪装之下的真实意图 |
| 段2 | （如有） | 标记信息不对等被打破的瞬间 |
| 段3 | （如有） | 压缩内心推理为一句 |
| 段4 | （如有） | 对核心反转的确认/重校准 |

### 步骤 4：集级可迁移分析（🆕 做一次，四段复用）

> ⚠️ 不再逐段独立做可迁移分析。同一集的四段共享角色设定图、场景（或有限场景切换）、灯光哲学，分析结论按"场景"组织，同场景内多段直接复用。

```markdown
## EP{NN} 可迁移分析

### 场景 A（{场景名}，seg{N}-seg{N}）
- 可迁移：空间布局、家具摆设 → 定场帧已覆盖
- 可迁移：角色服装细节 → 设定图已覆盖
- 不可迁移：对白、动作时序、情绪走向、运镜

### 场景 B（{场景名}，seg{N}）
- 可迁移/不可迁移：同上格式
```

如需补充参考图（服装/道具/灯光），在此步骤统一生成，不穿插在逐段精简中。

### 步骤 5：一次性精简四段 prompt

有了集级分析结论，每段精简是机械操作——移除已标记的可迁移项，在 prompt 正文中用内联 `图片N` 指代参考图用途（对齐官方指南）。

### 步骤 6：统一资产登记

assets.md 记录：
- 精简版 prompt（实际提交给 Seedance 的）
- 集级可迁移分析（一份，不是四份）
- 补图记录 + 衔接规划表 + 内心独白汇总表

> 不再同时存完整版和精简版 prompt。如需回溯完整版，从精简版 + 分析 + ref-images 可反推。

### 步骤 7：用户审核

四段 prompt（精简版）+ 内心独白 + 衔接规划 + 完整参数一次性发给用户审核。确认后进入提交阶段。

## 参数模板（每段）

```
--prompt "{精简版 prompt，正文中用内联 `图片N` 指代参考图用途}"

--ref-images（顺序必须与 prompt 中 `图片N` 编号一致）
  图片1: segN/frame01.jpg (定场构图和光影)
  图片2: segN/frame02.jpg (情绪高点构图和面部光照)
  图片3: {角色A}/front.jpg (角色A外观)
  图片4: {角色B}/front.jpg (角色B外观)
  图片5: {角色A}/expressions/{emotion}.jpg (角色A当前情绪面部)
  图片6: {角色B}/expressions/{emotion}.jpg (角色B当前情绪面部)
  [图片7-9: 补充参考图]

--audio（顺序必须与 prompt 中映射声明的音频编号一致）
  音频1: {角色A}/voice-ref.mp3
  音频2: {角色B}/voice-ref.mp3

--ratio 9:16 --duration 15 --generate-audio true
[每段都传 --return-last-frame true，尾帧全部下载到本地供 FFmpeg 前人工检查]
[尾帧不传入下一段 --ref-images，除非同一场景连续叙事]
```

## 提交顺序

段1 → 等待成功 → [可选：最佳帧反馈] → 段2 → 等待成功 → [可选：最佳帧反馈] → 段3 → 等待成功 → [可选：最佳帧反馈] → 段4 → 进入合成。

> ⚠️ 四段的 prompt 和参数在步骤 7 已审核通过。串行提交阶段只执行提交+等待+下载，不重新组装 prompt。
> 段级进度在 `05-videos/ep{NN}/assets.md` 中追踪（seg1 submitted/completed, seg2 submitted/completed...）。
> 原阶段 08-10 的独立文件不再需要——串行提交逻辑已包含在本阶段内。

### 段级并行化（🆕 可选，满足条件时可用）

> 默认串行提交。但如果四段分属不同场景且衔接方式为 crossfade 或黑场 fade（不需要尾帧传递），理论上可以并行提交，将单集视频生成时间从 4×等待 缩短到 1×等待。

**并行条件**（全部满足才可并行）：
- [ ] 四段之间没有硬切衔接（不需要尾帧传入下一段）
- [ ] 不执行最佳帧反馈（并行时无法用上一段的最佳帧）
- [ ] 四段的 ref-images 互相独立（不共享上一段的产出）

**并行方式**：
```bash
# 同时提交四段（不等待）
python3 <video-gen>/scripts/seedance.py create --prompt "{seg1}" ... &
python3 <video-gen>/scripts/seedance.py create --prompt "{seg2}" ... &
python3 <video-gen>/scripts/seedance.py create --prompt "{seg3}" ... &
python3 <video-gen>/scripts/seedance.py create --prompt "{seg4}" ... &
# 等待全部完成
wait
```

**风险**：并行提交可能触发 API 并发限制（取决于火山方舟账户的 QPS 配额）。如果遇到限流错误，退回串行模式。

### 段级最佳帧反馈（🆕 可选，可跳过）

> 核心思路：如果当前段生成的角色面部效果好，截取最佳帧替换后续段的角色设定图参考。
> 后续段的参考从"静态设定图"进化为"已在本集场景光影中验证过的角色面部"，一致性更高。
>
> ⚠️ 前提：当前段的角色效果已通过目测检查。如果当前段角色效果不好，不要截帧传播错误，应重新生成当前段。

**执行流程**（每段完成后，下一段提交前）：

```bash
# 1. 播放当前段视频，人工选择角色面部最清晰、最符合设定的时间点
# 2. 截取该帧
ffmpeg -i ./05-videos/ep01/seg1.mp4 -ss {时间} -vframes 1 \
  ./05-videos/ep01/seg1_best_face_A.jpg

# 多角色场景可能需要截取不同时间点（A 和 B 不一定在同一帧都最清晰）
ffmpeg -i ./05-videos/ep01/seg1.mp4 -ss {时间A} -vframes 1 \
  ./05-videos/ep01/seg1_best_face_A.jpg
ffmpeg -i ./05-videos/ep01/seg1.mp4 -ss {时间B} -vframes 1 \
  ./05-videos/ep01/seg1_best_face_B.jpg
```

**后续段 ref-images 调整**：

用截取的最佳帧替换原始角色设定图（front.png）的位置：

```
原始：图片3: {角色A}/front.jpg (角色A外观)
替换：图片3: seg1_best_face_A.jpg (角色A外观，段1最佳帧)
```

prompt 中内联指代同步更新：
```
角色形象参考`图片3`中的{角色A名}（段1最佳帧，替代设定图）。
```

**跳过条件**（以下任一满足即可跳过）：
- 当前段角色面部效果一般，不值得作为参考
- 当前段与下一段场景/光影差异大（不同场景的最佳帧反而会误导）
- 时间紧迫，不想增加操作步骤

**资产登记**：如果执行了最佳帧反馈，在 assets.md 中记录截帧时间点和文件名。

### Seedance 任务失败重试策略（🆕）

> Seedance API 可能超时、返回质量不合格、或触发安全过滤。以下定义失败后的处理流程。

| 失败类型 | 判断方式 | 处理 |
|----------|---------|------|
| API 超时/网络错误 | `seedance.py wait` 返回错误 | 原参数重试，最多 2 次 |
| 安全过滤拒绝 | 返回 content_filter 错误 | 调整 prompt 中可能触发过滤的描述（暴力/敏感词），重试 |
| 生成成功但质量不合格 | 人工目测：角色变形/面部崩塌/光影严重偏离 | 调整 prompt 或 ref-images 后重试，最多 2 次 |
| 连续 3 次失败 | 同一段连续 3 次未通过 | 停止该段，报告问题，和用户讨论是否调整 prompt 方向或跳过该段 |

**重试时的参数调整建议**：
- 角色变形 → 在 prompt 中强化"保持角色与设定图一致"
- 光影偏离 → 在 prompt 中强化光线描述
- 面部崩塌 → 检查 ref-images 中角色设定图是否清晰，考虑更换表情变体图
- 安全过滤 → 检查对白内容和场景描述，替换敏感表述

**资产登记**：失败的任务也要记录 task_id 和失败原因，便于后续分析失败模式。
