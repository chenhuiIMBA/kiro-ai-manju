# 阶段 04：角色设计

> 工具：Seedream 5.0（角色设定图 + 表情变体）+ TTS skill（音色参考，可选）
> ⚠️ 必须先用户确认角色 Prompt，再调用生成

## 前置条件

- `01-scripts/character-bible.md` 存在（角色内在画像）
- `03-storyboard/visual-style-spec.md` 存在
- 灯光哲学 + 配色方案已确认

## 角色设计层级分档（🆕）

> 不是所有角色都需要同等深度的视觉设计。按出场频率和叙事重要性分三档，避免浪费或缺失。

| 档位 | 适用角色 | 四视图 | 表情变体 | 音色参考 | 判断标准 |
|------|---------|--------|---------|---------|---------|
| A档·主角 | 几乎每集每段出现 | ✅ 正面+背面+左侧+右侧 | ✅ 3-4张（按情绪弧线定制） | ✅ 强烈建议 | 贯穿全剧的核心角色 |
| B档·重要配角 | 多集出现，有关键剧情线 | ⚠️ 仅正面图 | ✅ 2张（按角色功能定制） | 可选 | 跨集出现且有独立剧情线的配角 |
| C档·路人配角 | 单集/单段出现 | ❌ 不生成 | ❌ 不生成 | ❌ | 仅在特定场景出现的功能性角色 |

**为什么配角也需要正面参考图**：在多集多段的长剧中，Seedance 没有参考图会导致同一配角在不同集/段中面孔不一致。正面图足够锁定面孔辨识度，不需要四视图（配角很少有背对镜头的独立镜头）。

**分档时机**：步骤 1 编写 Prompt 之前，先对照 `character-bible.md` 和 `episode-skeleton.md`，为每个角色确定档位。

## 流程

### 步骤 1：编写角色视觉 Prompt

为每个角色编写 Seedream prompt。参照 character-bible.md 的外在描述，注入画风规格。

> ⚠️ **Prompt 审核门禁**：组装完成后，对照 `references/prompt-guide.md` §八「图片 Prompt 审查清单」角色设计专项 + 组图输出逐项自查。然后将完整 prompt + 参数发送给用户审核，确认后才可提交。

**Prompt 模板**：
```
{画风}风格角色设定图，{角色名}，{性别}，约{年龄}岁。
外貌特征：{脸型、发型、发色、眼睛颜色和形状、肤色、身高体型}。
服装设计：{上衣、下装、鞋子、配饰的详细描述}。
气质：{冷酷/温柔/活泼/神秘等}。
一套4张角色转面图，分别为正面全身站立、背面全身站立、左侧面全身站立、右侧面全身站立。
角色保持相同姿势和服装，仅旋转角度。
中灰背景（18% gray），专业角色设计参考图，线条干净利落，9:16竖版构图。
```

### 步骤 1.5：避免千篇一律——角色差异化设计

> 同一个"日系美少年"Prompt 跑 4 个角色，会产出 4 个长得像兄弟的人。必须在 Prompt 中注入**差异化锚点**。

#### 面容特征对撞表

每个角色的面容必须从以下维度中做出**互斥选择**：

| 维度 | 角色A（示例） | 角色B（示例） | 角色C（示例） |
|------|-------------|-------------|-------------|
| 脸型 | 瘦长瓜子脸 | 方正国字脸 | 圆润鹅蛋脸 |
| 眼型 | 细长凤眼，眼角上挑 | 大圆眼，眼角下垂 | 三角眼，眼神锐利 |
| 眉型 | 剑眉，眉尾上扬 | 柳叶弯眉，柔和 | 粗短浓眉，眉头紧锁 |
| 鼻型 | 高挺窄鼻 | 圆润小巧 | 鹰钩鼻，鼻梁微弯 |
| 嘴型 | 薄唇，嘴角自然下撇 | 厚唇，嘴角微扬 | 小嘴，唇线分明 |
| 下颌 | 尖下巴 | 方下巴 | 圆下巴 |

> ⚠️ 生成前检查：任意两个角色的面容特征是否有 ≥3 项不同？不够就调。

#### 体态/站姿差异

| 体态类型 | 描述 | 适合角色 |
|---------|------|---------|
| 挺拔直立 | "脊椎笔直，肩膀后展，下巴微抬" | 自信、傲慢、军人 |
| 慵懒微斜 | "重心偏向一侧，肩膀放松下垂，头微偏" | 随性、松弛、浪子 |
| 蜷缩内收 | "肩膀内收，微微含胸，视线向下" | 自卑、惊恐、隐藏 |
| 前倾攻击 | "上半身前倾，拳头微握，眼神锁定目标" | 好斗、紧迫、威胁 |

#### 服装差异化原则

> "古风长袍" × 4 个角色 = 4 个换色皮肤。必须注入**材质和穿法差异**：

- 角色A：丝绸质地，修身剪裁，有光泽
- 角色B：粗麻质地，宽松披挂，有磨损
- 角色C：双层叠穿，内紧外松，有功能口袋
- 角色D：单肩披风，不对称设计，随风飘动

### 步骤 2：用户确认

提交每个角色的 Prompt。用户确认后进入生成。

### 步骤 3：生成角色四视图（正面先行 + 图生图旋转 🆕）

> ⚠️ 不使用组图模式（`--seq auto --max-images 4`）——实测组图模式容易出现顶部裁切、方向不符、缺少视角等问题。
> 改为单张逐步生成：先文生图确定正面形象，再用图生图引用正面图生成其他方向，角色一致性更好。

**步骤 3a：生成正面全身图（文生图）**

```bash
python3 <video-gen>/scripts/seedream.py create \
  -p "{确认后的角色 prompt，指定正面全身站立}" \
  --size 9:16 \
  -d ./04-human-design/{角色名}/
# 重命名为 front.jpg
mv ./04-human-design/{角色名}/seedream_*.jpg ./04-human-design/{角色名}/front.jpg
```

用户确认正面图后再继续。正面图是后续所有方向图和表情变体的基准。

> ⚠️ **即时登记（强制）**：每生成一张图（正面/背面/左侧/右侧），必须立即执行：生成 → 重命名 → 写入 assets.md → 再生成下一张。禁止批量生成完再回头补录。

**步骤 3b：生成背面/左侧/右侧（图生图，引用正面图）**

```bash
# 背面
python3 <video-gen>/scripts/seedream.py create \
  -p "参考图中的角色形象，保持服装、发型、体型完全一致。将角色旋转为背面全身站立姿势，展示背部和后脑勺。纯色中灰背景，均匀柔光照明，9:16竖版构图。" \
  -i ./04-human-design/{角色名}/front.jpg \
  --size 9:16 \
  -d ./04-human-design/{角色名}/
# 重命名为 back.jpg

# 左侧
python3 <video-gen>/scripts/seedream.py create \
  -p "参考图中的角色形象，保持服装、发型、体型完全一致。将角色旋转为左侧面全身站立姿势，面朝画面左侧，展示左侧轮廓。纯色中灰背景，均匀柔光照明，9:16竖版构图。" \
  -i ./04-human-design/{角色名}/front.jpg \
  --size 9:16 \
  -d ./04-human-design/{角色名}/
# 重命名为 left.jpg

# 右侧
python3 <video-gen>/scripts/seedream.py create \
  -p "参考图中的角色形象，保持服装、发型、体型完全一致。将角色旋转为右侧面全身站立姿势，面朝画面右侧，展示右侧轮廓。纯色中灰背景，均匀柔光照明，9:16竖版构图。" \
  -i ./04-human-design/{角色名}/front.jpg \
  --size 9:16 \
  -d ./04-human-design/{角色名}/
# 重命名为 right.jpg
```

### 步骤 4：生成表情变体（按情绪弧线定制 🆕）

> ⚠️ **不使用固定模板**。表情变体的种类由以下三个因素决定，而非通用的"愤怒/悲伤/微笑/惊恐"四件套。

#### 表情变体选择三要素

1. **角色在剧中的情绪弧线**：对照 `episode-skeleton.md` 和 `character-bible.md`，这个角色在全剧中主要经历哪些情绪状态？如果角色从头到尾没有惊恐的戏，就不生成惊恐变体。
2. **角色性格的表达方式**：同样是"生气"，不同角色的表现完全不同——有人是隐忍咬唇，有人是冰冷压制，有人是直接爆发。表情变体必须匹配角色的表达方式，而非通用表情。
3. **Seedance 的实际需求**：表情变体的用途是在视频生成时挂入 `--ref-images`，帮助 Seedance 在情绪剧烈变化时保持面部一致性。只有角色在段内有明显情绪转折的表情才需要变体。

#### 执行流程

1. 对照 `episode-skeleton.md`，列出每个角色的关键情绪转折点
2. 从中提炼出 2-4 种最高频/最关键的表情状态
3. 为每种表情编写定制化的 Prompt（融入角色性格的表达方式）
4. 提交用户确认后逐张生成

#### 表情变体 Prompt 模板

```
参考图中的角色形象，保持面部特征和服装完全一致。
正面半身构图，表情为{情绪名}——{该角色特有的表现方式，2-3个具体面部细节}。
纯色中灰背景（#808080），均匀柔光照明，9:16竖版构图。
```

> ⚠️ `{该角色特有的表现方式}` 是关键——不能写通用的"咬牙切齿"，要写这个角色独有的表达。例如：
> - 内敛角色的愤怒："下颌线绷紧，薄唇紧抿，眼神变冷如冰，整张脸像被冻结"
> - 倔强角色的悲伤："眼眶泛红但拼命忍住，嘴唇微颤，眉尾下垂，努力不让眼泪掉下来"
> - 伪装者的冷笑："嘴角上扬但眼睛没有笑意，金丝眼镜后的目光冰冷计算"

#### 生成命令（逐张）

```bash
python3 <video-gen>/scripts/seedream.py create \
  -p "{定制化的表情 prompt}" \
  -i ./04-human-design/{角色名}/front.jpg \
  --size 9:16 \
  -d ./04-human-design/{角色名}/expressions/
# 重命名为 {表情名}.jpg
```

> ⚠️ **即时登记（强制）**：每生成一张表情变体，必须立即执行：生成 → 重命名 → 写入 assets.md → 再生成下一张。禁止批量生成完再回头补录。跨会话时 assets.md 是唯一的恢复锚点，漏登 = 丢失 Prompt 记录。

#### A档主角 vs B档配角的表情数量

- A档主角：3-4 张，覆盖角色弧线的关键情绪两极（如"戴面具"vs"面具碎了"）
- B档配角：2 张，覆盖角色最核心的功能性表情（如"伪装面"vs"真面目"）

### 步骤 4.5：角色灯光约束认知

> 角色设定图使用均匀柔光（保证4面图一致性），意味着 `--ref-images` 中的角色是"无影"的。
> 当故事板场景光为硬侧光时，角色设定图可能被 Seedance 误读。

对策（按优先级）：
1. **接受差距**：故事板帧场景光描述足够强，角色图仅约束外观
2. **灯光匹配变体**（精细项目）：用故事板的光影条件图生图改造角色正脸
3. **视频 prompt 补偿**：视频 prompt 强化光线描述，抵消弱光参考帧的拉扯

### 步骤 6：角色音色参考（🎤 强烈建议）

> 角色印象由眼+耳共同建立。Seedance 的 `--audio` 参数会模仿此音色生成配音。
> 前置：`tts` skill 已安装，`DOUBAO_TTS_API_KEY` 已设置。

为每个重要角色生成一段 5-8 秒的标志性音色参考：

```bash
python3 <tts-skill>/scripts/doubao_tts.py synthesize \
  --text "{最能体现角色性格的一句台词}" \
  --speaker {匹配角色气质的音色} \
  --emotion {角色默认情绪基调} \
  --encoding mp3 \
  -d ./04-human-design/{角色名}/

# 重命名为 voice-ref.mp3
mv ./04-human-design/{角色名}/tts_*.mp3 ./04-human-design/{角色名}/voice-ref.mp3
```
