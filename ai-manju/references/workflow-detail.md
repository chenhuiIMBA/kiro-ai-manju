# AI 漫剧工作流 - 分步详解

> 加载本文件以获取完整的、可执行的步骤和命令模板。

## 前置条件

- 已设置 `ARK_API_KEY` 环境变量（火山方舟 API Key）
- 已安装 `video-generation` skill（提供 seedance.py / seedream.py）
- 当前工作目录为项目根目录

---

## 阶段一：项目初始化

### 1.1 创建项目目录

```bash
python3 <ai-manju-skill>/scripts/init_project.py <项目名>
cd /root/.openclaw/projects/<项目名>/
```

### 1.2 收集创意输入

向用户询问：
- 故事题材（玄幻/都市/悬疑/甜宠/其他）
- 核心角色数量和基本设定方向
- 画风偏好（日漫/韩漫/国漫/写实/插画）
- 对话呈现风格：字幕模式 / 气泡台词模式 / 混合（日常字幕、情绪高潮气泡）
- 每集时长（默认 60 秒）
- 总集数（如 20 集）
- 目标受众（如 18-25 岁女性、二次元爱好者）

### 1.3 画风细化（强烈推荐）

> ⚠️ 「日漫」不是一种风格——90 年代赛璐珞和 2010s 京都动画的视觉基因完全不同。细化画风能大幅降低 Moodboard 阶段的迭代次数。

从以下维度细化画风，用于后续所有 prompt：

| 维度 | 可选方向 | 示例 |
|------|---------|------|
| 年代参照 | 80s赛璐珞 / 90s经典 / 2000s数字 / 2010s现代 / 当代 | 2010s 现代数字动画 |
| 作品/工作室参照 | 京都动画风 / 新海诚风 / 吉卜力风 / 霸权社风 / MAPPA 风 | 新海诚风——极致光影、照片级背景 |
| 线条特征 | 粗重外轮廓 / 纤细均一 / 有笔触变化 / 无线条厚涂 | 纤细均一、干净利落 |
| 上色方式 | 硬边平涂 / 软边渐变 / 厚涂 / 水彩渗化 | 软边渐变、柔和过渡 |
| 阴影风格 | 分层色块式 / 柔和模糊 / 硬边阴影 / 网点纸 | 分层色块式、2-3 层明暗 |

将选定特征写入 `storyboard/visual-style-spec.md`，供后续所有 prompt 模板引用。

## 阶段二：整季叙事大纲

### 2.1 产出内容

存入 `scripts/series-bible.md`：

```markdown
# {剧名} 整季大纲

## 主题
这部剧真正在讲什么。
要求：具体、可争论。不是"爱与正义"，而是"偏执的复仇最终吞噬复仇者本身"。

## 中心戏剧问题
驱动全剧的核心问题。
例："林小夜能在找到凶手的同时不失去自己的人性吗？"
每集都在回答这个问题的一小部分。

## 系列大弧
主角从{起点}到{终点}的完整旅程。要体现主题。

## 人物弧
### {角色A}
- **Want（表面欲望）**：角色 conscious 想要什么
- **Need（深层需求）**：角色 unconscious 真正需要什么（Want ≠ Need 才有戏剧）
- **Flaw（核心缺陷/错误信念）**：什么在阻碍他
- **Ghost（创伤事件）**：造成缺陷的过去经历
- 起始状态：{性格、处境}
- 成长路径：{遭遇什么、如何改变}
- 终局状态：{性格、处境}

## 角色关系图
```
{角色A} ←{关系类型}→ {角色B}
  ↓ {关系类型}           ↓ {关系类型}
{角色C} ←{关系类型}→ {角色D}
```
标注：信赖/敌对/暗恋/利用/保护/竞争/师徒等

## 分集大纲
| 集数 | 标题 | 核心事件 | 悬念钩子 | 主题推进 |
|------|------|---------|----------|---------|
| 01 | ... | ... | ... | 建立世界观和初始冲突 |
| 02 | ... | ... | ... | ... |
...
| 20 | ... | ... | 结局 | 主题的最终回答 |

## 冲突主线
- A 故事线（主线情节）：...
- B 故事线（支线/感情线）：...

## 戏剧反讽设计（可选但强力）
观众知道但角色不知道的关键信息。
例：观众在第 5 集就已知凶手身份，但主角到第 18 集才发现——这种信息差制造持续张力。
```

### 2.2 审核要点

- 主题是否具体可争论（不是空洞口号）
- 每集是否有明确的信息增量和主题推进
- 悬念钩子是否足够强（看完了这集想知道下集）
- 人物 Want ≠ Need 是否成立（相同就不对，失去戏剧张力）
- 人物弧是否有说服力
- 关系图是否产生天然的戏剧冲突

### 2.3 生成分段骨架（🆕）

整季大纲确认后，分批生成 80 段骨架（每次 5 集），存入 `scripts/episode-skeleton.md`。

每批带着前面已确认的骨架作为上下文，避免后半段质量下降。

骨架确认后，升级分集大纲表格，增加"信息揭露"和"角色弧位置"列。

### 2.4 初始化叙事状态追踪（🆕）

产出 `scripts/narrative-state.md`（初始版本）。从骨架中提取"未揭露信息"计划表，初始化角色状态。

此文件在每集剧本完成后更新（阶段五负责），是 AI 编写后续集剧本的核心上下文输入。

## 阶段三：风格板（Moodboard）

### 3.1 目的

用 Seedream 快速出 3-5 张不同场景、不同光影的参考图，让用户确认视觉方向。**避免盲写 prompt 反复修改角色和场景。**

### 3.2 产出

```bash
python3 <video-gen>/scripts/seedream.py create \
  -p "{画风}风格概念图。{场景描述}，{光影}，{色调}。高品质概念插画。" \
  --size 9:16 \
  -d ./moodboard/
```

建议覆盖以下场景类型（详见 [prompt-guide.md](prompt-guide.md) §一 Moodboard 模板表格）：
白天外景、夜间内景、室内日景（窗光）、逆光/剪影、人物特写、室内混合光源、大场景、黄金时刻、蓝调时刻、雨景/水景。

根据题材类型选取 5-8 种即可，不需要全部覆盖。

### 3.3 审核

用户确认风格后再进入角色设计。不满意则调整方向重新出。

## 阶段三·五：灯光哲学定调

> ⚠️ **灯光方案在角色设计之前完成，不在故事板之后。**
> 灯光是视觉骨架——它决定角色在什么光里被看见、场景用什么情绪被感知。

### 3.5.1 目的

从已确认的风格板中提炼整部剧的灯光哲学。这不是每场的具体灯光方案，而是贯穿全剧的光影 DNA。

存入 `storyboard/lighting-philosophy.md`。

### 3.5.2 灯光哲学文档

```markdown
# {剧名} 灯光哲学

## 灯光基调
- **高调/低调**：{high-key（明亮、低反差、轻快喜剧感） / low-key（暗调、高反差、戏剧性） / 混合}
- **影调**：{硬阴影（黑色电影感，锐利阴影边缘） / 软阴影（日系治愈，阴影过渡柔和）}
  - 选硬阴影则全剧影子都硬，选软阴影则全剧影子都软。不混用。

## 色彩策略
- **主导色温**：{暖色主导 / 冷色主导 / 中性基准}
- **冷暖配比**：{大致比例和切换规则}
  - 例：日常场景暖 70%:冷 30%，悬念场景冷 60%:暖 40%，闪回暖 100%
- **色彩对比策略**：{互补色 / 冷暖共存 / 单色渐变 / 混合}
  - 互补色：橙（肤色）+ 青（背景）— 最经典的人物分离
  - 冷暖共存：暖主光 + 冷环境 → 温暖中带疏离；冷主光 + 暖环境 → 冷漠中有人情
  - 单色渐变：全暖或全冷，靠亮度分层 → 高度风格化

## 灯光母题（Motif）
贯穿全剧的灯光签名。一到两个即可。
- 例：每当真相逼近，画面中出现一束从上方垂直打入的顶光，角色一半脸在光中一半在暗里
- 例：所有情感转折场景，背景中必有一盏闪烁的裸灯
- 例：女主获得力量时，她的影子总是比她本人更大

## 实用光源哲学
画面内的可见光源（窗、灯、烛、屏、车灯）如何驱动灯光设计：
- **现实主义**：所有可见光都必须有画面内的实用光源作为动机
- **表现主义**：允许无动机光源，但必须与情绪一致（恐怖/梦境/风格化）
- **混合**：日常场次实用光源驱动，关键情绪转折允许表现主义

## 曝光策略
- **曝光基准**：{以主角面部皮肤亮度为基准 / 以场景关键道具为基准}
- **高光策略**：{保留高光细节 / 允许局部过曝（什么场景下）}
- **暗部策略**：{纯黑 crushing（恐怖/悬疑）/ 保留暗部细节 lifted（文艺/日常）}
- **曝光弧线**：{随情绪变化升曝或降曝的规则}

## 分离光策略
角色从背景中分离的方式：
- **发型光/轮廓光**：{使用频率和场景类型}
  - 始终使用 / 仅暗调场景 / 仅关键角色 / 不使用（依赖景深分离）
  > ⚠️ **风格提示**：「始终使用轮廓光」是日系动画的核心美学特征——它让角色始终从背景中浮出，产生"角色是画面主角"的感觉。「不使用轮廓光」则更接近实拍电影感，适合追求写实质感的韩漫/国漫/写实风格。两种选择导致的视觉基调完全不同。
- **背景光分离**：{用背景亮度梯度还是色彩对比分离角色}

## 特殊灯光规则
- {镜头类型特定规则}
  - 例：所有特写必须有眼神光（catchlight），不允许眼睛无光
  - 例：过肩镜头中，前景角色肩部可入暗，但对面角色面部必须正确曝光
```

### 3.5.3 审核要点

- [ ] 灯光基调是否与题材匹配（甜宠不适合纯 low-key，悬疑不适合纯 high-key）
- [ ] 冷暖配比是否形成情绪弧线而非平铺
- [ ] 灯光母题是否具体可执行
- [ ] 分离光策略是否确保角色不从暗背景中消失
- [ ] 曝光策略是否在实际 AI 生成中可实现

### 3.5.5 配色方案（Color Palette）

> ⚠️ 色彩策略定义视觉方向，配色方案定义具体颜色。没有色板，每张 Seedream 帧的颜色都会有偏差——20 集后角色外套可能已从深紫漂成藏青。

从已确认的风格板和灯光哲学中提取关键颜色，存入 `storyboard/color-palette.md`：

```markdown
# {剧名} 配色方案

## 全剧色板

| 色板元素 | 颜色名称 | 语义标记 |
|----------|---------|----------|
| 主角肤色亮面 | 暖象牙白 | skin_highlight |
| 主角肤色暗面 | 浅玫瑰棕 | skin_shadow |
| 主角服装主色 | 深靛蓝 | charA_primary |
| 场景日景天空 | 淡青 | sky_day |
| 场景夜景主调 | 深紫蓝 | night_ambient |
| 日景环境阴影 | 暖灰绿 | env_shadow_day |
| 暖光源光色 | 暖金 | warm_key |
| 冷光源光色 | 冷蓝 | cool_key |
| ... | ... | ... |

## 使用规则

在故事板和视频 prompt 中使用语义标记引用色板，确保全剧色彩一致：
- 角色描述中引用："角色服装为深靛蓝（charA_primary）"
- 场景描述中引用："天空呈淡青色（sky_day）"
- 光源描述中引用："暖金色主光（warm_key）从右侧打入"
```

> 💡 实际色值由 AI 通过语义标记理解和关联，效果可能不完全精确。关键是通过一致的命名体系减少漂移，而非追求像素级颜色匹配。

## 阶段四：角色设计

### 4.1 生成角色设定文档（编剧层面）

在生成画面 prompt 之前，先为每个角色建立内在画像，存入 `scripts/character-bible.md`：

```markdown
## {角色名}
### 外在
- 外貌：{描述}
- 服装：{描述}
- 标志性动作/物品：{咬嘴唇/转戒指/永远戴着帽子}

### 内在（四维）
- **Want**：{表面欲望——他 conscious 想要什么}
- **Need**：{深层需求——他 unconscious 真正需要什么。关键：Want ≠ Need}
- **Flaw**：{核心缺陷/错误信念——什么在阻碍他得到 Need}
- **Ghost**：{创伤事件——什么过去造成了这个 Flaw}

### 语言指纹
- 说话风格：{简短/啰嗦/讽刺/温柔/命令式/犹豫}
- 口头禅/常用句式：{...}
- 对白时的小动作：{说话时回避眼神/手势很多/习惯性摸脖子}
```

### 4.2 生成角色 Prompt

参考 `video-generation` skill 的 `references/seedream-prompt.md` 指南。

对每个角色生成 4 张图（前/后/左/右）的 Seedream prompt。要求：
- 尺寸：9:16（`--size 9:16`）
- 使用组图模式：`--seq auto --max-images 4`
- Prompt 中明确指定 4 个方向："正面站立、背面站立、左侧站立、右侧站立"
- 角色要有特点，避免千篇一律
- 画风必须与 Moodboard 确认的风格一致

**Prompt 模板：**
```
{画风}风格角色设定图，{角色名}，{性别}，约{年龄}岁。
外貌特征：{脸型、发型、发色、眼睛颜色和形状、肤色、身高体型}。
服装设计：{上衣、下装、鞋子、配饰的详细描述}。
气质：{冷酷/温柔/活泼/神秘等}。
一套4张角色转面图，分别为正面全身站立、背面全身站立、左侧面全身站立、右侧面全身站立。
角色保持相同姿势和服装，仅旋转角度。
纯色浅灰背景，专业角色设计参考图，线条干净利落，9:16竖版构图。
```

### 4.3 用户确认后生成

```bash
python3 <video-gen>/scripts/seedream.py create \
  -p "<确认后的prompt>" \
  --size 9:16 \
  --seq auto --max-images 4 \
  -d ./human-design/{角色名}/
```

### 4.3.5 角色灯光约束

> ⚠️ 角色设定图使用均匀柔光是为了 4 面图一致性，但这意味着 `--ref-images` 中的角色是"无影"的。
> 当故事板场景光是硬侧光时，无影的角色设定图可能被 Seedance 误读，削弱场景光效果。

对策（按优先级）：
1. **接受差距**：故事板帧的场景光描述足够强，角色设定图仅约束外观
2. **灯光匹配变体**（精细项目）：用故事板的光影条件图生图改造角色正脸，存为 `human-design/{角色}/lighting-variants/`
3. **视频 prompt 补偿**：视频 prompt 中强化光线描述，抵消弱光参考帧的拉扯

### 4.3.6 表情变体图（Expression Sheet）

> ⠅ 重要角色必须生成表情变体。详见 `prompt-guide.md` §2.5。
> 表情变体图在后续视频生成中作为 `--ref-images` 的情绪锚点，防止面部一致性崩塌。

为每个重要角色生成 4 张表情变体（愤怒/悲伤/微笑/惊恐），存入 `human-design/{角色名}/expressions/`。
模板和命令见 [prompt-guide.md](prompt-guide.md) §2.5。

### 4.5 角色音色设计（🎤 强烈建议，需 tts skill）

> 🎤 **A+ 方案**：为每个角色生成标准音色参考音频，Seedance 的 `--audio` 参数会模仿此音色生成配音。
> 角色在观众脑中的印象是眼+耳一起建立的——不能只有视觉审查没有声音审查。
>
> 前置条件：已安装 `tts` skill 并设置 `DOUBAO_TTS_API_KEY` 环境变量（新版控制台）。

为每个重要角色生成一段 5-8 秒的标志性音色参考：

```bash
python3 <tts-skill>/scripts/doubao_tts.py synthesize \
  --text "{最能体现角色性格的一句台词，例如：我叫林小夜。你不用帮我。}" \
  --speaker {匹配角色气质的音色} \
  --emotion {角色默认情绪基调} \
  --encoding mp3 \
  -d ./human-design/{角色名}/

# 重命名为 voice-ref.mp3
mv ./human-design/{角色名}/tts_*.mp3 ./human-design/{角色名}/voice-ref.mp3
```

音色选择建议：

| 角色类型 | 推荐音色 | emotion |
|---------|---------|----------|

## 阶段五：剧本生成（先做一集）

### 5.0 加载叙事上下文（🆕）

写剧本前必须读取：series-bible + character-bible + episode-skeleton（本集骨架）+ narrative-state（前面集的累积状态）。

不要读取前面所有集的完整剧本——narrative-state 已浓缩关键信息。

### 5.1 先选结构模板

60 秒 = 4×15 秒段落。必须先选结构模板，再按四段填充。不能自由发挥。

**模板 A：微三幕（Micro Three-Act）**
```
钩子(3s) → 布设(10s) → 对抗(30s) → 高潮(10s) → 揭示/悬念(7s)
```
适用：完整叙事弧，有头有尾有钩子。

**模板 B：反转结构（Reversal）**
```
常态(10s) → 暗示异常(15s) → 反转爆发(25s) → 余震(10s)
```
适用：悬疑、惊悚、喜剧反转。

**模板 C：提问-回答（Question-Answer）**
```
抛出问题(5s) → 探索展开(45s) → 新问题(10s)
```
适用：推理、揭秘、信息驱动剧情。

**模板 D：对比结构（Contrast）**
```
A面(25s) → 切换 → B面(25s) → 并置揭示(10s)
```
适用：双线叙事、身份反差、表里不一。

### 5.2 剧本格式

```markdown

**场景目的**：...
**情绪节拍**：...
**灯光节拍**：...
**冲突引擎**：{障碍 → 选择 → 后果}

- 时间段：{start}s ~ {end}s
- 结构定位：{在选定模板中的阶段}

- 对白台词：{精确文本}
```

### 5.3 强制要求

**结构层面**
- ✅ 必须先选定结构模板（A/B/C/D），再填内容
- ✅ 每场戏必须标注"场景目的"（推进剧情 + 揭示角色，至少做两件事）
- ✅ 每集最多 4 次冲突引擎（分布在 4 段中，平均一段一次）
- ✅ 3 秒钩子（首帧决定留存率）

**对白层面**
- ✅ 对白有互动感，你来我往
- ✅ 每句对白标注潜台词 + 对白行动 + 🎤 情感标注
- ✅ 每个角色有语言指纹（对照 character-bible.md）：A 和 B 说同一件事应该完全不同
- ✅ 表情丰富（瞪眼、挑眉、抿嘴、冷笑、眼神闪烁、咬唇等）
- ✅ 动作具体（转身、握拳、后退一步、拍桌、缓缓靠近等）
- ✅ 有停顿/沉默设计（关键时刻的留白——沉默也是行动）
- ✅ 🎤 同一集内 emotion 有变化弧线（不能全 angry 或全 neutral），每段有不同的情绪走向
- ❌ 禁止空洞的"嗯""啊"语气词
- ❌ 禁止只有点头摇头的单调动作
- ❌ 禁止两个角色说话风格雷同

**镜头层面**
- ✅ 标注镜头序列（景别变化节奏）
- ❌ 禁止全程同一景别

### 5.4 拆解分镜

60 秒拆为 4 段（每段 ≤15 秒），每段包含：
- 时间段
- 结构定位（在选定模板中的哪个阶段）
- 情绪节拍
- 场景描述
- 出场角色
- 对白台词（精确文本，用双引号包裹）
- 动作/表情/情感提示

### 5.5 更新叙事状态追踪（🆕 强制）

剧本确认后，立即更新 `scripts/narrative-state.md`：
- 追加本集的"已揭露信息"
- 兑现的"未揭露信息"移入已揭露
- 更新角色态度轨迹、Want 进度、Flaw 松动程度
- 追加新伏笔，标记已兑现伏笔
- 覆盖写"上一集摘要"

不更新 narrative-state 就不能进入故事板阶段。

## 阶段六：故事板（Storyboard）

### 6.1 目的

用 Seedream 为每段分镜生成 2 张关键帧（定场 + 情绪特写），既做视觉规划，又作为后续 Seedance 的 `--ref-images` 入参。

### 6.2 每段应覆盖的镜头类型

| 帧 | 类型 | 用途 |
|----|------|------|
| 帧 1 | 定场镜头（远景/全景） | 场景空间 + 光源方向 + 色调氛围 |

### 6.3 生成命令

```bash
# 每帧单独生成（图生图，挂角色设定图）
python3 <video-gen>/scripts/seedream.py create \
  -p "{画面描述，含构图/景别/光影}" \
  -i ./human-design/{角色}/front.png \
  --size 9:16 \
  -d ./storyboard/ep01/seg1/
```

### 6.4 审核要点

- [ ] 景别序列有节奏感（远→中→近→特 或 反转）
- [ ] 色调与 Moodboard 一致
- [ ] 角色面部清晰可辨，与设定图一致
- [ ] 构图中角色位置合理（不挤在边缘）
- [ ] 帧与帧之间视觉连贯
- [ ] 每帧光影符合 `lighting-philosophy.md`（主光方向、光比、色温在灯光母题框架内）
- [ ] 相邻帧景别或视角有明显差异（AI 适配：不依赖精确角度数值，通过景别/视角/构图的明显变化避免跳切感。Seedance 仅做帧间平滑，不负责精确角度跳变）

**故事板不通过审核，不进视频生成阶段。**

### 6.5 故事板灯光核查

> 不再为每集另写灯光方案——灯光哲学在 §3.5 已定调。本阶段仅核查故事板帧是否执行了灯光哲学。

逐帧对照 `storyboard/lighting-philosophy.md`：

- [ ] 每帧主光方向明确，且与灯光哲学的场景规则一致
- [ ] 光比在灯光哲学定义的范围内（高调/低调基调约束）
- [ ] 色温符合色彩策略（主导色温 + 冷暖配比规则）
- [ ] 灯光母题在关键情绪帧中可见
- [ ] 实用光源驱动正常（现实主义模式下，无无源光）
- [ ] 分离光在暗调帧中有效（角色不融入背景）
- [ ] 特写帧有眼神光（catchlight）
- [ ] 灯光节拍与剧本的灯光节拍对齐

如有帧违反灯光哲学，必须重新生成该帧——不改哲学迁就帧。

### 6.6 镜头衔接与转场标注

> ⚠️ 故事板不只是"独立画面的集合"——帧与帧之间如何过渡直接影响观看体验。
> 在专业分镜中，每两个关键帧之间标注转场类型。虽然 Seedance 不一定精确执行转场指令，但 prompt 中的转场描述会影响过渡风格。

在故事板每段分镜的帧列表后，增加转场标注：

```markdown
### 帧间转场
| 帧过渡 | 转场类型 | Prompt 关键词 |
|--------|---------|-------------|
| 帧1 → 帧2 | 硬切 | 描述中不加过渡词 |
| 帧2 → 帧3 | 叠化 | "画面缓缓过渡"、"柔和叠化" |
| 帧3 → 帧4 | 甩镜 | "镜头快速甩向"、"急速转向" |
| 帧4 → 帧5 | 缓推 | "镜头缓缓推近" |
```

常用转场类型：
- **硬切**：直接切换，快节奏对话、冲突场景
- **叠化**：画面交叠 0.3-0.5s，情绪过渡、时间流逝
- **甩镜**：镜头快速甩向新方向，注意力转移、空间跳跃
- **缓推/缓拉**：镜头缓慢运动，情绪升温或退场感
- **黑场过渡**：先暗再亮，时间/场景大跳跃

### 7.2 视频 prompt 公式

参考 `references/prompt-guide.md` 的漫剧专用模板。

```
{画风}风格动画。
```

- ⚠️ **角色图 vs 场景光矛盾**：角色设定图是均匀柔光，故事板帧可能是硬侧光。Seedance 在两者间可能折中导致场景光被削弱。如果视频中角色面部光影过平，下个任务用故事板帧的光影条件生成角色匹配变体（见 §4.3.5）。

### 7.5 段间衔接规则

段间不是"画面连续"而是"时间过渡"——尾帧只在同一场景连续叙事时传入下一段。

| 衔接 | 条件 | 方式 | 尾帧传入下一段？ |
|------|------|------|-----------------|
| 黑场fade | 色温/场景大跳 | `fade=t=out:st=14.5:d=0.5` + `fade=t=in:st=0:d=0.5` | ❌ 不需要 |
| crossfade | 色温相近、不同空间 | `xfade=transition=fade` | ❌ 不需要 |
| 硬切 | 同一场景连续叙事 | `concat` | ✅ 传尾帧 |

> `--return-last-frame true` 每段都传，尾帧全部下载到本地供 FFmpeg 合成前人工检查衔接。尾帧不自动传入下一段 `--ref-images`（除非判断需要）。

## 阶段八：合成

> 🎬 4 段是独立的 Seedance 任务，不使用 `first_frame` role（多模态模式下 role 互斥）。段间可能存在微小的视觉跳跃。使用 crossfade 过渡消除衔接痕迹。

### 8.0 段间色温一致性检查（🆕 必做）

合成前截取每段首帧横向拼接对比，目测色温/亮度/肤色是否一致。差异明显时用 `colorbalance` 或 `eq` 滤镜轻微校正（以段 1 为基准）。详见 `stages/11-composite.md` 步骤 1。

### 8.1 FFmpeg 交叉淡入淡出（推荐）

每段各截掉衔接处 0.3s，用 0.5s crossfade 过渡：

```bash
# 4 段 crossfade 拼接（3 次渐变）
ffmpeg -i ./videos/ep01/seg1.mp4 -i ./videos/ep01/seg2.mp4 \
       -i ./videos/ep01/seg3.mp4 -i ./videos/ep01/seg4.mp4 \
  -filter_complex \
    "[0:v]trim=0:14.7,setpts=PTS-STARTPTS[v0]; \
     [0:a]atrim=0:14.7,asetpts=PTS-STARTPTS[a0]; \
     [1:v]trim=0.3:14.7,setpts=PTS-STARTPTS[v1]; \
     [1:a]atrim=0.3:14.7,asetpts=PTS-STARTPTS[a1]; \
     [2:v]trim=0.3:14.7,setpts=PTS-STARTPTS[v2]; \
     [2:a]atrim=0.3:14.7,asetpts=PTS-STARTPTS[a2]; \
     [3:v]trim=0.3:15,setpts=PTS-STARTPTS[v3]; \
     [3:a]atrim=0.3:15,asetpts=PTS-STARTPTS[a3]; \
     [v0][v1]xfade=transition=fade:duration=0.5:offset=14.5[v01]; \
     [v01][v2]xfade=transition=fade:duration=0.5:offset=29.0[v012]; \
     [v012][v3]xfade=transition=fade:duration=0.5:offset=43.5[vout]; \
     [a0][a1]acrossfade=d=0.5:c1=nofade:c2=nofade[a01]; \
     [a01][a2]acrossfade=d=0.5:c1=nofade:c2=nofade[a012]; \
     [a012][a3]acrossfade=d=0.5:c1=nofade:c2=nofade[aout]" \
  -map "[vout]" -map "[aout]" \
  -c:v libx264 -preset medium -crf 18 \
  ./output/ep01-final.mp4
```

> 每段首尾各截 0.3s（避免尾帧固定画面突兀+不稳定首帧），段间 0.5s crossfade。总计 3 次渐变，损失 ~1.8s，60s 成片缩短为 ~58.2s——对漫剧体验无感知影响。

### 8.2 硬切拼接（备选）

如果段间衔接本身很流畅，直接用 concat：

```bash
ffmpeg -i ./videos/ep01/seg1.mp4 -i ./videos/ep01/seg2.mp4 \
       -i ./videos/ep01/seg3.mp4 -i ./videos/ep01/seg4.mp4 \
  -filter_complex "[0:v][0:a][1:v][1:a][2:v][2:a][3:v][3:a]concat=n=4:v=1:a=1[outv][outa]" \
  -map "[outv]" -map "[outa]" \
  -c:v libx264 -preset medium -crf 18 \
  ./output/ep01-final.mp4
```

### 9.1 检查清单

**画面**

- [ ] 无画面撕裂/闪烁/变形
- [ ] 色彩/光影与灯光哲学一致

**光影专项**
- [ ] 主光方向在镜头间保持一致（非叙事动机变化时）
- [ ] 光比与场景情绪匹配，无随机漂移
- [ ] 色温在整段内无意外跳变
- [ ] 角色面部曝光一致（不突然过亮或过暗）
- [ ] 分离光有效（角色不从暗背景中消失）

- [ ] 特写中眼神光（catchlight）存在且方向一致
- [ ] 实用光源有画面内动机（如灯光哲学选择现实主义）
- [ ] 灯光母题在关键帧中可见

**声音**
- [ ] 配音清晰可辨，无含混、吞字
- [ ] 角色声音区分度明显（A 和 B 闭眼能分辨）
- [ ] 各角色音色与 voice-ref.mp3（如有）一致
- [ ] 情感表现符合剧本标注（愤怒够怒、悲伤不假）
- [ ] 语速节奏自然，与画面情绪同步
- [ ] 环境音效符合场景
- [ ] 音量无突变/爆音
- [ ] 对白无缺失（每句剧本对白都在成片中出现）

**AI 生成专项**
- [ ] 无手指/肢体变形（AI 经典问题：多指、扭曲、消失）
- [ ] 面部身份一致（同段视频内角色面部无漂移变脸）

- [ ] 无 AI 文字/字幕乱码（AI 生成的文字经常是无意义的符号）
- [ ] 无非自然闪烁/帧间色彩跳变（AI 特有的帧间不稳定）

**字幕**
- [ ] 字幕出现/消失时间与配音同步
- [ ] 字幕内容与对白一致
- [ ] 无错别字
- [ ] 字幕不遮挡关键画面内容

**叙事**
- [ ] 3 秒钩子有冲击力
- [ ] 对白节奏自然，角色声音区分明显
- [ ] 情绪弧线完整（结构模板落地效果）
- [ ] 结尾悬念有效
- [ ] 场景目的达成（推进剧情 + 揭示角色）

### 9.3 迭代精修（可选）

## 阶段十：批量生产

首集质量确认后，对剩余集重复阶段五~九。

每集之间独立——新集重新生成故事板和视频，不沿用上一集素材（角色设定图、character-bible 和 Moodboard 除外）。

### 批处理规则

- 逐一生成每集的剧本和故事板
- 不要同时提交多个 Seedance 任务
- 每集完整体验确认后进入下一集
- 可选：使用 TaskFlow 将每集拆为独立任务并行管理

### 10.2 跨集一致性审计（每 5 集一次）

> ⚠️ 20 集下来，AI 的渐进式漂移会让最后一集的角色长得不像第一集、声音不像同一个人。必须定期审计。

每完成 5 集后，暂停批量生产，执行审计。

#### 10.2.1 视觉审计

```bash
# 取第 1 集和第 N 集的角色面部特写帧（放到一起方便对比）
cp ./storyboard/ep01/seg1/*特写*.png ./consistency-check/faces/
cp ./storyboard/ep0N/seg1/*特写*.png ./consistency-check/faces/
```

审计清单：
- [ ] 取第 1 集和第 N 集的同角色面部特写帧，目测对比：眼型、发色、脸型是否一致
- [ ] 取同类型场景（如日景对话）的第 1 集和第 N 集帧，对比色调是否漂移
- [ ] 核对配色方案（color-palette.md）的语义标记是否在各集 prompt 中持续使用

**修复**（发现漂移时）：
- 将第 1 集角色设定图重新挂入后续 `--ref-images` 优先级首位
- 在 prompt 中增加 "严格按照角色设定图的外观特征生成，不引入风格漂移"

视觉审计帧存入 `consistency-check/faces/` 目录。

#### 10.2.2 声音审计

> 🎤 Seedance 原生配音在逐集生成中可能出现音色漂移——同一个角色第 10 集的声音可能和第 1 集不同。

核心逻辑：第 1 集的角色对白为金标准，每 5 集提取同角色对白，人工对比听。

**Step 1 — 建立金标准（第 1 集成片通过后）**

从第 1 集成片中按剧本时间戳截取每个角色的代表性对白片段：

```bash
# 每个角色取 2-3 句（不同情绪各一句），存入 audio/golden/ 永久保存
ffmpeg -i ./output/ep01-final.mp4 \
  -ss {对白起始秒} -t {对白时长} \
  -vn -acodec copy \
  ./audio/golden/{角色名}_ep01_{情绪}.mp3
```

**Step 2 — 定期审计（每 5 集）**

从第 N 集成片中截取同角色对白（尽量找同情绪场景，或同级情绪强度）：

```bash
ffmpeg -i ./output/ep0N-final.mp4 \
  -ss {对白起始秒} -t {对白时长} \
  -vn -acodec copy \
  ./consistency-check/audio/{角色名}_ep0N.mp3
```

审计帧存入 `consistency-check/audio/` 目录。

**Step 3 — 人工对比听**

| 检查项 | 判断标准 |
|---|---|
| 音色一致性 | 闭上眼睛，ep01 和 ep0N 的同角色像同一个人说的吗？ |
| 角色区分度 | A 和 B 的声音还明显不同吗？还是趋同了？ |
| 情感表现力 | 同样标记为 angry/悲伤/温柔，ep0N 还和 ep01 一样有张力吗？ |
| 音量/音质 | 有没有整体变轻、变闷、变尖、变糊？ |

> 💡 取至少 3 句不同情绪对比。一句听不出，两句可能巧合，三句基本能判断。

**Step 4 — 修复**

| 漂移程度 | 修复方案 |
|---|---|
| 轻微漂移 | 重新将 ep01 的 `voice-ref.mp3` 挂入后续集 `--audio`，优先级最高 |
| voice-ref 本身质量不够 | 用 TTS 精调参数重新生成 voice-ref，修改 emotion/speed/pitch 锁定更稳定的音色特征 |
| 严重漂移、修复无效 | 切换到 B 方案：TTS 逐句合成全部对白，FFmpeg 替换后续集成片音频轨 |

审计通过后方可继续后续 5 集。
