## 一、风格板 Prompt（Seedream 概念图）

### 模板

```
{画风}风格概念插画。
{场景类型}，{光线描述}，{色调}，{氛围}。
空镜/含人物均可，侧重视觉风格呈现。
高品质，9:16竖版构图。
```

### 建议覆盖以下类型

| 类型 | 示例 prompt | 灯光要点 |
|------|------------|---------|
| 白天外景 | `日系动漫风格概念插画。现代都市街道，午后阳光透过树叶洒落地面，暖黄色调，温暖日常氛围。` | 顶光/侧光，光比 2:1~3:1，暖色温 |
| 夜间内景 | `日系动漫风格概念插画。昏暗的室内房间，窗外月光照入，冷蓝色调，神秘安静氛围。` | 月光主光源（冷），可能有暖色人工光点缀 |
| 逆光/剪影 | `日系动漫风格概念插画。角色站在天台边缘，背对落日，身体呈黑色剪影，轮廓被金色光线勾勒，天空橙红渐变。` | 主体曝光不足，背景高亮，高反差 |
| 人物特写 | `日系动漫风格概念插画。女性角色面部特写，侧光打在脸上，眼神深邃，暖橙色与深蓝对比，戏剧性氛围。` | 柔光为主，眼神光必须存在 |

> ⚠️ **人物特写仅适用于阶段 04（角色设计）之后**。Moodboard 阶段角色尚未确认，不应生成具体角色面部。如需测试人物光影风格，使用模糊剪影或背影替代。
| 室内混合光源 | `日系动漫风格概念插画。咖啡厅靠窗座位，窗外阴天漫射冷光，桌面一盏暖黄台灯补光，冷暖和混合，安静氛围。` | 冷暖色温共存，注意光比平衡 |
| 大场景 | `日系动漫风格概念插画。俯瞰城市夜景，万家灯火，深紫与金色交织，辽阔壮观氛围。` | 无单一主光，靠整体光照层次 |
| 黄金时刻 | `日系动漫风格概念插画。日落前30分钟，整个场景被金色暖光包裹，所有物体的影子被拉长，天空从橙到紫柔和过渡，浪漫氛围。` | 低角度暖光，长阴影，色温 3500K |
| 蓝调时刻 | `日系动漫风格概念插画。日落后20分钟，城市华灯初上，天空呈深邃的蓝色，地面灯光点点，冷暖强烈对比，宁静感。` | 天空冷蓝为主，地面暖光点缀，色温 9000K+ |
| 室内日景（窗光） | `日系动漫风格概念插画。明亮的客厅，左侧大面积落地窗，自然光从窗外倾泻而入，地板上投射出窗框的几何阴影，暖白色调，通透清新氛围。` | 单侧窗光为主光源，光比 2:1~3:1，方向性强 |
| 雨景/水景 | `日系动漫风格概念插画。雨中街道，地面湿漉漉反射霓虹灯光，雨滴在灯光中闪烁，冷暖交织，水面上倒影摇曳，忧郁而唯美。` | 多次反射/散射光源，高光点密集，色彩饱和度被雨水提升 |

## 二、角色设计（编剧层面）

> 在生成画面 prompt 之前，必须为每个角色建立内在画像。
> 四维（Want/Need/Flaw/Ghost）是角色引擎，语言指纹是写台词的基础。

### 2.1 角色四维模板

```
{角色名}，{性别}，{年龄}

Want（表面欲望）：{他 conscious 想要什么}——这是他 pursuit 的目标
Need（深层需求）：{他 unconscious 真正需要什么}——关键：Want ≠ Need
Flaw（核心缺陷/错误信念）：{什么信念在阻碍他得到 Need}
  - 例："表达脆弱就是软弱"
  - 例："只有钱能保护我不受伤害"
  - 例："我不配被爱"
Ghost（创伤事件）：{什么过去经历造成了这个 Flaw}
  - 例：12岁时目睹父亲为保护自己而死，从此认定依赖他人会导致失去

标志性动作/物品：{他独有的行为特征}
  - 例：紧张时咬嘴唇、永远戴着亡母的戒指、说话时回避眼神接触
```

### 2.2 语言指纹模板

```
{角色名} 的说话风格：
- 节奏：{快速/缓慢/时快时慢/有停顿时}
- 句长：{短句/长句/混合/命令式短句}
- 态度：{直接/迂回/讽刺/自我贬低/居高临下/讨好}
- 词汇特征：{文雅/粗俗/专业术语/网络用语/老派}
- 口头禅或习惯句式：{...}
- 对白时的小动作：{说话时摸耳朵/习惯性看地面/手不停比划}
```

### 2.3 角色四维示例

```
林小夜，男，22岁

Want：找到杀死师傅的凶手，复仇
Need：放下仇恨，学会重新信任他人
Flaw：坚信"依赖他人只会带来失去"，因此孤绝、不信任任何人
Ghost：12岁孤儿时被师傅收养，两年后目睹师傅为保护自己被杀。他认定是自己的软弱导致了师傅的死。

标志性动作：思考时右手拇指摩挲左手腕上的旧皮绳——师傅留下的唯一遗物

语言指纹：
- 节奏：缓慢，有长停顿
- 句长：极短，通常不超过10个字
- 态度：冷淡、回避，极少直视对方
- 词汇特征：简洁直接，不废话
- 口头禅："不用。"（拒绝他人帮助时）
```

### 2.4 角色设计 Prompt（Seedream 组图）

> 以下用于生成视觉设计图。内在画像在 `scripts/character-bible.md` 中。

```
{画风}风格角色设定图，{角色名}，{性别}，约{年龄}岁。
外貌特征：{脸型、发型、发色、眼睛颜色和形状、肤色、身高体型}。
服装设计：{上衣、下装、鞋子、配饰的详细描述}。
气质：{冷酷/温柔/活泼/神秘等}。
一套4张角色转面图，分别为正面全身站立、背面全身站立、左侧面全身站立、右侧面全身站立。
角色保持相同姿势和服装，仅旋转角度。
纯色中灰背景（#808080，约 18% 灰，如相机标准灰卡），专业角色设计参考图，线条干净利落，9:16竖版构图。

> ⚠️ **灯光一致性**：4 面图使用均匀柔光照明，无明显方向性阴影。
> 避免单侧硬光，确保正面/背面/左侧/右侧切换时角色视觉特征一致。
```

### 2.5 表情变体图（Expression Sheet）

> ⚠️ **角色设计不只要解决"长什么样"，还要解决"在不同情绪下长什么样"。**
> AI 生视频时，如果参考图只有中性表情，情绪剧烈变化时面部一致性会崩。这是专业动画中的标准流程——表情表（Expression Sheet）。

在 4 面图生成后，为每个重要角色额外生成一组表情变体：

**Prompt 模板：**
```
{画风}风格角色表情变体图，{角色名}，{性别}，约{年龄}岁。
外貌特征与设定图完全一致：{脸型、发型、发色、眼睛颜色和形状}。
一套4张表情变体，分别为：①愤怒——{具体表现：咬牙切齿/怒目圆睁}，②悲伤——{具体表现：眼眶泛红/嘴唇微颤}，③微笑——{具体表现：嘴角上扬/眼神柔和}，④惊恐——{具体表现：瞳孔放大/嘴巴微张}。
每张均为正面半身构图，保持与设定图一致的面部特征和服装上半身。
纯色中灰背景（#808080，如相机标准灰卡），均匀柔光照明。
高品质角色设计参考图，9:16竖版构图。
```

**生成命令：**
```bash
python3 <video-gen>/scripts/seedream.py create \
  -p "<表情变体prompt>" \
  -i ./human-design/{角色名}/front.png \
  --size 9:16 \
  --seq auto --max-images 4 \
  -d ./human-design/{角色名}/expressions/
```

**文件整理：**
```
human-design/{角色名}/expressions/
├── angry.png
├── sad.png
├── smile.png
└── shocked.png
```

在后续 Seedance 视频生成中，根据当前段的情绪方向，将对应的表情变体图加入 `--ref-images`，提升情绪剧烈变化时的面部一致性。

### 2.6 示例

```
日系动漫风格角色设定图，林小夜，男，约22岁。
外貌特征：瓜子脸，黑色短发微翘，深棕色丹凤眼眼尾上挑，肤色偏白，身材修长清瘦。
服装设计：黑色高领无袖紧身衣，外罩深蓝色半透明长衫，腰间系银色细链，黑色束脚长裤，黑色短靴。
气质：冷峻，带一丝忧郁。
一套4张角色转面图，分别为正面全身站立、背面全身站立、左侧面全身站立、右侧面全身站立。
角色保持相同姿势和服装，仅旋转角度。
纯色中灰背景（#808080，约 18% 灰，如相机标准灰卡），专业角色设计参考图，线条干净利落，9:16竖版构图。

> ⚠️ **灯光一致性**：4 面图使用均匀柔光照明，无明显方向性阴影。
```

## 三、故事板关键帧 Prompt（Seedream 图生图）

### 核心原则

每帧必须标注：**景别 + 构图 + 光影 + 角色位置 + 情绪**。图生图时挂角色设定图。

### 光影描述指南

> ⚠️ **核心原则**：每帧的 `{光线}`/`{光影}` 不能留空，必须包含以下 4 项：
> **光源方向 + 光比 + 质地 + 色温**

**光源方向**：明确主光从哪里来
- `从右侧45°打入的窗光` | `顶部天窗洒落的柔光` | `逆光，角色被金色轮廓光勾勒`

**光比**：亮面与暗面的亮度对比
- `光比约 2:1`（柔和，日常场景）| `光比约 4:1`（戏剧性，冲突场景）| `光比约 8:1`（悬念/恐怖，暗部几乎无细节）

**质地**：光线的软硬
- `柔光`（阴天漫射/柔光箱，阴影过渡柔和）| `硬光`（直射阳光/裸灯，阴影边缘锐利）| `漫射光`（无明确方向的散射光，无阴影）

**色温**：冷暖倾向
- `暖光 3200K`（钨丝灯/日落）| `中性 5600K`（正午日光）| `冷光 8000K+`（月光/阴天）

**示例**（填充后的光影描述，语义描述优先 + 数值辅助）：
- 定场：`黄昏逆光从左后方打入仓库，角色暗面几乎不可见细节的戏剧性强反差（约 8:1 光比效果），硬光带锐利阴影`
- 中景：`右侧45°柔光补充角色面部，明暗过渡柔和如阴天窗光（约 2:1 光比），暖黄色温如日落钨丝灯`
- 特写：`左前方蝴蝶光打在角色面部，鼻下形成小蝴蝶影，柔光如柔光箱漫射，明暗差异微妙（约 2:1 光比）`

> ⚠️ **AI 执行注意事项**：当前文生图/视频模型对精确数值（如"光比 4:1"、"色温 3200K"）的理解是模糊的。它更多靠语义理解（"戏剧性强反差"、"暖金色如日落"）而非数值执行。因此所有光影描述应采用 **语义描述优先 + 数值辅助** 的写法。AI 可控性排序：
> - ✅ 高可控：色温方向（暖/冷）、光质（硬/柔）、主光方位（侧光/逆光/顶光）
> - ⚠️ 中可控：反差强度（强反差/柔反差）、氛围关键词（黑色电影感/日系治愈）
> - ❌ 低可控：精确光比数值、精确色温 Kelvin 值、精确曝光参数

### 按镜头类型分类

**定场镜头（远景/全景）**
```
{画风}风格，{场景}全景定场镜头。
{环境细节}，{光源方向+色温+光影描述}，{色调氛围}。
镜头使用约{24-50mm}标准广角，深景深，画面层次清晰。
构图开阔，建立空间感，9:16竖版。
高品质，精致细节，与风格板保持一致。
```

**中景（角色引入）**
```
{画风}风格，角色参考提供的设定图。
{角色名}全身站立于{场景}中，{姿态}，{表情}。
中景构图，角色位于画面中央偏上。
{光源方向+光比+质地}。
镜头使用约{35-50mm}标准焦段，中等景深。
角色与设定图保持一致，9:16竖版。
```

**双人中景/过肩镜头（对话互动）**
> 优先使用过肩镜头（OTS）：前景角色肩部虚化，聚焦于对面角色的面部反应，更有代入感。

```
{画风}风格，角色参考提供的设定图。
{角色A}在画面左侧，面向右侧；{角色B}在画面右侧，面向左侧。
两人正在{对话/互动}，{各自表情}。
场景：{环境}，{光源方向+光比+质地}，{分离光描述——轮廓光/色彩分离}。
中近景双人构图（或过肩镜头），180度轴线内拍摄，镜头使用约{50-85mm}中焦。
9:16竖版。角色与设定图保持一致。
```

**特写（表情/动作）**
```
{画风}风格，角色参考提供的设定图。
{角色名}面部特写，{具体表情}，{眼神方向}。
{光源方向+质地}打在脸上（如：右侧45°柔光打在面部，形成柔和的面部层次），眼神光（catchlight）可见。
镜头使用约{85-135mm}中长焦，浅景深，背景完全虚化。
大特写构图，角色与设定图保持一致，9:16竖版。
```

**俯拍/仰拍（特殊视角）**
```
{画风}风格，角色参考提供的设定图。
{仰拍/俯拍}{角色名}，{动作}，{表情}。
场景：{环境}，{光源方向+光比+质地}。
镜头使用约{24-35mm}广角（强化透视），{低角度仰拍/高角度俯拍}构图，强化{压迫感/渺小感/其他情绪}。
9:16竖版。角色与设定图保持一致。
```

## 三·五、镜头光学指南

> ⚠️ 故事板每帧和视频 prompt 必须标注镜头焦距。焦距决定的不只是\"看到多少\"——它改变空间关系和情感距离。
>
> ⚠️ **AI 语义执行说明**：以下焦距数值为全画幅相机等效语义标记。Seedance/Seedream 不执行光学计算，而是通过关联训练数据中的视觉模式来响应。始终用语义描述（\"空间压缩，背景虚化\"）配合数值使用，不要单独依赖数值。

### 焦距系统

| 焦距 | 视觉效果 | 情感语义 | 漫剧适用场景 |
|------|---------|---------|------------|
| 超广角 ≤18mm | 透视极度夸张，面部拉长变形，空间深度被放大 | 不安、压迫、心理扭曲、濒临崩溃 | 角色内心崩塌、噩梦、主观恐惧镜头 |
| 广角 24-28mm | 明显透视，空间感强，近距离面部略微变形 | 孤立感、被环境压迫、脆弱 | 角色被环境吞噬、弱势处境、狭窄空间 |
| 标准 35-50mm | 接近人眼透视，自然舒适 | 客观、真实、日常、亲近 | 大部分日常对话、中景双人镜头 |
| 中长焦 85mm | 空间压缩明显，面部扁平柔和，背景虚化强 | 浪漫、亲密、窥视感 | 情感特写、恋爱凝视、关键表情 |
| 长焦 ≥135mm | 空间极度压缩，前景与背景\"贴\"在一起 | 无法逃脱、命运纠缠、被监视 | 宿命感、跟踪视角、终极对峙 |

### 焦距与景别的搭配禁忌

| 景别 | 推荐焦距 | 禁忌 | 原因 |
|------|---------|------|------|
| 远景/全景 | 24-50mm | 长焦≥85mm | 长焦压缩会失去空间层次，建筑\"纸片化\" |
| 中景 | 35-85mm | 超广角≤18mm | 广角畸变让角色比例失调 |
| 中近景 | 50-85mm | 广角≤28mm | 面部会被拉长，鼻子变大 |
| 特写 | 85-135mm | 广角≤35mm | 广角怼脸 = 哈哈镜效果（除非刻意追求不安感） |
| 极端特写（眼睛/嘴唇） | ≥100mm | 广角≤50mm | 距离太近产生透视灾难 |

### Seedance Prompt 中的焦距表达

```
# ✅ 正确
镜头使用约85mm中长焦，浅景深，背景柔和虚化。
镜头使用约35mm标准焦段，深景深，前后景皆清晰。
超广角镜头，透视夸张，角色被环境压迫。

# ❌ 模糊
虚化背景。         （不写焦距 = AI 随机给）
```

### 景深（DOF）控制

| 景深类型 | 视觉效果 | 叙事功能 | 适用场景 |
|----------|---------|---------|---------|
| 深焦 Deep Focus | 前后景皆清晰，观众可自主选择看什么 | 客观冷静、角色与环境关系被强调 | 定场镜头、群像、空间揭示 |
| 中等景深 | 主体清晰，背景可辨识但不出挑 | 日常叙事，注意力在角色但环境仍存在 | 大部分对话 |
| 浅景深 Shallow | 仅主体清晰，其余虚化 | 主观沉浸、情绪隔离、强制聚焦 | 情感特写、关键台词、孤立感 |
| 移焦 Rack Focus | 焦点从 A 平滑移到 B | 注意力转移、权力关系变化、揭示 | 反应镜头、A 发现 B、权力反转 |

景深 prompt 表达：
```
浅景深，主体锐利，背景完全虚化成色块。
深景深，画面所有层次清晰可见，从前景到远景皆锐利。
镜头焦点从角色A面部平滑转移到角色B面部（移焦/rack focus）。
```

### 分离光（Separation Light）

> ⚠️ 不要只靠景深分离角色——光线分离是更隐蔽、更高级的手段。

| 分离方式 | 描述 | Prompt 表达 |
|----------|------|------------|
| 发型光/轮廓光 | 从后上方打光，在头发和肩部形成高光边缘 | `一道窄光从角色后上方掠过，在头发边缘形成金色轮廓光` |
| 背景亮度梯度 | 角色身后背景不是均匀亮度，而是在角色轮廓处形成明暗对比 | `角色身后的墙壁从左上到右下逐渐变暗，角色恰好站在亮度交界处` |
| 颧骨/下颌 kicker | 从侧后方掠过的窄光，勾勒面部骨骼线 | `从左侧后方一道细光掠过颧骨和下颌线，把面部从暗背景中切出来` |
| 色彩分离 | 角色暖色、背景冷色（或反之），靠色温差分离 | `角色被暖色主光照亮（3200K），背景呈冷蓝色调（8000K+）` |

## 四、视频生成 Prompt（Seedance 图生视频）

### 基础公式（对齐官方指南）

> Seedance 2.0 官方 prompt 指南的核心公式：**主体 + 运动**（必需）+ **环境 + 美学 + 运镜 + 音频**（非必需）。
> 官方示例 prompt 通常只有 50-150 字。模型通过 ref-images 获取视觉信息，文字只负责描述动态内容。

```
必需：{谁} + {在做什么}
非必需：{环境/场景} + {运镜} + {对白} + {字幕}
```

**核心原则**：
- **ref-images 承载静态视觉信息**（构图、光影、色调、角色外观、服装），prompt 不重复描述
- **prompt 只写动态信息**（动作时间线、对白、运镜、情绪变化）
- **目标长度 100-200 字**，不超过 250 字
- 不写画风声明（由 ref-images 传递）
- 不写光影技术参数（光比、焦距、色温、景深——由故事板帧传递）
- 不写节奏/口型指令（模型自动处理）

### 漫剧专用模板（精简版）

```
{一句话摘要：谁在做什么}。
画面中{N}个人{：人物列表和位置，如有需要}。

{动作时间线——按时间顺序，一个动作一个主语}。
场景参考`图片1`的构图和光影。
{角色A}（{拼音}）形象参考`图片3`，表情参考`图片5`。
{角色B}（{拼音}）形象参考`图片4`，表情参考`图片6`。

{角色A}说道："{台词A}"
{角色B}说道："{台词B}"

{运镜描述——用连续运镜而非"镜头切到"}。

💭 {如有内心独白}：
内心独白作为画外音出现——{语气描述}：
"{独白内容}"

画面底部出现字幕，字幕内容与对话一致。
```

**旧版（~380 字）**：

**砍掉了什么**：画风声明、焦距mm、光比数值、色温、景深类型、分离光/轮廓光描述、节奏/口型指令、婚纱面料细节（由 ref-images 承载）。
**新增了什么**：人数锁定、运镜替代切镜（`镜头缓缓推向` 替代 `镜头切到`）、镜子否定约束（`两个人的完整倒影`）。

### 图片与音频参考的内联指代（对齐官方指南）

> Seedance 2.0 官方 prompt 指南使用**内联指代**方式引用参考素材——在 prompt 正文中直接用 `图片N` 指代，不需要在末尾附加独立的映射声明段落。
> 音频参考无需在 prompt 中显式声明，模型会自动基于文本对白 + 参考音频生成匹配的人声。

**图片指代规则**：
- 按 `--ref-images` 的上传顺序编号：`图片1`、`图片2`……`图片N`
- 在 prompt 正文中**自然嵌入**每张图片的用途，明确指代参考什么
- 角色名首次出现标注拼音

**常用指代句式**（参考官方示例）：
```
场景参考`图片1`的构图和光影。
{角色名}形象参考`图片3`中的{角色名}（{拼音}）。
表情参考`图片5`。
构图参考`图片2`。
穿着`图片1`中的{服装}。
```

**音频规则**：
- 音频通过 `--audio` 参数传入，模型自动匹配人声
- 对白用**双引号**包裹，优化音频生成效果
- prompt 中无需显式声明音频用途

**ref-images 上传顺序约定**（与编号对应）：
1. 定场帧（场景空间+光影）
2. 情绪高点帧（构图+面部光照）
3. 角色A 设定图正面
4. 角色B 设定图正面（如有）
5. 角色A 表情变体（如有）
6. 角色B 表情变体（如有）
7-9. 补充参考图（尾帧/服装/道具等）

### ref-images 数量策略：替换优先于增加

> ⚠️ **ref-images 不是越多越好。** 5-6 张是当前最佳平衡点，超过 7 张风险上升。

**为什么不宜过多**：
- 参考图之间存在**竞争关系**——角色设定图是均匀柔光，故事板帧是场景光，两者光照矛盾。图越多，模型越倾向折中处理，导致场景光被削弱、角色面部光影偏平
- **注意力稀释**——6 张时每张约 16% 权重，9 张时每张约 11%。关键的角色设定图权重被稀释，面部一致性反而可能下降
- **矛盾增加**——每多一张图，就多一个可能与其他图冲突的视觉约束

**数量与效果**：

| 数量 | 效果 |
|------|------|
| 3-4 张 | 角色外观约束不够，面部容易漂移 |
| 5-6 张 | 最佳平衡——角色+场景+情绪都有锚定 |
| 7 张 | 可接受，但新增的那张必须解决具体问题 |
| 8-9 张 | 风险区——参考图矛盾增加，模型折中概率上升 |

**核心原则：不加张数，根据本段关键难点替换最不重要的那张。**

**替换决策表**：

| 本段关键难点 | 替换操作 | 被替换的图 | 理由 |
|------------|---------|-----------|------|
| 手部关键动作（牵手/交换戒指/握拳） | 用**手部姿态参考图**替换一张表情变体 | 表情变体 | 手部比表情更容易出错 |
| 换装（婚纱/礼服/便装切换） | 用**换装服装参考图**替换一张故事板帧 | 定场帧或情绪帧中较不关键的一张 | 换装准确性优先于场景构图 |
| 关键道具特写（戒指/照片/手机） | 用**道具特写参考图**替换一张表情变体 | 表情变体 | 道具准确性优先于表情 |
| 运镜跨度大（远景→特写连续推镜） | **增加** 1 张中间景别帧（总数 7） | 不替换 | 唯一值得加到 7 张的场景 |
| 无特殊难点 | 保持 5-6 张不变 | — | 不浪费配额 |

**替换示例**（EP06 seg3 交换戒指）：

```
# 当前（5张）：
图片1: 故事板帧1（花拱门下两人面对面）
图片2: 故事板帧2（戒指滑上无名指极近特写）
图片3: 苏晚宁正面设定图
图片4: 陆深正面设定图
图片5: 苏晚宁犹豫表情

# 优化（5张，替换不增加）：
图片1: 故事板帧1（花拱门下两人面对面）
图片2: 故事板帧2（戒指滑上无名指极近特写）
图片3: 苏晚宁正面设定图
图片4: 陆深正面设定图
图片5: 手部参考图（男性手握女性手，戒指在无名指上）← 替换表情变体
```

本段核心难点是戒指位置，不是苏晚宁的表情。表情可以靠 prompt 文字描述，手指位置靠文字描述不可靠。

> 内心独白（画外音）是漫剧揭示角色心理的核心手段——**角色嘴不动，观众听到的是她的心声**。
> 不是自言自语（角色嘴在动），不是低声嘟囔——是画外音，双重叙事。

**使用时机**：
- 角色表面行为与内心真实意图相反时——画面展示外部行为，独白暴露真实动机
- 角色独处、做出关键发现时——独白压缩"内心推理"为一句
- 信息不对等被打破时——独白标记角色认知的转折点

**格式要求**：
- Prompt 中用 `💭` 符号开头，明确标注"内心独白作为画外音出现"
- 描述时机："画面停在{谁的什么表情/构图上}"
- 描述语气：声线变化（压低/骤冷/颤抖/平静）——区别于对话语气
- 每段最多一句——独白是钉子不是地板

**Prompt 模板**：
```
💭 内心独白作为画外音出现——{语气描述}：
"{独白内容}"
```

## 五、镜头语法参考

> 指导剧本和故事板阶段的镜头设计。

### 景别序列

在 15 秒内设计 3-4 个景别变化：

| 推荐序列 | 节奏感 | 适用场景 |
|----------|--------|---------|
| 远景→中景→特写 | 渐进聚焦 | 角色登场、场景揭示 |
| 特写→中景→双人中景 | 由点及面 | 悬念开场、对话引入 |
| 中景→特写→远景 | 高潮→释放 | 情绪爆发后退场 |

### 180 度规则

双人对话时，摄影机始终在两人的同一侧。角色 A 在画面左侧看右，角色 B 在画面右侧看左——建立"对视"空间感。越轴需要动机（如角色情绪突变、势力格局反转）。

### 30 度规则

连续两个镜头之间，摄影机角度差必须 ≥30°。角度变化不足会产生"跳切"（jump cut）感，观众会察觉到不自然的跳跃。15 秒快节奏短剧特别容易踩这个坑。

### 视线匹配（Eyeline Match）

正反打时两人视线高度必须一致。A 看着画右的 B 时，A 的视线在画面上 1/3 的水平线上；切到 B 时，B 的视线必须在同一高度看画左。视线高度不匹配是正反打最明显的业余标志。

### 景别跳跃规则

相邻镜头的景别差距超过一级（如远景直接跳特写）是刻意为之的艺术选择，不是衔接错误。使用跳跃前必须有叙事动机——制造冲击力、强调孤立感、或打断观众预期。无动机的跳跃 = 视觉车祸。

### 运镜术语

| 术语 | 英文 | 效果 | 适用 |
|------|------|------|------|
| 缓缓推进 | Dolly In | 逐渐聚焦，情绪升温 | 重要台词、角色反应 |
| 缓缓拉远 | Dolly Out | 孤独感、结束感 | 集末悬念、退场 |
| 横向摇镜 | Pan | 展示空间，视线引导 | 场景建立、转移注意 |
| 纵向摇镜 | Tilt | 垂直探索，揭示高度 | 建筑、角色从脚到头 |
| 横移 | Truck/Track | 跟随角色横向移动 | 行走跟拍、空间过渡 |
| 升降 | Crane/Jib | 揭示空间关系，情绪升华 | 结局揭晓、大场面收尾 |
| 变焦点 | Rack Focus | 注意力引导，A→B不靠切镜头 | 对话反应、细节发现 |
| 斜角 | Dutch Angle | 表达不安、混乱、失衡 | 角色内心崩塌、梦境 |
| 手持摄影 | Handheld | 紧张、临场感、代入角色视角 | 冲突追逐、主观镜头 |

### 构图提示

- **视线留白**：角色看向的方向留更多空间
- **头部空间**：特写头顶到画框上沿保持合理距离——过紧 = 压抑，过松 = 漂浮
  - 中近景：头顶离上边框约 5-8% 画面高度
  - 特写：可切到发际线以上，更紧凑
- **三分法**：角色眼睛放在画面上 1/3 处
- **负空间**：角色视线前方留白不足 = 被困住的感觉。控制负空间大小就是在控制角色的"余地"
- **封闭 vs 开放构图**：
  - 封闭构图：角色被画框/门框/窗框锁住 → 被困住、无出路
  - 开放构图：角色身体部分出画、视线突破画框 → 自由、延伸、突破
- **前景层次**：在前景放置虚化的物体（窗帘、栏杆、另一个角色的肩膀）→ 增加空间深度和窥视感
- **引导线**：场景中的线条（走廊、栏杆、光束）指向角色 → 强制观众视线聚焦
- **取景中的取景**（Frame within a frame）：通过门框、窗户、镜子看角色 → 窥视感、隔离感、身份分裂（镜子）
- **高角度/低角度系统**：
  - 高角度俯拍 → 观众居高临下，角色弱势、渺小、被审判
  - 低角度仰拍 → 角色强势、压迫感、英雄化
  - 平视 → 平等、客观、日常
  - 极俯（鸟瞰）→ 宿命感、角色被命运摆布
  - 极仰（虫视）→ 压倒性的力量、神化

### 色彩对比系统

> 色温不只是"暖"和"冷"——色彩关系是无声的情绪语言。
>
> ⚠️ **画风兼容性说明**：以下色彩理论基于写实光影逻辑（适用于写实、韩漫、国漫、插画风格）。日漫/平面风格的光影规律完全不同——阴影是分层平涂而非渐变，色彩对比靠色块而非光比。使用日漫风格时，将"光比"理解为"色块明暗层级差"，将"色温过渡"理解为"色块色相偏移"。

| 色彩策略 | 配置 | 视觉效果 | 情绪语义 |
|----------|------|---------|---------|
| 互补色对比 | 橙（肤色）+ 青（背景） | 人物自然分离，最经典的人像方案 | 经典、平衡、专业 |
| 冷暖共存（暖主冷境） | 暖主光打在角色上 + 冷色环境 | 温暖中有疏离感 | 孤独的温暖、温存中的不安 |
| 冷暖共存（冷主暖境） | 冷光打在角色上 + 暖色环境 | 冷漠中有人情涌动 | 角色与世界的隔阂 |
| 单色渐变 | 全暖或全冷，仅靠亮度分层 | 高度风格化 | 梦境、闪回、情绪极端 |
| 色温过渡 | 一场戏内色温从 A 渐变到 B | 情绪转变可视化 | 温暖→冰冷=失去，冰冷→温暖=救赎 |
| 互补色冲突 | 红 vs 绿、紫 vs 黄 | 高度戏剧性 | 对抗、二元对立 |

Prompt 中的色彩表达：
```
# 互补色
角色被暖橙色主光照亮（3200K），背景呈冷青色（8000K+），形成经典橙青对比分离。

# 色温过渡
开场暖黄卧室光，随着角色得知噩耗，色温逐渐从 3200K 过渡到 8000K，最终画面完全被冷蓝色统治。

# 单色渐变  
整个画面仅用不同亮度的金色构成，无冷色，如旧照片般的暖调梦境。
```

### 竖屏构图要点（9:16）

> ⚠️ 以上构图理论基于横屏通用语境。本 skill 全部输出为 9:16 竖屏，以下为竖屏专项指导。

| 要点 | 规则 | 细节 |
|------|------|------|
| **角色面部高度** | 中近景/特写中，角色眼睛放在画面约 40% 高度处 | 横屏的上 1/3 ≈ 33%，但 9:16 画面更高，40% 能避免角色被压到下半部 |
| **纵向负空间** | 上方留白 = 仰望/渺小；下方留白 = 悬浮/失衡 | 9:16 的纵向空间是最大构图资源，慎重分配 |
| **双人对话** | 上下排列优先于左右排列 | 一人在前/上方、另一人在后/下方，比左右挤压更自然 |
| **定场镜头** | 低角度仰拍比平拍更有效 | 利用纵向空间展示建筑高度或天空层次，平拍会浪费上半画面 |
| **视线方向** | 向上看天然适配（大量上方空间），向下看需叙事动机 | 向下看压缩感强，角色视线被压在下边框 = 被压制/无助 |
| **头部空间** | 中近景头顶离上边框约 5-8% 画面高度 | 9:16 中更易出现"头上太空"的问题，比横屏收得更紧 |
| **景深在竖屏中** | 前景→中景→背景的纵向分层是 9:16 的核心深度来源 | 缺少纵向分层 = 画面扁平、空洞 |

### 🛡️ 细节防错规范

> ⚠️ 以下规则专门针对 Seedance 2.0 的已知弱项（手指、物体位置、人数、镜面反射等），降低细节性错误的概率。

**规则 1：一个动作一个主语，禁止复合句**
- [ ] 每个动作独立成句，主语明确。禁止在一句话中切换主语
- 例：`陆深伸出左手握住苏晚宁的右手。` + `苏晚宁低头看着戒指。`（两句）
- 反例：`陆深握住她的手，她低头看戒指，他的拇指停在她指节上。`（一句三个主语切换）

**规则 2：关键细节用否定约束加固**
- [ ] 涉及手指、戒指位置、物品位置等精细细节时，同时写正面描述和否定约束
- 例：`戒指滑上苏晚宁左手的无名指（第四根手指，不是中指，不是食指）。`
- 例：`画面中只有两个人，不出现第三个人。`
- 适用场景：手指辨认、物品位置、人数、镜面反射

**规则 3：每个时刻最多 2 个并行动作**
- [ ] 15 秒内的动作按时间线排列，每个时刻只描述 1-2 个动作。超过 2 个并行动作模型会丢掉一部分
- 例：先 `陆深转身面向苏晚宁。` 然后 `他的眼神从空白变成柔软，嘴唇微微张开。` 然后 `右手从裤袋里抽出来。`
- 反例：`陆深转身的同时右手从口袋抽出来，手指蜷缩，嘴唇微张，眼神从空白变柔软，背景灯串闪烁。`

**规则 4：静态细节交给参考图，prompt 只写动态**
- [ ] 服装面料、场景布局、角色外观等静态信息已由 ref-images 承载，prompt 中不重复描述
- [ ] prompt 只保留：对白、动作时序、情绪变化、运镜、光影对比
- 参考图越精准，prompt 越短，模型犯错空间越小

**规则 5：人数和位置在 prompt 开头锁定**
- [ ] prompt 第二句（一句话摘要之后）明确声明画面中的人数和左右位置
- 例：`画面中始终只有两个人：苏晚宁（女，画面左侧）和陆深（男，画面右侧），位置不变。`
- 涉及镜子/反射的场景额外声明：`全身镜中映出两个人的完整倒影。`

**规则 6：同一段内优先用连续运镜，减少切镜**
- [ ] 同一空间内的动作序列用连续运镜（推/拉/摇/移/跟）而非"镜头切到"
- [ ] 每次"镜头切到"都是一个出错点——模型需要在连续视频中生成两个独立画面，角色可能变脸、手指可能异常、光影可能跳变
- [ ] 连续运镜让模型只维护一个画面流，帧间连续性更好，细节错误概率更低
- 改写示例：
  - ❌ `镜头切到陆深面部特写。陆深的眼神从空白变成柔软。`
  - ✅ `镜头从苏晚宁缓缓推向陆深面部，景别从中景过渡到面部特写。陆深的眼神从空白变成柔软。`
- 适用场景与例外见下方「运镜优先规则」

### 🎬 运镜优先规则

> ⚠️ 本规则针对**段内**镜头设计（一段 15 秒内的画面组织），不影响段间转场。
> 核心原则：**能用运镜连接的画面不用切镜，必须切镜的场景才切镜**。

#### 运镜 vs 切镜选择表

| 场景类型 | 推荐方式 | 理由 |
|---------|---------|------|
| 同一空间内的动作序列 | ✅ 连续运镜 | 减少出错点，画面流畅 |
| 单人动作 + 环境展示 | ✅ 连续运镜（推/拉/跟） | 最适合运镜的场景 |
| 从全景到特写的景别变化 | ✅ 连续推镜 | 一个推镜完成，不需要切 |
| 双人对话（你来我往） | ⚠️ 保留切镜（正反打） | 对话节奏需要切镜，但可用摇镜替代部分正反打 |
| 情绪高潮的反应镜头 | ⚠️ 保留切镜（特写跳切） | 冲击力需要景别跳跃 |
| 不同空间 | ❌ 必须切镜 | 运镜无法跨空间 |
| 时间跳跃 | ❌ 必须切镜 | 运镜暗示连续时间 |

#### 常用运镜替代方案

| 原来的切镜写法 | 运镜替代写法 | 效果 |
|--------------|------------|------|
| `镜头切到角色B特写` | `镜头从角色A缓缓摇向角色B，推至面部特写` | 保持空间连续性 |
| `镜头切到手部特写` | `镜头缓缓下移推进到手部，景别从中景过渡到极近特写` | 引导视线自然聚焦 |
| `镜头切到远景` | `镜头缓缓拉远，从中景退到远景，露出完整空间` | 释放感、退场感 |
| `镜头切到另一个角度` | `镜头绕角色缓缓移动，从正面过渡到侧面` | 避免跳切，视角变化自然 |
| `镜头切到镜中倒影` | `镜头缓缓摇向镜子方向，画面过渡到镜中映像` | 镜面反射更连贯 |

#### Prompt 中的运镜描述模板

```
# 推镜（远→近，情绪升温）
镜头从{起始景别}缓缓推进到{目标景别}，聚焦{目标主体}。

# 拉镜（近→远，释放/退场）
镜头从{角色}面部缓缓拉远，退到{目标景别}，露出{环境元素}。

# 摇镜（A→B，注意力转移）
镜头从{角色A}缓缓摇向{角色B}，{角色B的反应描述}。

# 跟镜（跟随角色移动）
镜头跟随{角色}从{位置A}走向{位置B}，保持{景别}。

# 升降（揭示空间/情绪升华）
镜头从{低位/高位}缓缓{上升/下降}，揭示{空间关系/情绪变化}。
```

#### 在剧本阶段的应用

剧本 §5.2 的镜头序列设计中，优先使用运镜连接：

```markdown
# ❌ 旧写法（多次切镜）
**镜头序列**：手整理花艺的特写 → 镜头拉远露出苏晚宁全身 → 
陆深从入口走入的远景 → 两人在花拱门前的中景对视

# ✅ 新写法（标注运镜/切镜）
**镜头序列**：手整理花艺的特写 →[拉远] 露出苏晚宁全身中景 →
[摇镜] 陆深从画面右侧走入 →[推镜] 两人花拱门前中景对视
```

用 `→[运镜类型]` 标注连接方式。没有标注的 `→` 默认为切镜。
这样在 prompt 组装阶段可以直接按标注选择运镜描述或切镜描述。

### ❌ 常见错误

- [ ] 没有在 prompt 中用 `图片N` 指代参考图用途（模型会自己猜，不可控）
- [ ] 图片编号与 `--ref-images` 实际顺序不一致
- [ ] 运镜描述不具体（"镜头运动" → 具体是推/拉/摇/移/跟/固定？）
- [ ] 全程匀速推镜无呼吸（中间必须有一次停顿）
- [ ] 分离光缺失（暗色角色站在暗背景前 = 消失）
- [ ] 两角色外观/光照雷同（穿深色的宋砚和穿白色的顾鸢应该用不同的轮廓光）
- [ ] 开头直接写镜头参数（少了"一句话摘要"，模型对整体意图理解打折）
- [ ] 生僻/多音角色名未注音（宋砚→yàn 单音字不会读错，顾鸢→yuān，温庭轩→tíng xuān）

## 八、图片 Prompt 审查清单（Seedream）

> ⚠️ 提交 Seedream（图片生成）前，必须按以下维度逐项审查 prompt。
> 适用于：角色设计（§二）、风格板（§一）、故事板关键帧（§三）。

### 审查流程

1. **组装 prompt** → 2. **对照本清单自查** → 3. **发给用户审核** → 4. **确认后提交**

---

### 📋 通用规则（官方指南 §通用规则）

- [ ] 用自然语言清晰描述画面——主体 + 行为 + 环境，连贯自然
- [ ] 避免关键词堆砌（"一个女孩，撑伞，林荫街道" → "一个穿着华丽服装的女孩，撑着遮阳伞走在林荫道上"）
- [ ] 文字内容用双引号包裹（如海报标题"Seedream 4.5"）
- [ ] 风格/色彩/光影/构图用自然语言补充，不堆华丽词

### 🎨 图生图（官方指南 §图生图）

- [ ] 如果是要编辑已有图：明确了编辑操作（增加/删除/替换/修改）
- [ ] 如果是要保持部分不变：prompt 中强调"保持{XX}不变"
- [ ] 参考图通过 `-i` 参数传入，prompt 中用"参考图中的{XX}"指代
- [ ] 如果参考多个角色设定图：明确"图一的人物形象""图二的服装风格"

### 🔄 组图输出（官方指南 §多图输出）

- [ ] 如果生成组图（四视图/表情变体）：用了触发词"一套4张""一组表情变体"
- [ ] 组图中每张的区别被明确描述（"分别为正面、背面、左侧、右侧"）
- [ ] 角色跨帧一致性的约束写清楚了（"同一角色，保持相同姿势和服装，仅旋转角度"）

### 🧩 漫剧专项

#### 角色设计（§二）
- [ ] 角色外貌描述完整：脸型、发型、发色、眼型、肤色、身高体型
- [ ] 服装设计描述完整：上衣、下装、鞋子、配饰
- [ ] 气质/表情明确
- [ ] 背景颜色/材质统一（如"纯色中灰背景，约18%灰"）
- [ ] 四视图：灯光一致性约束（"均匀柔光照明，无明显方向性阴影"）
- [ ] 表情变体：每个表情有具体表现（不只是"愤怒"，而是"咬牙切齿、怒目圆睁"）
- [ ] 表情变体挂载了参考图（`-i front.jpg` 保持面部一致性）

#### 风格板（§一）
- [ ] 画风说明明确（"新海诚风格" 而非 "日漫"）
- [ ] 场景类型 + 光线 + 色调 + 氛围 四项覆盖
- [ ] 覆盖了必要的场景类型（白昼外景/夜间内景/人物特写等）

#### 故事板关键帧（§三）
- [ ] 景别标注（远景/中景/近景/特写）
- [ ] 光线完整：光源方向 + 光比 + 质地 + 色温
- [ ] 构图描述（视角/机位/景深）
- [ ] 相邻帧之间景别或视角有明显差异
- [ ] 挂载了角色设定图（`-i` 参数）
- [ ] 如有多角色：每个角色的位置描述

### ❌ Seedream 常见错误

- [ ] 没有用自然语言而是关键词堆砌（官方明确说 5.0 简洁连贯 > 堆砌华丽词）
- [ ] 组图 prompt 只说"多种表情"但没说明每种具体表现
- [ ] 图生图时没说明"保持什么不变"（编辑类图生图的核心）
- [ ] 四视图用了单侧硬光导致各面不一致（应均匀柔光）
- [ ] 多图输入时没区分"图一""图二"（官方要求明确指代）
- [ ] 文字内容没加双引号

## 九、Prompt 精简：视觉信息迁移到参考图

### 核心理念

**不硬砍字数。** 把静态视觉信息从文字迁移到参考图，文字只保留对白、动作时序、光影对比、运镜等必须用文字传达的动态信息。

流程：长 prompt → 分析哪些是静态视觉信息（可迁移）vs 动态时序信息（不可迁移）→ 静态信息由 ref-images 承载，从 prompt 中移除 → 最终 prompt = 动态信息 + 图片引用声明。

### 迁移规则

| # | 规则 |
|---|------|
| 1 | **迁移而非删除**——ref-images 已有的视觉信息从 prompt 中移除，不是"砍掉不要" |
| 2 | 移除后 prompt 必须明确声明依赖："构图参考图片1-N""角色外观参考图片N-N" |
| 3 | Seedream 补充参考图必须遵循官方指南（简洁连贯、图生图时指明不变部分） |
| 4 | 所有产出的参考图写入 assets.md（task_id 可追溯） |
| 5 | 服装/道具参考图用图生图，`-i` 挂角色设定图保持面孔一致 |
| 6 | 补充参考图不全拆——配额只有 9 张，服装+道具+灯光合并出 |
| 7 | **迁移不可破坏时序**——"谁在什么之后做了什么"的因果不能模糊 |
| 8 | **眼神和肢体动作中描述心理情感的语句一律不迁移**——"抬起下巴迎向目光""拇指停在婚戒上""瞳孔反射光点一闪而逝"等，是角色内心世界唯一的文字入口。任何静帧参考图都无法传递"动作+时间+情感"的复合语义 |

### 迁移判断：哪些可以迁移到参考图

可迁移（静态视觉信息——参考图能承载）：
- 场景空间层次和布局 → 故事板定场帧已覆盖
- 角色外观和面孔 → 角色设定图已覆盖
- 服装面料质感和款式 → 出服装细节参考图
- 道具外观细节 → 出道具特写参考图
- 构图和景别序列 → 故事板帧序列已覆盖
- 灯光氛围（光源位置/色温/光比）→ 故事板帧 + 灯光参考图已覆盖

不可迁移（必须文字保留）：
- 对白内容 + 情感标注
- 肢体动作的时序和因果关系
- **眼神/肢体动作中描述角色心理情感的语句**——参考图无法替代动态表情语义
- 运镜指令（推/拉/摇/移/跟/固定）
- 字幕指令
- 内联 `图片N` 指代（在 prompt 正文中自然引用参考图用途）
- **动态视觉意象**（如"光点在瞳孔中一闪而逝"）——需要时间维度才能被理解
- **声音设计**（沉默时刻、环境音提示、呼吸声节奏）

**不可迁移的典型示例**：
- ❌ 迁移"抬起下巴迎向目光" → 丢失主动/被动姿态
- ❌ 迁移"拇指从摩挲到停在婚戒上" → 丢失"从紧张到做了决定"的情绪弧
- ❌ 迁移"瞳孔反射光点一闪而逝" → 静帧参考图无法传递动态过程

### 执行步骤

```
1. 写出完整 prompt（不约束长度，写出所有细节）
2. 对照本清单 + §七/§八 审查清单，标识可迁移项和不可迁移项
3. 如需补充参考图，用 Seedream 生成（遵循 §八 审查清单 + 官方指南）
4. **一次性精简 prompt**：移除可迁移的静态视觉描述，保留不可迁移的动态信息
5. 在精简后的 prompt 中用内联 `图片N` 指代参考图用途（如"场景参考`图片1`中的构图""角色形象参考`图片3`"）
6. 写入 assets.md（完整版 prompt + 可迁移分析 + 补图记录 + 精简版 prompt）
7. 发用户审核完整参数
8. 通过后提交 Seedance
```
