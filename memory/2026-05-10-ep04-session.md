## 4. 经验教训

### 做得好的

1. **规则升级响应快**：用户提出"字数统计"、"时长范围 120-240s"、"分镜详细化"，当场固化到文档和脚本
2. **重做决策正确**：seg-1 v1 场景错误，seg-2 v1 望风不到位，都及时重做没有将就
3. **场景变体图分离设计**：zhongcai-jiao.png 独立生成，A/B 面视觉反差做对
4. **A10 教训实战应用**：seg-3 埋种独白+seg-8 太后内心独白首次使用，让微表情风险降低

### 可改进的

#### 1. 🔴 **剧本场景位置描述需要更精确**（新教训 A12）

**问题**：EP04 剧本 v2【0-5s】写"冷月宫庭院，杂草丛生"，我组装 seg-1 prompt 时也写庭院——但 `lengyuegong/main.png` 是殿内场景。

**根因**：
- 剧本层面写"庭院"（外）与场景库层面的"殿内"（内）不一致
- 我在 prompt 组装时没有核对"场景图实际内容"vs"剧本描述"

**改进**：
- 阶段 05 剧本校验：**每场戏标注精确空间位置**（殿内/庭院/廊下/主径/偏僻角落等），并与 03-scenes/ 下现有场景图的 assets.md 描述交叉核对
- 阶段 07 prompt 组装前：**强制核对场景图实际内容**——打开对应 assets.md 看"场景描述"字段，确认环境元素

#### 2. 🟡 **配角动作描述精度不够**（新教训 A13）

**问题**：seg-2 v1 小桃望风描述是"紧张地左右张望"——太笼统，模型理解为"闲散站着"。v2 改为"脸色苍白+死盯小木门+攥袖口发白+侧身探头"后效果显著提升。

**根因**：形容词（"紧张"、"担忧"）比动词（"死盯"、"攥"）对模型弱。模型需要**可观察的具体动作**。

**改进**：描述配角情绪时用"动词+身体部位+强度"三件套：
- ❌ "紧张地望风"
- ✅ "死死盯着某方向 + 双手攥紧袖口 + 侧身探头"

这条已经在 seg-3 ~ seg-11 的分镜描述中应用，效果稳定。

#### 3. 🟡 **分镜格式应一次性定好**

**问题**：分镜从 Prompt 纯文本 → 加字数 → 加"镜头设计（详细分镜）" 小节，连续升级 3 次才稳定。

**根因**：用户需求浮现是渐进式的，我没有从 EP03 结束就预见到。

**改进**：EP05 开始时确认最终分镜模板：
```markdown
### seg-N
**元信息**（时长/场景/首帧/剧本时间段/画面中角色/说话角色/镜头意图/对白）

#### 镜头设计（详细分镜）
**节拍 K（时间段）：节拍名称**
- 景别/机位/主体位置/运镜
- 画面元素
- 角色动作
- 对白（若有）
- 镜头意图
- 音效

#### Prompt (vN, 当前)
[Seedance prompt]

**字数统计**：正文 N 字 / 完整 M 字（含固定头 X 字）✅/⚠️

#### ref-images / audio / 命令参数 / 版本历史
```

#### 4. 🟢 **字数统计从 EP05 应该预填**

现在每个 seg 生成后才填字数。应改为**组装 prompt 时同步填**（因为此时已知 prompt 全文）。

### 问题根因

1. **阶段间协同不够**：剧本层用自然语言（"庭院"），场景库层用文档（"殿内"），阶段 07 组装 prompt 时我未交叉核对
2. **配角重要性评估不足**：小桃的望风戏是整个 seg-2 的情感锚点，但我当做背景动作处理
3. **模板成熟度曲线**：每个规则变化都是合理响应，但累积起来让早期 seg 重做
