## Shell / FFmpeg

### 1. while read + ffmpeg stdin 吞噬（2026-05-07）

**问题**：`while IFS= read -r line; do ffmpeg ...; done <<< "$LIST"` 中，ffmpeg 会从 heredoc 的 stdin 读取字符，导致下一轮 `read` 拿到残缺内容。

**修复**：循环中所有会读 stdin 的命令加 `</dev/null`。

**适用**：任何 shell 循环中调用 ffmpeg、ssh、docker exec 等会读 stdin 的程序。

### 2. ffmpeg 重编码必须显式指定编码参数（2026-05-07）

**问题**：`-vf subtitles=...` 触发视频流重编码，不指定 `-c:v libx264 -crf 18` 会退到默认 CRF 23，码率减半画质下降。

**修复**：任何 ffmpeg 重编码命令必须显式写 `-c:v libx264 -preset medium -crf 18`（或项目约定的参数）。

**验证**：合成后跑码率自检，final 与 raw 偏差 ≤ 5%。

**适用**：所有 ffmpeg 滤镜处理（字幕烧录、缩放、水印叠加等）。

## Whisper 字幕对齐

### 3. 段级对齐精度不够，必须用词级（2026-05-07）

**问题**：Whisper 段级（segment）时间戳精度 ±2-19s（因为 VAD 合并多句对白为一个 segment，标点拆分只能线性分配）。

**修复**：使用 `word_timestamps=True`，在字流中做模糊子串匹配，精度 ±0.1s。

**适用**：任何需要字幕与音频精确对齐的场景。

### 4. Whisper 中文默认输出繁体（2026-05-07）

**问题**：faster-whisper 中文识别默认输出繁体字，与简体剧本做相似度匹配时分数偏低。

**修复**：匹配前用 `zhconv.convert(text, 'zh-cn')` 统一转简体。配合 `initial_prompt="以下是普通话的简体中文对白。"` 可提升识别准确度。

**适用**：所有中文 ASR + 文本匹配场景。

## 工作流执行

### 5. 规则的约束对象必须明确区分（2026-05-07）

**问题**：规则 4"单 seg 最多 2 个角色同框"被误解为"ref-images 只传 2 个角色"。实际它约束的是画面构图（面部特写同框数），不约束 API 传参。

**教训**：执行规则前先确认它约束的是什么层面：
- **构图层面**：画面中谁的脸需要清晰 → 影响 prompt 写法和景别选择
- **传参层面**：ref-images 传什么 → 影响 API 调用参数
- **命名层面**：文件怎么命名 → 影响目录结构

**适用**：所有多层级规则体系（不限于视频生成）。

### 6. 工作流变更后信心度等级必须回退 P0（2026-05-07）

**问题**：工作流规则更新后如果沿用旧的信心度等级（P1/P2），新规则未经验证就进入批处理模式，可能产出不符合新规则的内容。

**修复**：`_convention.md` 新增"工作流变更降级"规则——任何阶段文件/prompt-guide/_convention 修改后，信心度强制回退 P0。

**适用**：任何有"自动化等级"的流程（CI/CD 的 canary 发布、A/B 测试的流量比例等同理）。

### 7. 批量生成前先写入段详情（方案 A）（2026-05-07）

**问题**：如果生成中断（API 超时、会话断开），未登记的 prompt 无法追溯复现。

**修复**：`batch_gen.py` 在提交任何任务之前，先把所有 seg 的完整 prompt + ref-images + audio 写入 assets.md。生成完成后只更新 task_id/seed/状态。

**适用**：任何"先规划后执行"的批量任务（不限于视频生成——批量图片生成、批量 API 调用等）。

### 8. 群演参考图按配额剩余决定传入（2026-05-07）

**问题**：群演参考图（众嫔妃）不是每段必传，但也不是完全不传。

**规则**：ref-images 上限 9 张。主要角色必传，群演参考图在配额剩余时尽量传入，接近上限时可省略。

**适用**：任何有"必选 + 可选"资源配额的场景。

## seg-config 组装

### 10. chars 必须包含画面中所有可辨识角色，不只是叙事焦点（2026-05-07）

**问题**：组装 seg-config.py 的 `chars` 列表时，只传入了"叙事焦点角色"（谁是主体、谁在说话），遗漏了画面中可辨识但非焦点的角色（如小桃在背景跺脚/捂脸/探头）。导致这些角色没有传入 multiview 参考图，生成时外观不可控。

**根因**：将"画面中出现"（视觉层面的规则）偷换为"叙事焦点"（剧本层面的判断）。本质是教训 #5 的同类错误——规则约束对象混淆。

**修复**：组装每个 seg 的 chars 时，逐行扫描 prompt 文本，凡是提到名字或描述了动作的角色库角色，一律加入 chars。判断标准是"画面中可辨识"而非"谁是主角"。

**常见遗漏场景**：
- 角色在背景中跺脚/捂脸/探头/跑过
- 角色作为施动者推/拉/搀扶主角
- 角色在前景虚化但可辨识

**例外**：只有画外音、不在画面中的角色 → 不加 chars，只加 audio。

**已固化**：07-video.md 新增"chars 自检规则（强制）"段落 + chars 字段规则已更新。

**适用**：任何需要"列出画面中出现的实体"的场景（不限于视频生成——图片生成的角色列表、场景中的道具列表等同理）。

### 11. 图片引用必须内联到主体首次出现位置，且不重复标注（2026-05-07）

**问题**：prompt 中的图片引用被单独列为一段（如"场景参考图片1。沈鹿溪形象参考图片2。"），或者在时间段节拍中重复标注图片编号。这两种写法都不符合 Seedance 的最佳实践——模型需要在上下文中理解"这张图对应谁"，单独列举会割裂语义关联。

**修复**：
1. 图片编号在该主体**第一次出现**的位置内联标注，融入描述句中
2. 同一主体只标注一次，后续提到时直接用名字
3. 标准格式：角色用 `{角色名}（{拼音}）（图片N）`，场景用 `{场景名}（图片N）`

**示例**：
- ✅ `选秀大殿（图片1）内，萧珩（Xiāo Héng）（图片3）坐在高台上...`
- ❌ `场景参考图片1的构图和光影。萧珩形象参考图片3。`（单独成段）
- ❌ `0-4s：中远景，萧珩（图片3）高台端坐...`（时间段中重复标注）

**已固化**：prompt-guide.md §四「图片指代规则」已更新，07-video.md prompt 模板已同步。

**适用**：所有使用 ref-images 的生成 prompt（Seedance 视频、Seedream 图生图）。

### 12. 剧本合理性检查必须在阶段 05 验收时完成，不能拖到阶段 07（2026-05-07）

**问题**：EP03 剧本中小桃（侍女）出现在集体请安大殿的跪拜队列中，直到阶段 07 组装 prompt 时才发现违和。导致 seg-config 和 assets.md 返工。

**根因**：阶段 05 验收清单只检查结构/时长/对白格式，没有检查"角色出现在该场景中是否合理"。

**修复**：05-script.md 验收清单新增"场景合理性检查"子清单，包含：角色出场合理性、空间逻辑、时间逻辑、行为动机。

**适用**：任何有"角色×场景"组合的创作流程。越早发现逻辑漏洞，返工成本越低。

### 13. 剧本-分镜-prompt 三层质量提升框架（2026-05-07）

**背景**：EP03 生成的视频效果不好，追溯发现根本原因不在 prompt，而在剧本层面——"镜头语义"字段太粗，潜台词覆盖不全，情绪弧不清，导致 prompt 组装时要临时发挥，画面随机。

**三层定位**：
1. **剧本层（05-script）**：吸引人 — 钩子/反差/悬念/情感/信息密度五杠杆自检
2. **分镜层（不独立成文档）**：忠实剧本 — 剧本每场戏加"镜头意图"字段，prompt 自检"镜头意图覆盖""潜台词视觉化"
3. **prompt 层（07-video）**：精准表达 — 每节拍强制"景别+机位+主体位置+运镜(带叙事意图)"

**关键设计**：
- 分镜信息固化在剧本的"镜头意图"字段和 prompt 的强制字段里，不新增独立资产
- 五杠杆吸引力自检要求 AI 自打分（1-10），低于 6 分必须给出改进方案
- 潜台词覆盖率 ≥ 80% 作为剧本验收硬门禁

**为什么不用独立分镜文档**：9 张 ref-images 上限下草图方案不可行；独立文字分镜清单会拖慢流程。把分镜信息分散到"剧本字段"和"prompt 字段"两处强制，既有审核锚点又不额外增加资产。

**已固化**：
- 05-script.md：新增"镜头意图"模板字段、"吸引力自检"段落（5 杠杆+自打分）、验收清单新增 3 条
- 07-video.md：Prompt 模板每节拍新增 4 个必填字段（景别+机位+主体位置+运镜）、Step 6 自检新增"剧本忠实度自检"子项

**适用**：所有"剧本 → 视觉生成"的创作流程。核心思想是把"凭感觉发挥"替换为"强制思考维度"。

### 14. 全局约束必须下沉到每个阶段的验收清单，否则形同虚设（2026-05-07）

**问题**：`_convention.md` 的"设计原则"里写了"每次新建/修改/删除资产后同步更新 ASSETS-INDEX"，但各阶段执行时只看阶段文件的验收清单，没有看全局设计原则。结果 EP03 新建了廊下场景后漏了更新 ASSETS-INDEX。

**根因**：**"全局约束"和"阶段执行"是两层不同的认知**。写在全局的规则是"背景知识"，AI 执行具体任务时不会主动触发；写在阶段验收清单里的规则才是"执行时必查项"。

**修复**：
1. **升级为门禁 5**：_convention.md 的"四大门禁"改为"五大门禁"，新增 ASSETS-INDEX 门禁
2. **下沉到阶段验收**：在会产生资产的每个阶段的验收清单显式加 `[ ] ASSETS-INDEX.md 已同步更新`（02/04/06/07/08）

**适用**：任何"全局规则 × 分阶段执行"的工作流设计。全局规则必须在每个阶段的 checklist 里有对应条目，否则执行时会漏。

**已固化**：
- _convention.md：新增门禁 5
- 02-moodboard.md / 04-character.md / 06-scenes.md / 07-video.md / 08-composite.md：验收清单加入 ASSETS-INDEX 同步项

### 15. prompt 组装必须双写（seg-config.py + assets.md）（2026-05-08）

**问题**：P0 工作流 Step 4 组装 prompt 只写进了 seg-config.py，没有同步到 assets.md 的段详情块。结果 seg-1 生成完成后，assets.md 的段详情块里 prompt 还是"_待 Step 4 组装_"占位符，直到被用户指出才补填。

**根因**：seg-config.py 是"执行输入"，assets.md 是"审计凭证"（门禁 2 的核心作用）。把 prompt 只放执行层没放审计层 = 会话中断时 prompt 原文丢失。

**修复**：07-video.md 的 P0 Step 4 明确要求 prompt 必须**双写**：
- `seg-config.py` 的 `prompt` 字段（执行输入）
- `assets.md` 段详情块的 Prompt 区段 + ref-images 映射表 + audio + 命令参数（审计凭证）

Step 7 只需补 task_id + seed + 版本历史到 assets.md，prompt 已在 Step 4 写入。

**术语区分**（避免再搞混）：
- **ASSETS-INDEX.md** = 资产速查表（项目级，目录名/状态/数量粒度）
- **assets.md** = 资产清单（目录级，每张图/每段视频的完整 prompt + task_id + 参数）

**已固化**：07-video.md P0 Step 4 / Step 7 描述已更新。

**适用**：任何"执行配置 ≠ 审计记录"的工作流。如果两处数据源不同步，审计必然漏。必须强制双写或工具化单源生成。

### 16. 对白/动作覆盖自检必须以剧本正文为准，不能只查汇总表（2026-05-08）

**问题**：EP03 剧本第 3-30s 段的画面描述里有一句功能性对白"太监宣布：皇上有旨，新入宫嫔妃逐一上前请安"，但剧本末尾的**对白汇总表**没有收录它（作者把它当成叙事性描述，没附 emotion/speed/loudness 等字段）。我在 07-video Step 3 的"对白覆盖"自检时只对照汇总表做检查——表里没有就通过了，结果 seg-2 和 seg-3 的 prompt 都漏了这句。导致成片里沈鹿溪"突然独自走到殿中央"失去了"被点名出列"的触发因。

**根因**：和教训 #5 / #10 是同类错误——**自检的约束对象被偷换了**。
- 本应约束：剧本**正文**里所有画面内对白和关键动作
- 实际约束：剧本**汇总表**里的结构化条目

汇总表是审计凭证，但**它不是全集**——作者可能把功能性/叙事性对白（太监宣旨、群体台词、画外旁白、环境性 NPC 对话）留在正文不登记，因为这类对白没有"情感标注"需要填。

**修复**：
1. **05-script.md（剧本阶段）**：新增"对白正文-汇总表一致性"验收项——汇总表对白数 = 正文中所有 `"..."`（或引号对白）的数量；差异必须显式解释（如"太监宣旨为功能性对白，不附情感标注"）
2. **07-video.md Step 3（seg 切分自检）**：对白覆盖检查改为**双源对照**——先扫描剧本正文里所有对白（功能性+情感性），再对照 seg 切分是否覆盖所有对白
3. **07-video.md Step 3 新增"功能性对白处理规则"**：
   - 功能性对白（宣旨/旁白/广播）在 seg 切分时必须明确归属（哪个 seg 的哪个节拍）
   - 归属位置应符合剧情因果（如"宣旨 → 出列"必须"宣旨 seg 在前，出列 seg 在后"或"同 seg 内宣旨节拍在前"）

**固化位置**：
- 05-script.md 验收清单（场景合理性检查上方）
- 07-video.md Step 3 自检清单（对白覆盖项）

**适用**：任何"结构化表格 + 正文叙事"并存的审计流程——不能只信表格，表格可能是正文的"子集"而非"全集"。
