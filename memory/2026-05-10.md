## 4. 经验教训

### 做得好的

1. **规则固化的反射弧足够快**：发现问题→讨论→更新 MEMORY→更新阶段文档 checklist，当天内完成，不留技术债
2. **备份策略稳妥**：每个 seg 重做前都备份到 `_backup/seg{N}_v{M}/`，方便对比回溯
3. **对齐精度新高**：Whisper 词级对齐 10/11 成功，比 EP01/02 的段级对齐精度大幅提升
4. **A7 教训在 seg-10 立竿见影**：小桃→侍女的情绪基调切换用"镜头拉远穿过廊柱"作为视觉触发因，一次通过
5. **规则 10 验证成功**：seg-7 v4 (6s) → v5 (8s) 对比明显，过渡节拍有了叙事空间

### 可改进的

1. **seg-7 v1 应该一开始就判不通过**
   - v1 明显有嫔妃倒退走，但我报验收时没明确指出
   - 用户来复核才发现→浪费一次迭代的信任度
   - **改进**：AI 生成后自己先看一眼再报给用户，明显问题不要让用户发现

2. **空间词和画面词的混淆导致 seg-7 v2 失败**
   - "殿深处"在剧本里是"朝大殿外面"的意思，但模型理解成"大殿内深处"（龙椅方向）
   - **这是 M1 元教训的新形态**——"建筑空间语义" vs "画面坐标语义"混淆
   - **改进**：已在 seg-7 v3/v6 中确立原则"群体运动方向优先用动作动词而非空间位置词"

3. **seg-6 散场触发因应该一开始就在剧本里**
   - 剧本初版只有"朕知道了"，没有"回宫吧"散场触发因
   - 到 seg-7 反复生成失败才补救
   - **改进**：阶段 05 剧本编写时就应有"情境转换触发因覆盖"自检（A7 已固化）

4. **服务端堵塞时的策略**
   - 中间遇到服务端堵塞（seg-7 v3 耗时 11 分钟，seg-7 v5 耗时 5 分钟）
   - 之前习惯高频查 `getProcessOutput`，浪费 token
   - **已改进**：用 `sleep N && status` 主动控制节奏

5. **prompt 描述过度精简的代价**
   - seg-7 v3 把 v1 的景别承接节拍删掉，导致和 seg-6 断层
   - 精简 prompt 不能以牺牲功能节拍为代价
   - **改进**：精简原则应从"减少字数"改为"减少歧义描述"

### 问题根因

1. **P0 工作流执行深度不够**：P0 本应是"逐 seg 严格把关"，但 seg-7 v1 报通过说明我对"验收通过"的标准太宽。**今后 P0 每 seg 我先自己视觉评估一遍再报审**

2. **剧本和视频生成的质量耦合不紧**：seg-6→seg-7 散场衔接问题本质是剧本层缺失，不是视频层问题。**阶段 05 和 07 之间的验收衔接需要强化**（A7 已固化）
